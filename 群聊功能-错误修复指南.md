# 群聊功能错误修复指南

## 当前错误

### 1. GroupMembersModal.vue 加载失败 (500 错误)
```
Failed to load resource: the server responded with a status of 500 (Internal Server Error)
```

### 2. Vue Router 动态导入失败
```
[Vue Router warn]: uncaught error during route navigation
TypeError: Failed to fetch dynamically imported module
```

## 解决方案

### 方案 1: 重启开发服务器（推荐）

1. **停止当前开发服务器**
   - 按 `Ctrl + C` 停止
   
2. **清理缓存和重新安装依赖**
   ```bash
   # 删除 node_modules 和 out 文件夹
   rm -rf node_modules out
   
   # 重新安装依赖
   npm install
   ```

3. **重新启动开发服务器**
   ```bash
   npm run dev
   ```

### 方案 2: 清理 Vite 缓存

1. **删除 Vite 缓存**
   ```bash
   # 删除 .vite 缓存文件夹
   rm -rf node_modules/.vite
   ```

2. **重启开发服务器**
   ```bash
   npm run dev
   ```

### 方案 3: 硬刷新浏览器

1. 在 DevTools 中按 `Ctrl + Shift + R` (Windows) 或 `Cmd + Shift + R` (Mac)
2. 或者右键点击刷新按钮，选择"清空缓存并硬性重新加载"

### 方案 4: 检查文件编码

确保所有 `.vue` 文件使用 UTF-8 编码（无 BOM）

## 常见问题

### Q: 为什么会出现 500 错误？
A: 通常是因为：
- Vite 开发服务器缓存了旧版本的文件
- 文件在保存时被截断或损坏
- 热重载失败

### Q: 为什么动态导入会失败？
A: 可能原因：
- 路由配置中的路径错误
- 文件不存在或路径不正确
- 开发服务器需要重启

### Q: 重启后还是有错误怎么办？
A: 尝试以下步骤：
1. 检查文件是否完整（没有被截断）
2. 检查文件编码是否为 UTF-8
3. 检查是否有语法错误
4. 运行 `npm run typecheck:web` 检查类型错误

## 文件完整性检查

### GroupMembersModal.vue

- `<script setup lang="ts">` 标签

- 定义 `sortedMembers` computed

文件应该包含：
- `<template>` 标签（完整的 HTML 结构）
- `<script setup lang="ts">` 标签

## 快速修复命令

```bash
# Windows PowerShell
Remove-Item -Recurse -Force node_modules\.vite
npm run dev

# 或者完全重新安装
Remove-Item -Recurse -Force node_modules, out
npm install
npm run dev
```

## 预防措施

1. **定期清理缓存**
   - 每次大改动后清理 Vite 缓存
   
2. **使用版本控制**
   - 确保所有文件都已提交到 Git
   - 可以随时回滚到工作版本

3. **增量开发**
   - 每次只修改少量文件
   - 及时测试验证

4. **保存文件完整性**
   - 确保文件保存完成后再刷新
   - 避免在保存过程中中断

---

**建议**: 先尝试方案 1（重启开发服务器），这通常能解决 90% 的问题。
