# 在线状态实时推送功能完成总结

## 📋 需求回顾

**用户反馈**：
> "前端好友列表要手动刷新才能更新在线状态，不好。如果有一方退出了，前端要能快速更新这个状态。"

**核心需求**：
- 好友上线时，自动显示在线（无需手动刷新）
- 好友离线时，自动显示离线（无需手动刷新）
- 实时性强，延迟 < 1 秒
- 多个位置同步更新（好友列表、会话列表、聊天头部）

## ✅ 已完成的工作

### 1. 前端实现

#### WebSocket 订阅

在 `websocket.ts` 中添加了在线状态订阅：

```typescript
/**
 * 订阅在线状态变化通知
 */
private subscribeToOnlineStatus() {
  const subscription = this.client.subscribe(`/user/queue/online-status`, (message) => {
    this.handleOnlineStatusChange(JSON.parse(message.body))
  })
  this.subscriptions.set('online-status', subscription)
}
```

#### 状态变化处理

```typescript
/**
 * 处理在线状态变化通知
 */
private async handleOnlineStatusChange(notification: any) {
  // 1. 更新联系人列表
  const contact = contactStore.contacts.find(c => c.userId === notification.userId)
  if (contact) {
    contact.isOnline = notification.isOnline
  }
  
  // 2. 更新会话列表（私聊）
  const session = sessionStore.sessions.find(s => 
    s.sessionUserType === 'PRIVATE' && s.userId === notification.userId
  )
  if (session) {
    session.isOnline = notification.isOnline
  }
}
```

#### 自动订阅

WebSocket 连接成功后，自动订阅在线状态通知：

```typescript
private subscribeToPrivateMessages() {
  // 订阅私有消息
  // 订阅好友请求
  // 订阅在线状态 ← 新增
  this.subscribeToOnlineStatus()
}
```

```java
@Service
public class OnlineStatusNotificationService {
    
    /**
     * 通知所有在线好友：某用户上线了
     */
    public void notifyFriendsUserOnline(String userId, String username) {
        // 1. 查询该用户的所有好友
        List<String> friendIds = contactService.getFriendIds(userId);
        
        // 2. 构建通知消息
        OnlineStatusNotification notification = OnlineStatusNotification.builder()
            .userId(userId)
            .username(username)
            .isOnline(true)
            .timestamp(System.currentTimeMillis())
            .build();
        
        // 3. 向所有在线好友推送通知
        for (String friendId : friendIds) {
            if (onlineStatusService.isUserOnline(friendId)) {
                messagingTemplate.convertAndSendToUser(
                    friendId,
                    "/queue/online-status",
                    notification
                );
            }
        }
    }
}
```

## 🔄 完整工作流程

### 好友上线流程

```
1. 用户 A 登录
   ↓
2. 后端 AuthService.login()
   ↓
3. 标记用户 A 在线
   Redis: SET online:user:{userA}
   ↓
4. 查询用户 A 的所有好友
   假设有好友 B、C、D
   ↓
5. 检查好友是否在线
   B: 在线 ✅
   C: 离线 ❌
   D: 在线 ✅
   ↓
6. 向在线好友推送通知
   → 推送给 B: {"userId":"A","isOnline":true}
   → 推送给 D: {"userId":"A","isOnline":true}
   ↓
7. 好友 B、D 的前端收到 WebSocket 消息
   ↓
8. handleOnlineStatusChange() 处理通知
   ↓
9. 更新 Contact Store 和 Session Store
   contact.isOnline = true
   session.isOnline = true
   ↓
10. Vue 响应式系统自动更新 UI
   ↓
11. 好友 B、D 看到 A 在线（青色圆点 + "在线"文字）
```

### 好友离线流程

```
1. 用户 A 退出登录
   ↓
2. 后端 AuthService.logout()
   ↓
3. 标记用户 A 离线
   Redis: DEL online:user:{userA}
   ↓
4. 查询用户 A 的所有好友
   ↓
5. 向在线好友推送通知
   → 推送给 B: {"userId":"A","isOnline":false}
   → 推送给 D: {"userId":"A","isOnline":false}
   ↓
6. 好友 B、D 的前端收到通知
   ↓
7. 自动更新 UI
   ↓
8. 好友 B、D 看到 A 离线（灰色圆点 + "离线"文字）
```
