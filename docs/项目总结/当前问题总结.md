# 当前问题总结

## 问题现象

点击联系人"lyh1"时，出现错误：
```
Uncaught (in promise) TypeError: Cannot read properties of null (reading 'Q')
```

## 根本原因

**后端返回的联系人数据中 `userId` 字段为 `null` 或缺失**

## 数据流程

1. 前端调用 `GET /api/contact/list` 获取联系人列表
2. 后端返回数据（问题出在这里）：
   ```json
   {
     "id": "relation-123",
     "userId": null,        // ❌ 问题：应该是好友的用户ID
     "username": "lyh1",
     "showName": "在线"
   }
   ```
3. 用户点击联系人
4. 前端尝试使用 `contact.userId` 创建会话
5. 因为 `userId` 是 `null`，导致错误

## 前端已做的修复

✅ 添加了数据验证  
✅ 添加了错误提示  
✅ 过滤掉没有 `userId` 的联系人  

## 后端需要修复

### 必需修复（优先级：高）

**接口**: `GET /api/contact/list`

**问题**: 返回的数据中 `userId` 为 `null`

**修复**: 确保返回好友的用户ID

**正确的返回格式**:
```json
{
  "code": 200,
  "message": "success",
  "data": [
    {
      "id": "relation-uuid-123",
      "userId": "friend-user-uuid-456",  // ⚠️ 必需：好友的用户ID（不是关系ID）
      "username": "lyh1",
      "showName": "在线",
      "avatar": "",
      "remark": null,
      "addedTime": "2024-02-27T10:00:00",
      "publicKey": "..."
    }
  ]
}
```

### 实现建议

```java
public List<ContactRelationVO> getContactList(String currentUserId) {
    List<ContactRelation> relations = contactRelationRepository
        .findByUserIdAndStatus(currentUserId, RelationStatus.ACCEPTED);
    
    return relations.stream()
        .map(relation -> {
            // 获取好友的用户ID（不是当前用户的ID）
            String friendUserId = relation.getFromUserId().equals(currentUserId) 
                ? relation.getToUserId() 
                : relation.getFromUserId();
            
            // 查询好友信息
            User friend = userRepository.findById(friendUserId).orElseThrow();
            
            ContactRelationVO vo = new ContactRelationVO();
            vo.setId(relation.getId());
            vo.setUserId(friend.getId());          // ⚠️ 设置好友的用户ID
            vo.setUsername(friend.getUsername());
            vo.setShowName(friend.getShowName());
            vo.setAvatar(friend.getAvatar());
            // ...
            
            return vo;
        })
        .collect(Collectors.toList());
}
```
