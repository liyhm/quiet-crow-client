# WebSocket 重连测试指南

## 🎯 快速测试步骤

### 测试 1: 后端重启自动重连（5 分钟）

#### 步骤

1. **启动前端和后端**
   - 前端正常运行
   - 后端正常运行
   - 登录一个账号

2. **验证初始状态**
   - 打开浏览器控制台（F12）
   - 查看是否有：`✅ WebSocket 连接成功！`
   - 发送一条消息，验证正常

3. **关闭后端**
   - 停止后端服务（Ctrl+C）
   - 观察控制台输出

4. **观察重连尝试**
   - 应该看到：
     ```
     ❌ WebSocket 断开连接
     🔄 准备重连 WebSocket (尝试 1/15)，2 秒后重试...
     🔄 开始第 1 次重连...
     ❌ 重连失败
     🔄 准备重连 WebSocket (尝试 2/15)，4 秒后重试...
     ```

5. **重新启动后端**
   - 启动后端服务
   - 等待几秒

6. **验证自动重连**
   - 应该看到：
     ```
     🔄 开始第 X 次重连...
     🔌 先停用现有连接
     🔄 重新初始化 WebSocket 客户端
     📡 激活 WebSocket 客户端
     ✅ WebSocket 连接成功！
     📡 订阅私有消息队列: /user/queue/messages
     ✅ 私有消息队列订阅成功
     ```

7. **验证功能恢复**
   - 发送一条消息
   - 接收一条消息
   - 查看好友列表在线状态

#### 预期结果

- ✅ 后端重启后，前端在 30 秒内自动重连
- ✅ 重连成功后，消息收发正常
- ✅ 在线状态正确显示
- ✅ 无需手动刷新页面

#### 如果失败

- 检查后端是否完全启动（等待 10-15 秒）
- 检查控制台是否有错误信息
- 如果超过 15 次重连仍失败，手动刷新页面

---

### 测试 2: 在线状态显示（3 分钟）

#### 步骤

1. **准备两个账号**
   - 账号 A: lyh1
   - 账号 B: lyh2
   - 确保两个账号已互为好友

2. **账号 A 登录**
   - 使用账号 A 登录
   - 打开好友列表
   - 此时账号 B 应该显示离线（灰色圆点）

3. **账号 B 登录**
   - 在另一个浏览器/窗口使用账号 B 登录
   - 等待 1-2 秒

4. **验证账号 A 的好友列表**
   - 刷新账号 A 的好友列表（点击联系人标签）
   - 账号 B 应该显示在线（青色圆点 + "在线"文字）

5. **验证账号 B 的好友列表**
   - 查看账号 B 的好友列表
   - 账号 A 应该显示在线

6. **账号 B 退出登录**
   - 点击退出登录
   - 等待 1-2 秒

7. **验证账号 A 的好友列表**
   - 刷新账号 A 的好友列表
   - 账号 B 应该显示离线（灰色圆点 + "离线"文字）

#### 预期结果

- ✅ 登录后立即显示在线
- ✅ 退出后立即显示离线
- ✅ 在线状态实时更新
- ✅ 好友列表、会话列表、聊天头部都正确显示

#### 如果失败

**问题 1: 登录后还是显示离线**
- 等待 2-3 秒再查看
- 检查后端日志是否有 "🟢 [WebSocket] 用户 xxx 已连接"
- 检查 Redis: `redis-cli GET online:user:{userId}`
- 如果 Redis 中没有数据，重启后端

**问题 2: 退出后还是显示在线**
- 检查前端控制台是否有 "✅ WebSocket 已断开"
- 检查 Redis: `redis-cli GET online:user:{userId}` 应该为空
- 如果还有数据，检查后端日志

---

### 测试 3: 退出登录（2 分钟）

#### 步骤

1. **登录账号**
   - 使用任意账号登录
   - 打开控制台（F12）

2. **验证 WebSocket 连接**
   - 应该看到：`✅ WebSocket 连接成功！`

3. **验证在线状态**
   - 打开 Redis 客户端
   - 执行：`redis-cli GET online:user:{userId}`
   - 应该返回一个时间戳

4. **点击退出登录**
   - 点击头像 → 退出登录
   - 观察控制台输出

5. **验证 WebSocket 断开**
   - 应该看到：
     ```
     🚪 开始登出...
     🔌 断开 WebSocket 连接...
     📡 取消订阅: private
     📡 取消订阅: friend-requests
     ✅ WebSocket 已断开
     ✅ 已清除所有会话密钥
     ✅ 登出完成
     ```

6. **验证离线状态**
   - 执行：`redis-cli GET online:user:{userId}`
   - 应该返回 `(nil)`（空）

7. **验证不会重连**
   - 等待 10 秒
   - 控制台不应该有重连日志
   - 不应该看到 "🔄 准备重连 WebSocket"

#### 预期结果

- ✅ WebSocket 正确断开
- ✅ 在线状态正确清除
- ✅ 不会触发重连
- ✅ 退出流程流畅无卡顿

#### 如果失败

**问题 1: WebSocket 没有断开**
- 检查 `MainPage.vue` 的 `handleLogout` 函数
- 确保调用了 `websocketService.disconnect()`

**问题 2: 退出后还显示在线**
- 检查后端日志是否有 "🔴 [WebSocket] 用户 xxx 已断开"
- 检查 Redis 数据是否清除
- 可能是 WebSocket 没有正确断开

**问题 3: 退出后触发重连**
- 检查 `websocketService.disconnect()` 是否清理了重连定时器
- 检查是否在 `disconnect()` 后又调用了 `connect()`

## 🔍 调试技巧

### 1. 查看 WebSocket 连接状态

**控制台命令**：
```javascript
// 在浏览器控制台执行
console.log('WebSocket 状态:', websocketService.client?.connected)
```

### 2. 查看 Redis 在线状态

**Redis 命令**：
```bash
# 查看单个用户
redis-cli GET online:user:{userId}

# 查看所有在线用户
redis-cli KEYS online:user:*

# 查看在线用户数量
redis-cli KEYS online:user:* | wc -l
```

### 3. 查看后端日志

**关键日志**：
```
🟢 [WebSocket] 用户 xxx 已连接，标记为在线
🔴 [WebSocket] 用户 xxx 已断开，标记为离线
🟢 用户 xxx 已标记为在线
🔴 用户 xxx 已标记为离线
```

### 4. 强制触发重连

**控制台命令**：
```javascript
// 强制断开 WebSocket
websocketService.disconnect()

// 等待 2 秒后重新连接
setTimeout(() => {
  websocketService.initialize()
  websocketService.connect()
}, 2000)
```

## ⚠️ 常见问题

### Q1: 后端重启后，前端还是显示离线？

**A**: 等待 2-3 秒，WebSocket 重连需要时间。如果超过 30 秒还是离线，手动刷新页面。

### Q2: 退出登录后，还显示在线？

**A**: 检查前端是否正确断开 WebSocket。查看控制台是否有 "✅ WebSocket 已断开"。

### Q3: 重连一直失败？

**A**: 检查后端是否完全启动。后端启动可能需要 10-15 秒。

### Q4: 在线状态不更新？

**A**: 
1. 检查后端日志是否有 WebSocket 连接/断开日志
2. 检查 Redis 中的在线状态数据
3. 刷新好友列表（点击联系人标签）

### Q5: 控制台没有重连日志？

**A**: 
1. 检查 WebSocket 是否真的断开了
2. 检查是否在退出登录（退出登录不会重连）
3. 检查是否达到最大重连次数
