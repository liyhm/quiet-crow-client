## 可能的问题和解决方案

### 问题1: 点击联系人后没有反应

**症状**: 点击联系人后，没有切换到消息标签，控制台显示错误

**可能原因**:
1. 后端没有实现 `POST /api/session/private` 接口
2. 接口返回的数据格式不正确
3. 网络错误

**解决方案**:
1. 检查控制台的错误信息
2. 检查网络请求（F12 → Network 标签）
3. 确认后端接口是否实现

**需要后端实现**:
```java
@PostMapping("/private")
public Result<SessionResponse> createPrivateSession(@RequestBody CreatePrivateSessionRequest request) {
    // 实现创建私聊会话的逻辑
}
```

### 问题2: 会话创建成功，但列表仍然为空

**症状**: 控制台显示"会话创建成功"，但会话列表仍然显示"暂无会话"

**可能原因**:
1. `GET /api/session/my` 返回的数据格式不正确
2. 会话创建后，`session_user` 表没有正确插入记录

**解决方案**:
1. 查看控制台中"获取到会话数据"的日志
2. 检查后端数据库中的 `session` 和 `session_user` 表
3. 确认 `GET /api/session/my` 返回的数据格式

**预期返回格式**:
```json
{
  "code": 200,
  "message": "success",
  "data": [
    {
      "id": "session-user-uuid",
      "sessionId": "session-uuid",
      "userId": "target-user-id",
      "showName": "鲍勃",
      "sessionUserType": "PRIVATE"
    }
  ]
}
```

### 问题3: 发送消息失败

**症状**: 输入消息后按 Enter，消息没有显示，控制台显示错误

**可能原因**:
1. WebSocket 未连接
2. 后端 WebSocket 处理器未实现
3. 消息格式不正确

**解决方案**:
1. 检查 WebSocket 连接状态（控制台应该显示"已连接"）
2. 确认后端 WebSocket 处理器已实现
3. 查看控制台的详细错误信息

## 调试技巧

### 1. 查看网络请求

1. 打开浏览器开发者工具（F12）
2. 切换到"Network"标签
3. 点击联系人
4. 查看 `POST /api/session/private` 请求：
   - 请求参数是否正确
   - 响应状态码（200 = 成功，404 = 未找到，500 = 服务器错误）
   - 响应数据格式

### 2. 查看控制台日志

前端已经添加了详细的日志输出，包括：
- 📞 选择联系人
- 🔗 API 调用
- ✅ 成功信息
- ❌ 错误信息

### 3. 检查后端日志

查看后端控制台或日志文件，确认：
- 是否收到了 `POST /api/session/private` 请求
- 请求参数是否正确
- 是否有异常或错误

### 4. 检查数据库

使用数据库工具查看：
- `session` 表：是否创建了新会话
- `session_user` 表：是否为两个用户都创建了记录
- `user` 表：用户ID是否正确

## 后端需要实现的接口

### 必需接口

1. **创建私聊会话**
   - 端点: `POST /api/session/private`
   - 请求: `{targetUserId: "user-id"}`
   - 响应: `{sessionId: "...", type: "PRIVATE", participants: [...], exists: false}`

2. **获取会话列表**
   - 端点: `GET /api/session/my`
   - 响应: `[{sessionId: "...", userId: "...", showName: "...", sessionUserType: "PRIVATE"}]`

3. **WebSocket 消息处理**
   - 接收消息并保存到数据库
   - 转发消息给会话中的其他用户

### 可选接口

1. **获取历史消息**
   - 端点: `GET /api/session/{sessionId}/messages`
   - 响应: 消息列表

2. **标记消息已读**
   - 端点: `POST /api/session/{sessionId}/read`
