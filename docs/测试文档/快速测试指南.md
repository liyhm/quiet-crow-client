# 快速测试指南

## 测试目的
验证后端修改后，消息收发和已读功能是否正常工作。

---

## 准备工作

### 1. 确保后端运行
```bash
# 后端应该运行在
http://localhost:8080
```

### 2. 准备两个测试账号
- 用户 A（发送方）
- 用户 B（接收方）

### 3. 打开两个浏览器窗口
- 窗口 1：登录用户 A
- 窗口 2：登录用户 B

---

## 测试步骤

### 测试 1：WebSocket 连接

**窗口 1 和窗口 2 都执行**：

1. 打开浏览器控制台（F12）
2. 查看是否有以下日志：

```
✅ WebSocket 连接成功！
✅ 私有消息队列订阅成功
```

**预期结果**：
- ✅ 两个窗口都显示连接成功
- ❌ 如果连接失败，检查后端 WebSocket 配置

---

### 测试 2：消息发送

**在窗口 1（用户 A）**：

1. 选择与用户 B 的会话
2. 输入消息："测试消息 1"
3. 点击发送

**查看控制台**：
```
📤 发送消息到 WebSocket: {...}
✅ 消息已发送
```

**预期结果**：
- ✅ 消息显示在右边（绿色气泡）
- ✅ 显示发送时间
- ✅ 控制台有发送成功日志

---

### 测试 3：消息接收（最关键）

**在窗口 2（用户 B）**：

**查看控制台是否有**：
```
📬 收到私有消息: {...}
📨 收到 WebSocket 消息: {...}
📩 处理接收到的消息: {...}
🔓 消息解密成功: 测试消息 1
✅ 消息已添加到 store
✅ 消息已处理，会话列表已更新
```

**查看界面**：
- 会话列表应该显示未读数（红点或数字）
- 点击会话，应该看到消息

**预期结果**：
- ✅ 控制台有接收日志
- ✅ 消息显示在左边（白色气泡）
- ✅ 会话列表显示最新消息

**如果没有接收到**：
- ❌ 后端没有转发消息
- 需要实现消息转发功能

---

### 测试 4：消息显示位置

**在窗口 1（用户 A）**：

查看控制台：
```
📨 当前消息列表: {
  currentUserId: "user-a-id",
  messages: [
    { 
      senderId: "user-a-id", 
      isMine: true,        // 应该是 true
      content: "测试消息 1" 
    }
  ]
}
```

**在窗口 2（用户 B）**：

查看控制台：
```
📨 当前消息列表: {
  currentUserId: "user-b-id",
  messages: [
    { 
      senderId: "user-a-id", 
      isMine: false,       // 应该是 false
      content: "测试消息 1" 
    }
  ]
}
```

**预期结果**：
- ✅ 用户 A 看到消息在右边（`isMine: true`）
- ✅ 用户 B 看到消息在左边（`isMine: false`）
- ✅ `senderId` 和 `currentUserId` 正确匹配

**如果都在左边**：
- 检查 `senderId` 和 `currentUserId` 的值
- 可能格式不一致（UUID vs 用户名）

---

### 测试 5：已读功能

**在窗口 2（用户 B）**：

1. 点击与用户 A 的会话
2. 查看控制台：

```
📖 标记 1 条未读消息为已读
✅ 批量标记已读成功，共 1 条消息
```

3. 刷新页面（F5）
4. 再次查看会话列表

**预期结果**：
- ✅ 未读数变为 0
- ✅ 刷新后未读数仍然是 0（持久化成功）

**如果刷新后又显示未读**：
- ❌ 后端没有保存已读状态
- 检查数据库 `is_read` 字段

---

### 测试 6：双向消息

**在窗口 2（用户 B）**：

1. 回复消息："收到，测试消息 2"
2. 点击发送

**在窗口 1（用户 A）**：

查看是否收到消息

**预期结果**：
- ✅ 用户 A 收到消息
- ✅ 消息显示在左边（对方发送的）
- ✅ 用户 B 的消息显示在右边（自己发送的）

## 问题诊断

### 问题 1：收不到消息

**症状**：
- 发送方发送成功
- 接收方控制台没有任何日志

**原因**：
- 后端没有实现消息转发

**解决**：
- 参考 `后端需求-实时消息转发功能.md`
- 实现 WebSocket 消息处理器

---

### 问题 2：消息都在左边

**症状**：
- 自己发送的消息也显示在左边

**原因**：
- `senderId` 和 `currentUserId` 不匹配

**排查**：
1. 查看控制台的 `senderId` 和 `currentUserId`
2. 确认格式是否一致（都是 UUID 或都是用户名）
3. 如果不一致，需要统一格式

---

### 问题 3：消息内容不显示

**症状**：
- 只显示时间和头像，看不到文字

**原因**：
- 解密失败
- `content` 字段为空

**排查**：
1. 查看控制台是否有 `❌ 解密消息失败`
2. 检查会话密钥是否正确
3. 检查后端返回的 `messageContent` 字段

---

### 问题 4：刷新后又显示未读

**症状**：
- 标记已读后，刷新页面又显示未读

**原因**：
- 后端没有保存已读状态

**排查**：
1. 查看 Network 标签，确认标记已读的请求成功
2. 检查数据库 `b_session_log` 表的 `is_read` 字段
3. 如果字段没有更新，检查后端代码

---

## 快速检查清单

### 前端检查

- [ ] WebSocket 连接成功
- [ ] 发送消息成功（控制台有日志）
- [ ] 消息显示在正确位置（左/右）
- [ ] 消息内容正确显示
- [ ] 标记已读成功（控制台有日志）

### 后端检查

- [ ] WebSocket 端点可访问（`/ws/info`）
- [ ] 消息保存到数据库
- [ ] 消息转发给其他用户
- [ ] 已读状态保存到数据库
- [ ] 未读数统计正确

### 数据库检查

```sql
-- 1. 查看消息记录
SELECT id, send_user, message_content, is_read, send_time
FROM b_session_log
WHERE session_id = 'xxx'
ORDER BY send_time DESC
LIMIT 10;

-- 2. 统计未读消息
SELECT COUNT(*)
FROM b_session_log
WHERE session_id = 'xxx'
  AND is_read = false
  AND send_user != 'current-user-id';
```
