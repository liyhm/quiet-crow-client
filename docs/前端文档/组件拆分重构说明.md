# 组件拆分重构说明

## 重构目标

将 MainPage.vue (805行) 拆分成多个小型、可复用的组件，提高代码可维护性和复用性。

## 拆分后的组件结构

### 1. 通用组件 (widgets/)

#### TitleBar.vue (标题栏)
- **路径**: `src/renderer/src/widgets/titleBar/TitleBar.vue`
- **功能**: 窗口标题栏和控制按钮（最小化、最大化、关闭）
- **Props**: 
  - `title: string` - 标题文本
- **Events**: 
  - `minimize` - 最小化窗口
  - `maximize` - 最大化窗口
  - `close` - 关闭窗口
- **复用性**: ⭐⭐⭐⭐⭐ 可用于所有需要自定义标题栏的窗口

#### ChatHeader.vue (聊天头部)
- **路径**: `src/renderer/src/widgets/chatHeader/ChatHeader.vue`
- **功能**: 显示当前会话名称和菜单按钮
- **Props**: 
  - `sessionName: string` - 会话名称
- **Events**: 
  - `showMenu` - 显示菜单
- **复用性**: ⭐⭐⭐⭐ 可用于所有聊天界面

#### MessageBubble.vue (消息气泡)
- **路径**: `src/renderer/src/widgets/messageBubble/MessageBubble.vue`
- **功能**: 单条消息的显示（发送/接收样式）
- **Props**: 
  - `content: string` - 消息内容
  - `timestamp: string` - 时间戳
  - `isMine: boolean` - 是否是自己发送的
  - `senderName: string` - 发送者名称
- **Events**: 
  - `delete` - 删除消息
  - `contextmenu` - 右键菜单
- **复用性**: ⭐⭐⭐⭐⭐ 可用于所有消息显示场景

#### MessageList.vue (消息列表)
- **路径**: `src/renderer/src/widgets/messageList/MessageList.vue`
- **功能**: 消息列表容器，支持拖放文件
- **Props**: 
  - `messages: Message[]` - 消息数组
  - `currentUserId: string` - 当前用户ID
  - `senderNames: Record<string, string>` - 发送者名称映射
- **Events**: 
  - `deleteMessage` - 删除消息
  - `showContextMenu` - 显示右键菜单
  - `fileDrop` - 文件拖放
- **Methods**: 
  - `scrollToBottom()` - 滚动到底部
- **复用性**: ⭐⭐⭐⭐ 可用于所有消息列表场景

#### MessageInput.vue (消息输入)
- **路径**: `src/renderer/src/widgets/messageInput/MessageInput.vue`
- **功能**: 消息输入框和发送按钮
- **Props**: 
  - `modelValue: string` - 输入内容（v-model）
- **Events**: 
  - `update:modelValue` - 更新输入内容
  - `send` - 发送消息
  - `selectFile` - 选择文件
  - `selectImage` - 选择图片
  - `paste` - 粘贴事件
- **Methods**: 
  - `focus()` - 聚焦输入框
- **复用性**: ⭐⭐⭐⭐⭐ 可用于所有需要消息输入的场景

#### EmptyState.vue (空状态)
- **路径**: `src/renderer/src/widgets/emptyState/EmptyState.vue`
- **功能**: 显示空状态提示
- **Props**: 
  - `message?: string` - 提示文本（默认："连个说话的人都没有，真是悲哀。"）
- **复用性**: ⭐⭐⭐⭐⭐ 可用于所有空状态场景

#### ProfileModal.vue (个人信息弹窗)
- **路径**: `src/renderer/src/widgets/profileModal/ProfileModal.vue`
- **功能**: 个人信息编辑弹窗
- **Props**: 
  - `show: boolean` - 是否显示
  - `user: User | null` - 用户信息
  - `displayName: string` - 昵称（v-model）
  - `phone: string` - 手机号（v-model）
  - `uploading?: boolean` - 是否正在上传头像
  - `saving?: boolean` - 是否正在保存
- **Events**: 
  - `close` - 关闭弹窗
  - `save` - 保存信息
  - `logout` - 退出登录
  - `uploadAvatar` - 上传头像
  - `update:displayName` - 更新昵称
  - `update:phone` - 更新手机号
- **复用性**: ⭐⭐⭐ 可用于个人信息管理场景

#### SettingsModal.vue (设置弹窗)
- **路径**: `src/renderer/src/widgets/settingsModal/SettingsModal.vue`
- **功能**: 设置菜单弹窗
- **Props**: 
  - `show: boolean` - 是否显示
- **Events**: 
  - `close` - 关闭弹窗
  - `showProfile` - 显示个人信息
  - `logout` - 退出登录
- **复用性**: ⭐⭐⭐ 可用于设置菜单场景

#### ContextMenu.vue (右键菜单)
- **路径**: `src/renderer/src/widgets/contextMenu/ContextMenu.vue`
- **功能**: 消息右键菜单
- **Props**: 
  - `show: boolean` - 是否显示
  - `x: number` - X坐标
  - `y: number` - Y坐标
- **Events**: 
  - `close` - 关闭菜单
  - `delete` - 删除操作
- **复用性**: ⭐⭐⭐⭐ 可扩展用于其他右键菜单场景

### 2. 页面组件 (pages/)

#### MainPage.vue (主页面 - 重构后)
- **路径**: `src/renderer/src/pages/main/MainPage.vue`
- **行数**: 约 350 行（从 805 行减少 56%）
- **功能**: 组合所有子组件，处理业务逻辑
- **依赖组件**: 
  - TitleBar
  - Sidebar
  - SessionList
  - ContactsPage
  - ChatHeader
  - MessageList
  - MessageInput
  - EmptyState
  - ProfileModal
  - SettingsModal
  - ContextMenu

## 重构优势

### 1. 代码可维护性 ⬆️
- 每个组件职责单一，易于理解和修改
- 组件间依赖清晰，降低耦合度
- 代码行数减少，降低认知负担

### 2. 代码复用性 ⬆️
- MessageBubble 可用于所有消息显示场景
- EmptyState 可用于所有空状态场景
- TitleBar 可用于所有窗口
- MessageInput 可用于所有输入场景

### 3. 测试友好性 ⬆️
- 每个组件可独立测试
- Props 和 Events 定义清晰
- 易于编写单元测试

### 4. 团队协作 ⬆️
- 多人可同时开发不同组件
- 组件接口明确，减少沟通成本
- 易于代码审查


## 迁移指南

### 从旧版本迁移

如果你有使用旧版 MainPage.vue 的代码，需要：

1. 更新导入路径（如果有直接引用 MainPage 内部逻辑）
2. 使用新的组件替代原有的内联模板
3. 检查 Props 和 Events 是否匹配

### 备份文件

原始文件已备份为：`src/renderer/src/pages/main/MainPage.vue.backup`

如需回滚，执行：
```bash
mv src/renderer/src/pages/main/MainPage.vue.backup src/renderer/src/pages/main/MainPage.vue
```
