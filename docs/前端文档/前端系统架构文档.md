## 🛠 技术栈

### 核心框架
- **Electron**: v28+ - 桌面应用框架
- **Vue 3**: Composition API - 前端框架
- **TypeScript**: 类型安全
- **Vite**: 构建工具

### 状态管理
- **Pinia**: Vue 3 官方状态管理库

### UI 框架
- **Tailwind CSS v4**: 原子化 CSS 框架
- **自定义赛博朋克主题**: 深色毛玻璃风格

### 加密库
- **crypto-js**: AES 加密（与后端通信）
- **node-forge**: RSA 加密（端到端加密）

### 路由
- **Vue Router**: 单页应用路由

## 🏗 架构设计

### 分层架构（Feature-Sliced Design 启发）

```
┌─────────────────────────────────────┐
│         App Layer (应用层)           │
│  - 路由配置                          │
│  - 全局状态提供者                     │
│  - 全局样式                          │
└─────────────────────────────────────┘
              ↓
┌─────────────────────────────────────┐
│        Pages Layer (页面层)          │
│  - LoginPage (登录页)                │
│  - MainPage (主页)                   │
└─────────────────────────────────────┘
              ↓
┌─────────────────────────────────────┐
│      Features Layer (功能层)         │
│  - 认证功能                          │
│  - 消息发送功能                       │
└─────────────────────────────────────┘
              ↓
┌─────────────────────────────────────┐
│      Entities Layer (实体层)         │
│  - User (用户)                       │
│  - Contact (联系人)                  │
│  - Message (消息)                    │
│  - Session (会话)                    │
│  - Crypto (加密)                     │
│  - Network (网络)                    │
└─────────────────────────────────────┘
              ↓
┌─────────────────────────────────────┐
│       Shared Layer (共享层)          │
│  - API 服务                          │
│  - 工具函数                          │
│  - 类型定义                          │
│  - 配置文件                          │
└─────────────────────────────────────┘
```

## 🧩 核心模块

### 1. 认证模块 (Auth)

**位置**: `src/renderer/src/entities/user/model/useAuthStore.ts`

**功能**:
- 用户登录/注册
- Token 管理
- 用户信息存储
- 登录状态持久化

**关键方法**:
```typescript
- login(credentials): 登录
- register(params): 注册
- logout(): 登出
- checkAuth(): 检查认证状态
- fetchUserInfo(): 获取用户信息
```

### 2. 加密模块 (Crypto)

**位置**: `src/renderer/src/entities/crypto/model/useCryptoStore.ts`

**功能**:
- RSA 密钥对生成（端到端加密）
- 公钥上传到服务器
- 私钥安全存储（Electron safeStorage）
- 消息加密/解密

**加密方案**:
- **后端通信**: AES-ECB 加密（与后端约定的密钥）
- **端到端加密**: RSA-2048 加密（用户间消息）

### 3. 消息模块 (Message)

**位置**: `src/renderer/src/entities/message/model/useMessageStore.ts`

**功能**:
- 消息列表管理
- 消息发送/接收
- 消息加密/解密
- 消息已读状态管理

### 4. 会话模块 (Session)

**位置**: `src/renderer/src/entities/session/model/useSessionStore.ts`

**功能**:
- 会话列表管理
- 当前会话切换
- 未读消息计数
- 会话删除（软删除）

### 5. 联系人模块 (Contact)

**位置**: `src/renderer/src/entities/contact/model/useContactStore.ts`

**功能**:
- 联系人列表管理
- 好友请求处理
- 联系人搜索
- 公钥获取

### 6. 网络模块 (Network)

**位置**: `src/renderer/src/entities/network/model/useNetworkStore.ts`

**功能**:
- WebSocket 连接状态管理
- 网络状态监控
- 重连机制

## 📊 状态管理

### Pinia Store 架构

```typescript
// Store 结构示例
defineStore('storeName', () => {
  // 1. State (响应式状态)
  const state = ref<Type>(initialValue)
  
  // 2. Getters (计算属性)
  const computed = computed(() => derivedValue)
  
  // 3. Actions (方法)
  const action = async () => {
    // 业务逻辑
  }
  
  // 4. 返回公开接口
  return {
    state,
    computed,
    action
  }
})
```

### Store 依赖关系

```
useAuthStore (认证)
    ↓
useCryptoStore (加密)
    ↓
useSessionStore (会话) ← useContactStore (联系人)
    ↓
useMessageStore (消息)
    ↓
useNetworkStore (网络)
```

```typescript
    component: LoginPage,

  },
  {
    path: '/',

```

### 路由守卫

```typescript
router.beforeEach((to, from, next) => {
  const authStore = useAuthStore()
  
  if (to.meta.requiresAuth && !authStore.isAuthenticated) {
    next('/login')
  } else {
    next()
  }
})
```

## 🔐 加密系统

### 双层加密架构

```
┌─────────────────────────────────────┐
│      前端 → 后端 (HTTP 通信)         │
│      AES-ECB 加密                    │
│      密钥: 前后端约定                 │
└─────────────────────────────────────┘
              ↓
┌─────────────────────────────────────┐
│      用户 A → 用户 B (消息)          │
│      RSA-2048 加密                   │
│      公钥: 从服务器获取               │
│      私钥: 本地安全存储               │
└─────────────────────────────────────┘
```

### 加密流程

**1. 后端通信加密** (`cryptoBackend.ts`)
```typescript
// 加密
const encrypted = CryptoJS.AES.encrypt(data, SECRET_KEY).toString()

// 解密
const decrypted = CryptoJS.AES.decrypt(encrypted, SECRET_KEY).toString(CryptoJS.enc.Utf8)
```

**2. 端到端加密** (`useCryptoStore.ts`)
```typescript
// 生成密钥对
const keyPair = forge.pki.rsa.generateKeyPair(2048)

// 加密消息
const encrypted = forge.util.encode64(
  publicKey.encrypt(message, 'RSA-OAEP')
)

// 解密消息
const decrypted = privateKey.decrypt(
  forge.util.decode64(encrypted), 'RSA-OAEP'
)
```

### 私钥存储

使用 Electron 的 `safeStorage` API 安全存储私钥：

```typescript
// 主进程 (main/index.ts)
ipcMain.handle('secure-save', async (_, key, value) => {
  const encrypted = safeStorage.encryptString(value)
  fs.writeFileSync(keyPath, encrypted)
})

ipcMain.handle('secure-read', async (_, key) => {
  const encrypted = fs.readFileSync(keyPath)
  const decrypted = safeStorage.decryptString(encrypted)
  return decrypted
})
```

## 🔄 WebSocket 通信

**位置**: `src/renderer/src/shared/api/websocket.ts`

### 连接管理

```typescript
class WebSocketService {
  private ws: WebSocket | null = null
  private reconnectAttempts = 0
  private maxReconnectAttempts = 5
  
  connect() {
    this.ws = new WebSocket(WS_URL)
    this.setupEventHandlers()
  }
  
  reconnect() {
    if (this.reconnectAttempts < this.maxReconnectAttempts) {
      setTimeout(() => this.connect(), 3000)
    }
  }
}
```

### 消息类型

| 类型 | 说明 |
|------|------|
| `CHAT_MESSAGE` | 聊天消息 |
| `FRIEND_REQUEST` | 好友请求 |
| `MESSAGE_READ` | 消息已读通知 |
| `USER_STATUS` | 用户状态变化 |

### 消息处理流程

```
WebSocket 接收消息
    ↓
解密消息内容
    ↓
根据类型分发到对应 Store
    ↓
更新 UI
```

## 📝 代码规范

### TypeScript 规范

- 所有 API 响应必须定义类型
- 使用 `interface` 定义数据结构
- 避免使用 `any`

### Vue 规范

- 使用 Composition API
- 组件使用 `<script setup lang="ts">`
- Props 和 Emits 必须类型化

### 样式规范

- 优先使用 Tailwind 类名
- 避免内联样式
- 复杂样式使用 `<style scoped>`
