# 前端水位线模型集成完成

## 概述

已完成后端水位线模型的前端集成，实现了类似微信的会话删除和消息管理机制。

## 已实现功能

### 1. 删除会话（历史截断）✅

**功能**:
- 用户删除会话后，会话从列表中消失
- 本地消息缓存被清空
- 收到新消息时，会话自动重新出现
- 只能看到新消息，历史消息被截断

**实现位置**:
- `src/renderer/src/entities/session/model/useSessionStore.ts` - `deleteSession()`
- `src/renderer/src/entities/message/model/useMessageStore.ts` - `clearSessionMessages()`

**代码示例**:
```typescript
const deleteSession = async (sessionId: string) => {
  // 1. 调用后端 API 删除会话（使用水位线模型）
  await sessionApi.deleteSession(sessionId)
  
  // 2. 清空该会话的消息缓存
  messageStore.clearSessionMessages(sessionId)
  
  // 3. 清除选中状态
  if (activeSessionId.value === sessionId) {
    activeSessionId.value = null
  }
  
  // 4. 从本地列表中移除
  sessions.value = sessions.value.filter((s) => s.id !== sessionId)
}
```

**功能**:

- 仅对当前用户隐藏，其他用户不受影响
- 刷新页面后，消息仍然不可见（后端过滤）

**实现位置**:

- `src/renderer/src/shared/api/session.ts` - 添加 `deleteSingleMessage()` 方法
- `src/renderer/src/entities/message/model/useMessageStore.ts` - 添加 `deleteSingleMessage()` 方法
- `src/renderer/src/pages/main/MainPage.vue` - 添加删除按钮和右键菜单

**UI 实现**:
1. 悬停在自己发送的消息上，显示删除按钮（×）
2. 右键点击消息，显示删除菜单
3. 点击删除后，弹出确认对话框
4. 确认后，消息从列表中移除

**代码示例**:
```typescript
// API 配置
SESSION: {
  DELETE_MESSAGE: (messageId: string) => `/api/session/message/${messageId}`
}

// API 方法
deleteSingleMessage(messageId: string) {
  return request.delete(API_ENDPOINTS.SESSION.DELETE_MESSAGE(messageId))
}

// Store 方法
const deleteSingleMessage = async (sessionId: string, messageId: string) => {
  // 1. 调用后端 API
  await sessionApi.deleteSingleMessage(messageId)
  
  // 2. 从本地消息列表中移除
  const messages = messagesMap.value[sessionId]
  if (messages) {
    messagesMap.value[sessionId] = messages.filter((m) => m.id !== messageId)
  }
}
```

### 3. 会话自动唤醒 ✅

**功能**:
- 删除会话后收到新消息，会话自动重新出现
- 会话出现在列表顶部
- 本地消息缓存为空，只显示新消息

**实现位置**:
- `src/renderer/src/shared/api/websocket.ts` - `handleIncomingMessage()`

**代码示例**:
```typescript
private async handleIncomingMessage(message: any) {
  // 1. 检查会话是否存在
  const sessionExists = sessionStore.sessions.find(s => s.id === message.sessionId)
  
  if (!sessionExists) {
    // 2. 会话不存在，重新加载会话列表
    // 后端会自动创建 session_user 记录（私聊）
    await sessionStore.loadSessions()
  }
  
  // 3. 接收并解密消息
  const receivedMessage = await messageStore.receiveMessage(message)
  
  if (receivedMessage) {
    // 4. 更新会话预览
    const preview = receivedMessage.content?.substring(0, 20) || '新消息'
    sessionStore.updateLastMessage(message.sessionId, preview, receivedMessage.timestamp)
    
    // 5. 增加未读计数
    sessionStore.incrementUnread(message.sessionId)
  }
}
```

## 后端依赖

### 必需的后端功能

1. **删除会话（水位线模型）**
   - 端点: `DELETE /api/session/{sessionId}`
   - 行为: 设置 `is_visible = false`，记录水位线
   - 不是物理删除

2. **删除单条消息**
   - 端点: `DELETE /api/session/message/{messageId}`
   - 行为: 记录到 `b_user_deleted_message` 表
   - 仅对当前用户隐藏

3. **获取会话列表（过滤）**
   - 端点: `GET /api/session/my`
   - 行为: 只返回 `is_visible = true` 的会话

4. **获取历史消息（过滤）**
   - 端点: `GET /api/session/{sessionId}/messages`
   - 行为: 
     - 应用水位线过滤：`sequence_id > clear_watermark`
     - 过滤掉用户单独删除的消息

5. **发送消息时自动唤醒会话**
   - 行为: 发送消息时，自动将所有成员的 `is_visible` 设置为 `true`

## 注意事项

### 1. 消息 ID 的使用

删除单条消息时，使用消息的 `id` 字段（UUID），不是 `sequence_id`：

```typescript
// ✅ 正确
await deleteSingleMessage(message.id)

// ❌ 错误
await deleteSingleMessage(message.sequenceId)
```

### 2. 会话列表刷新

删除会话后收到新消息时，需要重新获取会话列表：

```typescript
if (!sessionExists) {
  // 会话可能被删除了，重新获取列表
  await fetchSessions()
}
```

### 3. 历史消息加载

删除会话后，历史消息会被截断。加载更多消息时可能返回空数组：

```typescript
const newMessages = await fetchMessages(sessionId, cursor, 50)
if (newMessages.length === 0) {
  // 没有更多消息了（可能因为水位线过滤）
  hasMore.value = false
}
```

### 4. 只能删除自己的消息

UI 上只在自己发送的消息上显示删除按钮：

```vue
<!-- 只在自己发送的消息上显示删除按钮 -->
<div v-if="message.senderId === currentUserId">
  <button @click="handleDeleteMessage(message.id)">删除</button>
</div>
```
