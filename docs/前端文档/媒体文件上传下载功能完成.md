## 🔧 技术实现

### 文件上传流程

```typescript
// 1. 选择文件
const handleSendFiles = async (files: File[]) => {
  for (const file of files) {
    await uploadFile(file)
  }
}

// 2. 检查文件大小
if (file.size > 50 * 1024 * 1024) {
  showAlert('文件大小不能超过 50MB', 'error')
  return
}

// 3. 加密文件
const encryptedData = await FileCryptoService.encryptFile(file, sessionKey)
const encryptedFileName = await FileCryptoService.encryptText(file.name, sessionKey)

// 4. 上传文件
const result = await mediaApi.uploadMedia({
  file: encryptedBlob,
  metadata: {
    sessionId,
    fileName: encryptedFileName,
    fileSize: file.size,
    mimeType: file.type || 'application/octet-stream',
    messageType: 'IMAGE' | 'VIDEO'
  }
})

// 5. 发送消息
const messageContent = JSON.stringify({
  fileId: result.fileId,
  fileName: file.name,
  fileSize: file.size
})
websocketService.sendMessage(sessionId, {
  content: encryptedContent,
  messageType: '2' // IMAGE
})
```

### 文件下载流程

```typescript
// 1. 下载加密文件
const result = await mediaApi.downloadMedia(fileId)

// 2. 解密文件
const decryptedData = await FileCryptoService.decryptFile(result.data, sessionKey)

// 3. 创建 Blob URL
const blob = new Blob([decryptedData], { type: result.mimeType })
const url = URL.createObjectURL(blob)

// 4. 显示或下载
mediaUrl.value = url
```

### 消息类型判断

```typescript
const getMessageType = (messageType: string, content?: string): MessageType => {
  switch (messageType) {
    case '1':
      return MessageType.TEXT
    case '2':
      return MessageType.IMAGE
    case '3':
      // 根据文件扩展名判断
      if (content) {
        const parsed = JSON.parse(content)
        const fileName = parsed.fileName?.toLowerCase() || ''
        if (fileName.endsWith('.mp4') || fileName.endsWith('.mov') || ...) {
          return MessageType.VIDEO
        }
      }
      return MessageType.FILE
  }
}
```

## 🐛 已修复的问题

### 问题 1: 视频显示为文件图标
- **原因**：messageType '3' 统一当作 FILE 处理
- **解决**：添加 VIDEO 类型，根据文件扩展名判断

### 问题 2: 图片/视频预览太小
- **原因**：使用 `max-w-[90vw] max-h-[90vh]` 限制了大小
- **解决**：改为 `width: 100%; height: 100%; object-fit: contain`

### 问题 3: 消息解密失败
- **原因**：后端返回的消息格式不正确
- **解决**：前端增加容错处理，显示友好错误信息

### 问题 4: 会话列表显示 fileId
- **原因**：没有检查消息是否是文件类型
- **解决**：解析 JSON，如果有 fileId 则显示 `[文件]`
