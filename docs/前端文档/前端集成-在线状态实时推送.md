# 前端集成：在线状态实时推送

## 📋 功能说明

实现在线状态的实时推送，当好友上线/离线时，自动更新好友列表和会话列表，无需手动刷新。

## ✅ 已完成的前端实现

### 1. WebSocket 订阅

已在 `websocket.ts` 中添加在线状态订阅：

```typescript
/**
 * 订阅在线状态变化通知
 * 
 * 监听好友的在线/离线状态变化，实时更新前端显示
 */
private subscribeToOnlineStatus() {
  if (!this.client) {
    console.error('❌ WebSocket client 未初始化')
    return
  }

  console.log('📡 订阅在线状态通知:', `/user/queue/online-status`)

  const subscription = this.client.subscribe(`/user/queue/online-status`, (message) => {
    console.log('📬 收到在线状态通知:', message.body)
    this.handleOnlineStatusChange(JSON.parse(message.body))
  })

  this.subscriptions.set('online-status', subscription)
  console.log('✅ 在线状态通知订阅成功')
}
```

### 2. 状态变化处理

```typescript
/**
 * 处理在线状态变化通知
 * 
 * @param notification 通知内容
 * @param notification.userId 用户ID
 * @param notification.username 用户名
 * @param notification.isOnline 是否在线
 * @param notification.timestamp 时间戳
 */
private async handleOnlineStatusChange(notification: any) {
  console.log('========================================')
  console.log('🔔 [handleOnlineStatusChange] 处理在线状态变化')
  console.log('🔔 [handleOnlineStatusChange] 用户ID:', notification.userId)
  console.log('🔔 [handleOnlineStatusChange] 用户名:', notification.username)
  console.log('🔔 [handleOnlineStatusChange] 在线状态:', notification.isOnline ? '在线' : '离线')
  console.log('🔔 [handleOnlineStatusChange] 时间戳:', notification.timestamp)

  const { useContactStore } = await import('@/entities/contact/model/useContactStore')
  const contactStore = useContactStore()

  // 更新联系人列表中的在线状态
  const contact = contactStore.contacts.find(c => c.userId === notification.userId)
  if (contact) {
    contact.isOnline = notification.isOnline
    console.log('✅ 已更新联系人在线状态:', contact.displayName, notification.isOnline ? '在线' : '离线')
  } else {
    console.log('⚠️ 未找到联系人:', notification.userId)
  }

  // 同时更新会话列表中的在线状态（如果有私聊会话）
  const { useSessionStore } = await import('@/entities/session/model/useSessionStore')
  const sessionStore = useSessionStore()
  
  const session = sessionStore.sessions.find(s => 
    s.sessionUserType === 'PRIVATE' && s.userId === notification.userId
  )
  if (session) {
    session.isOnline = notification.isOnline
    console.log('✅ 已更新会话在线状态:', session.showName, notification.isOnline ? '在线' : '离线')
  }

  console.log('========================================')
}
```

### 3. 自动订阅

在 WebSocket 连接成功后，自动订阅在线状态通知：

```typescript
private subscribeToPrivateMessages() {
  // ... 订阅私有消息 ...
  
  // 订阅好友请求通知
  this.subscribeToFriendRequests()
  
  // 订阅在线状态变化通知
  this.subscribeToOnlineStatus()

```

## 🔄 工作流程

### 好友上线流程

```
1. 用户 A 登录
   ↓
2. 后端推送通知给所有在线好友
   → 推送给用户 B: {"userId":"A","isOnline":true}
   ↓
3. 用户 B 的前端收到 WebSocket 消息
   ↓
4. handleOnlineStatusChange() 处理通知
   ↓
5. 更新 Contact Store 中的在线状态
   contact.isOnline = true
   ↓
6. 更新 Session Store 中的在线状态（如果有私聊）
   session.isOnline = true
   ↓
7. Vue 响应式系统自动更新 UI
   ↓
8. 用户 B 看到 A 在线（青色圆点 + "在线"文字）
```

### 好友离线流程

```
1. 用户 A 退出登录
   ↓
2. 后端推送通知给所有在线好友
   → 推送给用户 B: {"userId":"A","isOnline":false}
   ↓
3. 用户 B 的前端收到 WebSocket 消息
   ↓
4. handleOnlineStatusChange() 处理通知
   ↓
5. 更新 Contact Store 中的在线状态
   contact.isOnline = false
   ↓
6. 更新 Session Store 中的在线状态（如果有私聊）
   session.isOnline = false
   ↓
7. Vue 响应式系统自动更新 UI
   ↓
8. 用户 B 看到 A 离线（灰色圆点 + "离线"文字）
```

## ⚠️ 注意事项

### 1. Vue 响应式

确保 `isOnline` 字段是响应式的：

```typescript
// Contact 接口定义
export interface Contact {
  userId: string
  username: string
  displayName: string
  avatar: string
  status: string
  statusMessage: string
  isOnline?: boolean  // ✅ 可选字段，支持响应式更新
}
```

### 2. 直接修改对象属性

可以直接修改对象属性，Vue 3 会自动追踪：

```typescript
// ✅ 正确：直接修改
contact.isOnline = notification.isOnline

// ❌ 不需要：Vue 2 的写法
// Vue.set(contact, 'isOnline', notification.isOnline)
```

### 3. 未找到联系人

如果收到通知但找不到联系人，可能是：
- 好友关系还未加载
- 好友已被删除
- userId 不匹配

控制台会显示：
```
⚠️ 未找到联系人: xxx
```

### 4. WebSocket 未连接

如果 WebSocket 未连接，不会收到通知：
- 检查 WebSocket 连接状态
- 检查是否已登录
- 检查网络连接

## 📊 性能考虑

### 1. 响应式更新

- Vue 3 的响应式系统非常高效
- 直接修改对象属性，自动触发 UI 更新
- 无需手动调用 `forceUpdate()`

### 2. 批量更新

如果同时收到多个通知，Vue 会自动批量更新：

```typescript
// 收到 3 个通知
handleOnlineStatusChange({userId: 'A', isOnline: true})
handleOnlineStatusChange({userId: 'B', isOnline: true})
handleOnlineStatusChange({userId: 'C', isOnline: false})

// Vue 会批量更新 UI，只渲染一次
```

### 3. 内存占用

- 每个联系人只增加一个 `isOnline` 字段（1 字节）
- 100 个联系人 ≈ 100 字节
- 内存占用可忽略不计
