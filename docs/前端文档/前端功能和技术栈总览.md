# Quiet Crow 前端功能和技术栈总览

## 📱 应用概述

Quiet Crow 是一个基于 Electron + Vue 3 的端到端加密聊天应用，提供安全、实时的即时通讯功能。

**应用类型**: 桌面应用（跨平台）  
**支持平台**: Windows、macOS、Linux  
**开发框架**: Electron + Vue 3 + TypeScript

---

## 🎯 已实现的功能

### 1. 用户认证模块

#### 1.1 登录页面 (`LoginPage.vue`)
- ✅ 用户名/密码登录
- ✅ 用户注册
- ✅ 表单验证
- ✅ 错误提示
- ✅ 自动跳转

**技术实现**:
- 使用 `useAuthStore` 管理认证状态
- 调用 `authApi.login()` 和 `authApi.register()`
- Token 存储在 localStorage
- 登录成功后自动初始化 WebSocket 连接

#### 1.2 用户信息管理
- ✅ 查看个人信息
- ✅ 编辑昵称
- ✅ 编辑手机号
- ✅ 头像上传（支持 jpg/png/gif，最大 5MB）
- ✅ 退出登录

**技术实现**:
- 使用 `userApi.update()` 更新用户信息
- 使用 `userApi.uploadAvatar()` 上传头像
- 使用 FormData 处理文件上传

---

### 2. 消息聊天模块

#### 2.1 主页面 (`MainPage.vue`)
- ✅ 三栏布局（侧边栏 + 会话列表 + 聊天区域）
- ✅ 自定义标题栏（可拖拽）
- ✅ 窗口控制（最小化、最大化、关闭）
- ✅ 连接状态显示

**技术实现**:
- 使用 Flexbox 布局
- 使用 Electron IPC 控制窗口
- 实时显示 WebSocket 连接状态

#### 2.2 会话列表 (`SessionList.vue`)
- ✅ 显示所有会话（私聊/群聊）
- ✅ 会话搜索
- ✅ 最后消息预览
- ✅ 未读消息数量显示
- ✅ 会话时间显示
- ✅ 会话排序（按最后消息时间）
- ✅ 会话删除（软删除）
- ✅ 会话置顶（计划中）

**技术实现**:
- 使用 `useSessionStore` 管理会话状态
- 调用 `sessionApi.getList()` 获取会话列表
- 调用 `sessionApi.deleteSession()` 删除会话
- 实时更新会话列表（WebSocket 推送）

#### 2.3 消息聊天区域
- ✅ 消息发送（文本）
- ✅ 消息接收（实时）
- ✅ 消息加密/解密（AES-ECB）
- ✅ 消息历史加载
- ✅ 消息时间显示
- ✅ 消息已读状态
- ✅ 消息删除（单条）
- ✅ 自动滚动到底部
- ✅ 拖拽上传文件（UI 已实现，功能待完成）
- ✅ 表情选择器（UI 已实现，功能待完成）

**技术实现**:
- 使用 `useMessageStore` 管理消息状态
- 使用 `useCryptoStore` 处理加密/解密
- 调用 `sessionApi.getMessages()` 获取历史消息
- 调用 `messageApi.deleteSingle()` 删除消息
- WebSocket 实时接收新消息

#### 2.4 消息加密
- ✅ AES-ECB 加密算法
- ✅ 会话密钥管理
- ✅ 自动加密/解密
- ✅ 密钥交换（后端处理）

**技术实现**:
- 使用 `crypto-js` 库
- 使用 `useCryptoStore` 管理密钥
- 调用 `encryptionApi.getSessionKey()` 获取会话密钥

---

### 3. 联系人模块

#### 3.1 联系人页面 (`ContactsPage.vue`)
- ✅ 联系人列表显示
- ✅ 联系人搜索
- ✅ 添加好友
- ✅ 好友请求列表
- ✅ 接受/拒绝好友请求
- ✅ 删除好友
- ✅ 点击联系人创建会话

**技术实现**:
- 使用 `useContactStore` 管理联系人状态
- 调用 `contactsApi.getList()` 获取联系人列表
- 调用 `contactsApi.sendRequest()` 发送好友请求
- 调用 `contactsApi.acceptRequest()` 接受好友请求
- 调用 `contactsApi.rejectRequest()` 拒绝好友请求

#### 3.2 添加好友
- ✅ 搜索用户（按用户名/昵称）
- ✅ 发送好友请求
- ✅ 自定义验证消息
- ✅ 搜索结果显示

**技术实现**:
- 调用 `userApi.search()` 搜索用户
- 调用 `contactsApi.sendRequestByUsername()` 发送请求

#### 3.3 好友请求实时推送
- ✅ 收到新请求时实时提醒
- ✅ 侧边栏红色数字徽章
- ✅ 请求被接受后自动刷新
- ✅ 桌面通知提醒

**技术实现**:
- WebSocket 订阅 `/user/queue/friend-requests`
- 处理三种通知类型：NEW_REQUEST、REQUEST_ACCEPTED、REQUEST_REJECTED
- 使用 computed 属性自动更新红色徽章
- 使用 Notification API 显示桌面通知

---

### 4. 侧边栏模块

#### 4.1 侧边栏 (`Sidebar.vue`)
- ✅ 用户头像显示
- ✅ 消息图标（带未读数）
- ✅ 联系人图标（带好友请求数）
- ✅ 设置图标
- ✅ 标签切换（消息/联系人）

**技术实现**:
- 使用 `useAuthStore` 获取用户信息
- 使用 `useSessionStore` 计算未读消息数
- 使用 `useContactStore` 计算好友请求数
- 使用 computed 属性自动更新徽章

---

### 5. WebSocket 实时通讯

#### 5.1 WebSocket 服务 (`websocket.ts`)
- ✅ 自动连接/断开
- ✅ 心跳检测
- ✅ 断线重连（手动）
- ✅ 消息订阅
- ✅ 消息发送
- ✅ 好友请求推送
- ✅ 连接状态管理

**技术实现**:
- 使用 SockJS + STOMP 协议
- 订阅 `/user/queue/messages` 接收私聊消息
- 订阅 `/user/queue/friend-requests` 接收好友请求通知
- 订阅 `/topic/sessions/{sessionId}` 接收群聊消息
- 使用 `useNetworkStore` 管理连接状态

#### 5.2 消息处理
- ✅ 接收消息自动解密
- ✅ 发送消息自动加密
- ✅ 消息去重
- ✅ 未读数计算
- ✅ 会话列表更新
- ✅ 会话自动复活（删除后收到新消息）

**技术实现**:
- 在 `handleIncomingMessage()` 中处理接收消息
- 在 `sendMessage()` 中处理发送消息
- 使用消息 ID 去重
- 自动更新会话的最后消息和未读数

## 🛠️ 技术栈详解

### 核心框架

#### 1. Electron (v39.2.6)
- **用途**: 桌面应用框架
- **功能**:
  - 跨平台支持（Windows、macOS、Linux）
  - 原生窗口控制
  - 系统托盘
  - 自动更新（electron-updater）

#### 2. Vue 3 (v3.5.25)
- **用途**: 前端 UI 框架
- **特性**:
  - Composition API
  - `<script setup>` 语法
  - 响应式系统
  - 组件化开发

#### 3. TypeScript (v5.9.3)
- **用途**: 类型安全
- **优势**:
  - 静态类型检查
  - 更好的 IDE 支持
  - 代码提示和自动补全

---

### 状态管理

#### Pinia (v3.0.4)
- **用途**: 状态管理库
- **Store 列表**:
  1. `useAuthStore` - 用户认证状态
  2. `useSessionStore` - 会话列表状态
  3. `useMessageStore` - 消息状态
  4. `useCryptoStore` - 加密密钥状态
  5. `useContactStore` - 联系人状态
  6. `useNetworkStore` - 网络连接状态

**特点**:
- 类型安全
- 模块化设计
- 支持 DevTools
- 轻量级

---

### 路由管理

#### Vue Router (v5.0.3)
- **用途**: 页面路由
- **路由列表**:
  - `/login` - 登录页面
  - `/main` - 主页面（消息/联系人）

---

### HTTP 请求

#### Axios (v1.13.5)
- **用途**: HTTP 客户端
- **功能**:
  - 请求/响应拦截器
  - 自动添加 Token
  - 错误处理
  - 请求取消

**配置**:
- Base URL: 从环境变量读取
- Timeout: 10 秒
- 自动 Token 刷新

---

### WebSocket 通讯

#### SockJS Client (v1.6.1) + STOMP (v7.3.0)

**订阅队列**:
- `/user/queue/messages` - 私聊消息
- `/user/queue/friend-requests` - 好友请求通知

## 🎨 设计模式

### 1. Feature-Sliced Design (FSD)
- **entities**: 业务实体和状态管理
- **features**: 功能模块
- **widgets**: 复合组件
- **pages**: 页面组件
- **shared**: 共享资源

### 2. Composition API
- 使用 `<script setup>` 语法
- 逻辑复用通过 composables
- 响应式数据管理

### 3. Store 模式
- 使用 Pinia 管理全局状态
- 每个实体一个 Store
- 模块化设计

---

## 🔐 安全特性

### 1. 端到端加密
- **算法**: AES-ECB
- **密钥管理**: 每个会话独立密钥
- **加密范围**: 所有消息内容

### 2. Token 认证
- **存储**: localStorage
- **传输**: HTTP Header
- **刷新**: 自动刷新机制

### 3. 安全存储
- **IndexedDB**: 加密密钥存储
- **localStorage**: Token 存储
- **内存**: 敏感数据不持久化
