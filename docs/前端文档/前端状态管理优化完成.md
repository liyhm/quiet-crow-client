const deleteSession = async (sessionId: string) => {
  // 1. 调用后端 API 删除会话（真删除）

### 3. 接收新消息逻辑优化 ✅

**文件**: `src/renderer/src/shared/api/websocket.ts`

**优化内容**:
```typescript
/**
 * 处理接收到的 WebSocket 消息
 * 
 * 流程：
 * 1. 检查本地会话列表中是否存在该会话
 * 2. 如果不存在（可能被删除或是新会话），重新加载会话列表
 *    - 私聊：后端会自动创建 session_user 记录，会话会重新出现
 *    - 群聊：不会自动重新加入
 * 3. 解密消息并添加到消息列表
 * 4. 更新会话的最后消息预览（使用解密后的内容）
 * 5. 增加未读计数（如果不是当前活动会话）
 * 
 * 会话复活逻辑：
 * - 用户删除会话后，本地消息缓存已清空
 * - 对方发送新消息时，会话重新出现在列表顶部
 * - 由于本地缓存为空，只会显示新消息，表现为"干净的新会话"
 */
private async handleIncomingMessage(message: any) {
  // 1. 检查会话是否存在
  const sessionExists = sessionStore.sessions.find(s => s.id === message.sessionId)
  
  if (!sessionExists) {
    // 2. 会话不存在，重新加载会话列表
    await sessionStore.loadSessions()
  }
  
  // 3. 接收并解密消息
  const receivedMessage = await messageStore.receiveMessage(message)
  
  if (receivedMessage) {
    // 4. 更新会话预览（使用解密后的内容）
    const preview = receivedMessage.content?.substring(0, 20) || '新消息'
    sessionStore.updateLastMessage(message.sessionId, preview, receivedMessage.timestamp)
    
    // 5. 增加未读计数
    sessionStore.incrementUnread(message.sessionId)
  }
}
```

**关键改进**:
- 添加了完整的业务逻辑注释
- 说明了会话复活的原理
- 区分了私聊和群聊的处理方式

## 完整业务流程

### 场景 1: 删除会话

```
用户点击删除
    ↓
deleteSession(sessionId)
    ↓
后端 API: DELETE /session/{sessionId}  ← 真删除
    ↓
clearSessionMessages(sessionId)  ← 清空消息缓存
    ↓
activeSessionId = null  ← 清除选中状态
    ↓
sessions.filter(...)  ← 从列表移除
    ↓
UI 自动更新
```

### 场景 2: 删除后收到新消息（会话复活）

```
用户 A 删除与用户 B 的会话
    ↓
本地状态：
  - sessions 中没有该会话
  - messagesMap[sessionId] 为空
    ↓
用户 B 发送新消息
    ↓
handleIncomingMessage(message)
    ↓
检查会话是否存在 → 不存在
    ↓
loadSessions()  ← 重新加载会话列表
    ↓
后端自动创建 session_user（私聊）
    ↓
会话重新出现在列表顶部
    ↓
receiveMessage(message)  ← 解密并添加消息
    ↓
updateLastMessage(...)  ← 更新预览
    ↓
UI 显示：
  - 会话列表顶部出现该会话
  - 点击进入只看到新消息（旧消息已清空）
  - 表现为"干净的新会话"
```

### 场景 3: 加载消息

```
用户点击会话
    ↓
setActiveSession(sessionId)
    ↓
loadMessages(sessionId)
    ↓
ensureSessionKey(sessionId)  ← 确保有密钥
    ↓
sessionApi.getMessages(sessionId)  ← 拉取历史消息
    ↓
解密所有消息
    ↓
messagesMap[sessionId] = decryptedMessages
    ↓
markBatchAsRead(...)  ← 标记已读
    ↓
UI 显示消息列表
```

## 数据结构

### SessionStore

```typescript
{
  sessions: ChatSession[]           // 会话列表
  activeSessionId: string | null    // 当前活动会话
}
```

### MessageStore

```typescript
{
  messagesMap: Record<string, Message[]>  // 按 sessionId 隔离的消息列表
  pendingQueue: Message[]                 // 待发送队列
}
```

## 关键方法

### SessionStore

- `loadSessions()` - 加载会话列表
- `deleteSession(sessionId)` - 删除会话（含消息缓存清理）
- `setActiveSession(sessionId)` - 设置活动会话
- `updateLastMessage(sessionId, message, timestamp)` - 更新最后消息
- `incrementUnread(sessionId)` - 增加未读计数

### MessageStore

- `loadMessages(sessionId)` - 加载消息列表
- `sendMessage(sessionId, content)` - 发送消息
- `receiveMessage(message)` - 接收并解密消息
- `clearSessionMessages(sessionId)` - 清空会话消息缓存 ← 新增
- `markBatchAsRead(sessionId, messageIds)` - 批量标记已读

### WebSocketService

- `handleIncomingMessage(message)` - 处理接收到的消息（含会话复活逻辑）
