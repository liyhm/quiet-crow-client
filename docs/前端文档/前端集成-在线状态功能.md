# 在线状态 - 前端集成指南

## 📋 概述

后端已实现用户在线状态功能，会话列表和群成员列表都会返回实时的在线状态。

---

## 🔌 API 变更

### 1. 会话列表接口

**接口**：`GET /api/session/my`

**新增字段**：`isOnline`

```typescript
interface Session {
  id: string
  sessionId: string
  userId?: string  // 私聊时对方的用户ID
  showName: string
  avatar?: string
  sessionUserType: 'PRIVATE' | 'GROUP'
  lastMessage?: string
  lastMessageTime?: string
  unreadCount: number
  isOnline?: boolean  // ← 新增：在线状态（私聊有效，群聊为null）
}
```

**响应示例**：

```json
{
  "code": 200,
  "data": [
    {
      "id": "session_user_123",
      "sessionId": "session_456",
      "userId": "user_789",
      "showName": "张三",
      "sessionUserType": "PRIVATE",
      "isOnline": true  // ← 对方在线
    },
    {
      "id": "session_user_124",
      "sessionId": "session_457",
      "showName": "技术交流群",
      "sessionUserType": "GROUP",
      "isOnline": null  // ← 群聊不显示在线状态
    }
  ]
}
```

### 2. 群成员列表接口

**接口**：`GET /api/session/{sessionId}/members`

**更新字段**：`isOnline`（之前固定返回 false，现在返回真实状态）

```typescript
interface GroupMember {
  userId: string
  username: string
  showName: string
  avatar?: string
  userType: 'OWNER' | 'MEMBER'
  isOnline: boolean  // ← 更新：返回真实在线状态
}
```

**响应示例**：

```json
{
  "code": 200,
  "data": [
    {
      "userId": "user_001",
      "showName": "张三",
      "userType": "OWNER",
      "isOnline": true  // ← 真实在线状态
    },
    {
      "userId": "user_002",
      "showName": "李四",
      "userType": "MEMBER",
      "isOnline": false
    }
  ]
}
```

## 🎨 前端实现

### 1. 会话列表显示在线状态

```vue
<template>
  <div class="session-list">
    <div 
      v-for="session in sessions" 
      :key="session.id" 
      class="session-item"
    >
      <!-- 头像 -->
      <div class="avatar-container">
        <img :src="session.avatar" class="avatar" />
        
        <!-- 私聊显示在线状态指示器 -->
        <span 
          v-if="session.sessionUserType === 'PRIVATE' && session.isOnline !== null"
          :class="['online-indicator', session.isOnline ? 'online' : 'offline']"
        />
      </div>

      <!-- 会话信息 -->
      <div class="session-info">
        <div class="name">{{ session.showName }}</div>
        <div class="last-message">{{ session.lastMessage }}</div>
      </div>

      <!-- 未读数 -->
      <span v-if="session.unreadCount > 0" class="unread-badge">
        {{ session.unreadCount }}
      </span>
    </div>
  </div>
</template>

<style scoped>
.avatar-container {
  position: relative;
}

.online-indicator {
  position: absolute;
  bottom: 0;
  right: 0;
  width: 12px;
  height: 12px;
  border-radius: 50%;
  border: 2px solid white;
}

.online-indicator.online {
  background-color: #52c41a;  /* 绿色 */
}

.online-indicator.offline {
  background-color: #d9d9d9;  /* 灰色 */
}
</style>
```

## 🔄 实时更新（可选）

如果需要实时更新在线状态，可以定期刷新会话列表：

```typescript
// 每 30 秒刷新一次会话列表
const refreshInterval = setInterval(async () => {
  await sessionStore.loadSessions()
}, 30000)

// 组件卸载时清除定时器
onUnmounted(() => {
  clearInterval(refreshInterval)
})
```

或者监听 WebSocket 在线状态变更通知（需要后端额外实现）：

```typescript
// 订阅在线状态变更通知
stompClient.subscribe('/user/queue/online-status', (message) => {
  const notification = JSON.parse(message.body)
  
  if (notification.type === 'USER_ONLINE_STATUS_CHANGED') {
    // 更新本地会话列表中的在线状态
    sessionStore.updateUserOnlineStatus(
      notification.userId,
      notification.isOnline
    )
  }
})
```

## ⚠️ 注意事项

### 1. 在线状态的含义

- `isOnline: true` - 用户已连接 WebSocket，处于在线状态
- `isOnline: false` - 用户未连接 WebSocket，处于离线状态
- `isOnline: null` - 群聊会话，不显示在线状态

### 2. 在线状态延迟

- 用户关闭页面后，WebSocket 会立即断开，在线状态会立即更新为离线
- 但如果用户网络波动，可能出现短暂的离线状态
- 建议显示"最近在线"而不是"离线"，提供更好的用户体验

### 3. 性能考虑

- 在线状态查询使用 Redis，性能很高
- 会话列表查询时会批量检查在线状态
- 不会影响接口响应速度

### 4. 兼容性

- 如果后端 Redis 不可用，`isOnline` 会返回 `false`
- 前端应该优雅处理，不影响其他功能

## 🧪 测试场景

### 测试 1: 私聊在线状态

1. 用户 A 登录，建立 WebSocket 连接
2. 用户 B 登录，查看会话列表
3. 用户 B 应该看到用户 A 的在线状态为 `true`
4. 用户 A 关闭页面
5. 用户 B 刷新会话列表
6. 用户 B 应该看到用户 A 的在线状态为 `false`

### 测试 2: 群成员在线状态

1. 用户 A、B、C 都在同一个群
2. 用户 A 登录，建立 WebSocket 连接
3. 用户 B 登录，查看群成员列表
4. 用户 B 应该看到：
   - 用户 A: `isOnline: true`
   - 用户 C: `isOnline: false`（未登录）

### 测试 3: 多次登录

1. 用户 A 在设备 1 登录
2. 用户 A 在设备 2 登录（单点登录，设备 1 被踢下线）
3. 用户 B 查看会话列表
4. 用户 B 应该看到用户 A 的在线状态为 `true`（设备 2）
