# 群名称修改 - 前端集成指南

## 📋 概述

本文档提供群名称修改功能的前端集成指南，包括API调用、WebSocket通知处理和完整的代码示例。

**功能特性**：
- 所有群成员都可以修改群名称（不限于群主）
- 实时WebSocket通知其他在线成员
- 群名称长度限制：1-50字符
- 自动去除首尾空格

## 📤 修改群名称

### 1. API 调用

**接口地址**：`PUT /api/session/{sessionId}/name`

**请求示例**：

```typescript
// src/renderer/src/shared/api/session.ts
import request from './request'

export const sessionApi = {
  /**
   * 修改群名称
   * @param sessionId 会话ID
   * @param name 新的群名称
   */
  updateGroupName(sessionId: string, name: string) {
    return request.put<any, {
      sessionId: string
      name: string
      updatedBy: string
      updatedAt: string
    }>(`/api/session/${sessionId}/name`, { name })
  }
}
```

### 2. 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| name | string | 是 | 新的群名称，长度1-50字符 |

### 3. 成功响应

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "sessionId": "session_123456",
    "name": "新的群名称",
    "updatedBy": "user_789",
    "updatedAt": "2024-03-04T10:30:00"
  }
}
```

### 4. 错误响应

| 状态码 | 错误信息 | 说明 |
|--------|----------|------|
| 400 | 群名称不能为空 | 群名称为空或只有空格 |
| 400 | 群名称长度不能超过50个字符 | 群名称过长 |
| 400 | 该会话不是群聊 | 尝试修改私聊会话名称 |
| 403 | 您不是该群的成员 | 用户不是群成员 |
| 404 | 会话不存在 | 会话ID无效 |
| 401 | 未授权 | 未登录或token过期 |

## 🎨 前端实现

### 1. GroupMembersModal 组件实现

```vue
<!-- src/renderer/src/widgets/groupMembersModal/GroupMembersModal.vue -->
<template>
  <div class="group-info">
    <div class="flex items-center gap-2 group">
      <!-- 编辑模式 -->
      <input
        v-if="isEditingName"
        v-model="editingGroupName"
        @blur="handleSaveGroupName"
        @keyup.enter="handleSaveGroupName"
        @keyup.esc="cancelEditName"
        class="bg-[#151518] border border-[#00E5FF]/40 text-[#E0E0E0] text-[14px] font-bold tracking-widest px-2 py-1 rounded focus:outline-none focus:border-[#00E5FF]"
        style="width: 180px;"
        ref="nameInput"
      />
      <!-- 显示模式 -->
      <span
        v-else
        @click="startEditName"
        class="text-[#E0E0E0] text-[14px] font-bold tracking-widest drop-shadow-md cursor-pointer hover:text-[#00E5FF] transition-colors"
        title="点击修改群名称"
      >
        {{ groupName }}
      </span>
      <!-- 编辑图标 -->
      <button
        v-if="!isEditingName"
        @click="startEditName"
        class="opacity-0 group-hover:opacity-100 text-[#00E5FF]/60 hover:text-[#00E5FF] transition-all"
        title="修改群名称"
      >
        <svg class="w-3 h-3" fill="none" stroke="currentColor" viewBox="0 0 24 24">
          <path
            stroke-linecap="round"
            stroke-linejoin="round"
            stroke-width="2"
            d="M15.232 5.232l3.536 3.536m-2.036-5.036a2.5 2.5 0 113.536 3.536L6.5 21.036H3v-3.572L16.732 3.732z"
          />
        </svg>
      </button>
    </div>
  </div>
</template>

<script setup lang="ts">
import { ref, nextTick } from 'vue'

const props = defineProps<{
  groupName: string
}>()

const emit = defineEmits<{
  (e: 'updateGroupName', newName: string): void
}>()

// 群名称编辑
const isEditingName = ref(false)
const editingGroupName = ref('')
const nameInput = ref<HTMLInputElement>()

const startEditName = () => {
  editingGroupName.value = props.groupName
  isEditingName.value = true
  nextTick(() => {
    nameInput.value?.focus()
    nameInput.value?.select()
  })
}

const cancelEditName = () => {
  isEditingName.value = false
  editingGroupName.value = ''
}

const handleSaveGroupName = () => {
  const newName = editingGroupName.value.trim()
  if (newName && newName !== props.groupName) {
    emit('updateGroupName', newName)
  }
  isEditingName.value = false
  editingGroupName.value = ''
}
</script>
```

const handleUpdateGroupName = async (newName: string): Promise<void> => {
  if (!activeSession.value?.id) return

  try {
    console.log('🔨 更新群名称:', newName)

  }
}

// 最简单的实现方式
const updateGroupName = async (sessionId: string, newName: string) => {

    return result.data
  } catch (error) {
    console.error('更新失败:', error)
    throw error
  }
}

## 📡 WebSocket 通知

### 1. 通知格式

当群名称被修改后，后端会向所有在线群成员推送通知：

```json
{
  "type": "GROUP_NAME_UPDATED",
  "sessionId": "session_123456",
  "name": "新的群名称",
  "updatedBy": "user_789",
  "updatedByName": "张三",
  "updatedAt": "2024-03-04T10:30:00"
}
```

### 2. 监听通知

```typescript
// src/renderer/src/shared/api/websocket.ts
import { useSessionStore } from '@/entities/session/model/useSessionStore'

export class WebSocketService {
  private client: any

  connect() {
    // ... WebSocket连接代码 ...

    // 订阅通知队列
    this.client.subscribe('/user/queue/notifications', (message: any) => {
      const notification = JSON.parse(message.body)
      this.handleNotification(notification)
    })
  }

  private handleNotification(notification: any) {
    console.log('📥 收到通知:', notification)

    if (notification.type === 'GROUP_NAME_UPDATED') {
      this.handleGroupNameUpdated(notification)
    }
  }

  private async handleGroupNameUpdated(notification: any) {
    const sessionStore = useSessionStore()

    console.log('🔔 群名称已更新:', {
      sessionId: notification.sessionId,
      newName: notification.name,
      updatedBy: notification.updatedByName
    })

    // 更新本地会话列表
    await sessionStore.loadSessions()

    // 如果是当前活动会话，显示提示
    if (sessionStore.activeSession?.id === notification.sessionId) {
      showNotification(
        `${notification.updatedByName} 将群名称改为 "${notification.name}"`,
        'info'
      )
    }
  }
}
```

### 3. 实时更新UI

```typescript
// src/renderer/src/entities/session/model/useSessionStore.ts
export const useSessionStore = defineStore('session', () => {
  const sessions = ref<Session[]>([])

  /**
   * 更新会话名称（用于WebSocket通知）
   */
  const updateSessionName = (sessionId: string, newName: string) => {
    const session = sessions.value.find(s => s.id === sessionId)
    if (session) {
      session.showName = newName
      console.log('✅ 会话名称已更新:', { sessionId, newName })
    }
  }

  return {
    sessions,
    updateSessionName
  }
})
```

## ⚠️ 错误处理

### 1. 前端验证

```typescript
/**
 * 验证群名称
 */
const validateGroupName = (name: string): { valid: boolean; error?: string } => {
  const trimmedName = name.trim()

  if (!trimmedName) {
    return { valid: false, error: '群名称不能为空' }
  }

  if (trimmedName.length > 50) {
    return { valid: false, error: '群名称长度不能超过50个字符' }
  }

  return { valid: true }
}

// 使用示例
const handleUpdate = async (newName: string) => {
  const validation = validateGroupName(newName)
  if (!validation.valid) {
    showAlert(validation.error!, 'error')
    return
  }

  // 继续更新...
}
```

### 2. 错误提示

```typescript
/**
 * 统一错误处理
 */
const handleUpdateError = (error: any) => {
  let message = '更新失败'

  if (error.response) {
    const status = error.response.status
    const data = error.response.data

    switch (status) {
      case 400:
        message = data.message || '请求参数错误'
        break
      case 401:
        message = '未登录，请先登录'
        // 跳转到登录页
        router.push('/login')
        break
      case 403:
        message = '您不是该群的成员'
        break
      case 404:
        message = '会话不存在'
        break
      default:
        message = data.message || '服务器错误'
    }
  } else if (error.message) {
    message = error.message
  }

  showAlert(message, 'error')
}
```

### 2. 乐观更新

```typescript
/**
 * 乐观更新：先更新UI，失败后回滚
 */
const optimisticUpdate = async (sessionId: string, newName: string) => {
  const sessionStore = useSessionStore()
  const oldName = sessionStore.activeSession?.showName

  try {
    // 1. 立即更新UI
    sessionStore.updateSessionName(sessionId, newName)

    // 2. 调用API
    await sessionApi.updateGroupName(sessionId, newName)
  } catch (error) {
    // 3. 失败后回滚
    if (oldName) {
      sessionStore.updateSessionName(sessionId, oldName)
    }
    throw error
  }
}
```

## 🆘 常见问题

### Q1: 修改后其他成员看不到新名称？

**原因**：WebSocket未连接或通知未处理

**解决**：
1. 检查WebSocket连接状态
2. 确认订阅了`/user/queue/notifications`队列
3. 检查通知处理逻辑

### Q2: 提示"您不是该群的成员"？

**原因**：用户不在该群中或会话ID错误

**解决**：
1. 确认sessionId正确
2. 检查用户是否真的是群成员
3. 刷新会话列表

### Q3: 群名称包含emoji显示异常？

**原因**：字符编码问题

**解决**：
- 后端已支持UTF-8编码
- 前端确保使用UTF-8编码
- emoji会占用多个字符长度，注意长度限制

### Q4: 修改后刷新页面名称又变回去了？

**原因**：本地缓存未更新

**解决**：
```typescript
// 修改成功后刷新会话列表
await sessionStore.loadSessions()
```
