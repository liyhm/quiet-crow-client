# 前端状态管理优化方案

## 概述

本文档描述了类似"微信"的 IM 聊天界面的前端状态管理和业务逻辑优化方案。

---

## 核心数据结构设计

### 1. 会话状态 (SessionStore)

```typescript
{
  // 会话列表（按最后消息时间排序）
  sessions: ChatSession[]
  
  // 当前激活的会话 ID
  activeSessionId: string | null
}
```

### 2. 消息状态 (MessageStore)

```typescript
{
  // 消息映射表：按 sessionId 隔离的消息列表
  messagesMap: Record<string, Message[]>
  
  // 待发送队列（离线时使用）
  pendingQueue: Message[]
}
```

## 核心业务逻辑

### 1. 删除会话 (deleteSession)

**流程**:
1. 调用后端 API 删除会话
2. 从本地 Store 中移除该会话
3. 清空该会话的本地消息缓存
4. 如果是当前活动会话，清除选中状态

**优化点**:
- 添加消息缓存清理逻辑
- 确保状态一致性

```typescript
const deleteSession = async (sessionId: string) => {
  try {
    // 1. 调用后端 API 删除会话
    await sessionApi.deleteSession(sessionId)
    
    // 2. 清空该会话的消息缓存
    const messageStore = useMessageStore()
    messageStore.clearSessionMessages(sessionId)
    
    // 3. 如果是当前活动会话，清除选中状态
    if (activeSessionId.value === sessionId) {
      activeSessionId.value = null
    }
    
    // 4. 从本地列表中移除
    sessions.value = sessions.value.filter((s) => s.id !== sessionId)
    
    return { success: true }
  } catch (error) {
    return { success: false, error: String(error) }
  }
}
```

### 2. 接收新消息 (receiveNewMessage)

**流程**:
1. 检查本地会话列表中是否存在该会话
2. 如果不存在，重新加载会话列表（后端会返回新会话）
3. 解密消息并添加到消息列表
4. 更新会话的最后消息预览和时间
5. 增加未读计数（如果不是当前活动会话）

**优化点**:
- 会话复活逻辑：删除后收到新消息，会话自动重新出现
- 使用解密后的内容作为预览
- 确保会话列表实时更新

```typescript
const handleIncomingMessage = async (message: any) => {
  const messageStore = useMessageStore()
  const sessionStore = useSessionStore()
  
  // 1. 检查会话是否存在
  const sessionExists = sessionStore.sessions.find(s => s.id === message.sessionId)
  
  if (!sessionExists) {
    // 2. 会话不存在，重新加载会话列表
    // 后端会自动创建 session_user 记录（私聊）
    await sessionStore.loadSessions()
  }
  
  // 3. 接收并解密消息
  const receivedMessage = await messageStore.receiveMessage(message)
  
  if (receivedMessage) {
    // 4. 更新会话的最后消息预览（使用解密后的内容）
    const preview = receivedMessage.content?.substring(0, 20) || '新消息'
    sessionStore.updateLastMessage(
      message.sessionId,
      preview,
      receivedMessage.timestamp
    )
    
    // 5. 增加未读计数
    sessionStore.incrementUnread(message.sessionId)
  }
}
```

### 3. 加载消息 (loadMessages)

**流程**:
1. 进入聊天面板时调用
2. 确保会话密钥存在
3. 从后端拉取消息列表
4. 解密所有消息
5. 按时间排序并存储到 messagesMap
6. 标记未读消息为已读

**优化点**:
- 自动获取/生成会话密钥
- 解密失败时显示友好提示
- 自动标记已读

```typescript
const loadMessages = async (sessionId: string) => {
  try {
    // 1. 确保有会话密钥
    await ensureSessionKey(sessionId)
    
    // 2. 从后端拉取消息列表
    const messages = await sessionApi.getMessages(sessionId)
    
    // 3. 解密所有消息
    const decryptedMessages: Message[] = []
    for (const msg of messages) {
      try {
        const content = backendCrypto.decryptWithSession(sessionId, msg.messageContent)
        decryptedMessages.push({
          id: msg.id,
          sessionId: msg.sessionId,
          senderId: msg.sendUser,
          content,
          timestamp: msg.sendTime,
          status: MessageStatus.DELIVERED,
          isRead: msg.isRead,
          type: msg.messageType === '1' ? MessageType.TEXT : MessageType.FILE
        })
      } catch (error) {
        // 解密失败时显示友好提示
        decryptedMessages.push({
          ...msg,
          content: '[解密失败]'
        })
      }
    }
    
    // 4. 按时间排序并存储
    messagesMap.value[sessionId] = decryptedMessages.sort((a, b) => 
      new Date(a.timestamp).getTime() - new Date(b.timestamp).getTime()
    )
    
    // 5. 标记未读消息为已读
    const authStore = useAuthStore()
    const unreadMessages = decryptedMessages.filter(
      m => !m.isRead && m.senderId !== authStore.userId
    )
    if (unreadMessages.length > 0) {
      await markBatchAsRead(sessionId, unreadMessages.map(m => m.id))
    }
    
    return { success: true }
  } catch (error) {
    return { success: false, error: String(error) }
  }
}
```

## 关键优化点

### 1. 消息缓存管理

**问题**: 删除会话后，本地消息缓存没有清理

**解决**: 在 `deleteSession` 中添加 `clearSessionMessages(sessionId)` 方法

```typescript
// MessageStore 新增方法
const clearSessionMessages = (sessionId: string) => {
  delete messagesMap.value[sessionId]
  console.log('🗑️ 已清空会话消息缓存:', sessionId)
}
```

### 2. 会话复活逻辑

**场景**: 用户删除会话后，对方发送新消息

**处理**:
- 前端检测到会话不存在
- 调用 `loadSessions()` 重新加载会话列表
- 后端自动创建 `session_user` 记录（私聊）
- 会话重新出现在列表顶部
- 本地消息缓存为空，只显示新消息

**效果**: 表现为一个"干净的新会话"

### 3. 消息预览优化

**问题**: 会话列表显示加密的 Base64 字符串

**解决**: 
- 在 `loadSessions` 时解密最后一条消息
- 在 `handleIncomingMessage` 时使用解密后的内容作为预览
- 截取前 20-30 个字符

### 4. 已读状态管理

**优化**:
- 打开会话时自动批量标记未读消息
- 只标记对方发送的消息（不标记自己发送的）
- 使用批量接口提高性能

## 错误处理

### 1. 网络错误

```typescript
try {
  await sessionApi.deleteSession(sessionId)
} catch (error) {
  // 显示错误提示
  console.error('删除会话失败:', error)
  return { success: false, error: String(error) }
}
```

### 2. 解密失败

```typescript
try {
  const content = backendCrypto.decryptWithSession(sessionId, encrypted)
} catch (error) {
  // 显示友好提示，不中断流程
  content = '[解密失败]'
}
```

### 3. 会话密钥缺失

```typescript
try {
  const keyResponse = await encryptionApi.getSessionKey(sessionId)
  backendCrypto.setSessionKey(sessionId, keyResponse.aesKey)
} catch {
  // 密钥不存在，生成新密钥
  const keyResponse = await encryptionApi.generateSessionKey(sessionId)
  backendCrypto.setSessionKey(sessionId, keyResponse.aesKey)
}
```

  if (!messagesMap.value[sessionId]) {
    messagesMap.value[sessionId] = []
  }

## 总结

本优化方案实现了：

1. **清晰的状态管理**: 使用 `sessionsMap` 和 `messagesMap` 隔离数据
2. **完善的删除逻辑**: 删除会话时清空消息缓存
3. **优雅的会话复活**: 删除后收到新消息，会话自动重新出现
4. **实时消息处理**: WebSocket 推送消息，自动更新 UI
5. **友好的错误处理**: 解密失败、网络错误等场景的处理
6. **性能优化**: 消息缓存限制、分页加载等

所有代码都包含详细注释，易于维护和扩展。
