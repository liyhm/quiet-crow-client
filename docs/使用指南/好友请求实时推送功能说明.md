# 好友请求实时推送功能说明

## ✅ 功能概述

好友请求实时推送功能已完成，实现了以下特性：

1. **实时提醒**：收到新的好友请求时，联系人图标立即显示红色数字
2. **自动刷新**：好友请求被接受后，联系人列表自动更新，无需手动刷新
3. **桌面通知**：收到好友请求或请求被接受时，显示桌面通知

## 🎯 解决的问题

### 问题 1：收到好友请求没有提醒
**现象**：需要点击联系人列表才能看到新的好友请求

**解决方案**：
- 侧边栏联系人图标显示红色数字徽章
- 数字表示待处理的好友请求数量
- 通过 WebSocket 实时推送，立即更新

### 问题 2：好友请求被接受后没有自动刷新
**现象**：
- 联系人图标的红色数字不会自动消失
- 联系人列表不会自动显示新好友
- 需要手动刷新页面

**解决方案**：
- 通过 WebSocket 实时推送通知
- 自动刷新好友请求列表
- 自动刷新联系人列表
- 红色数字自动更新

## 📱 用户体验

### 场景 1：收到新的好友请求

1. 用户 B 向你发送好友请求
2. 你的联系人图标立即显示红色数字 "1"
3. 收到桌面通知："新的好友请求 - 用户 B 想添加你为好友"
4. 点击联系人图标，查看好友请求详情

### 场景 2：好友请求被接受

1. 你向用户 B 发送好友请求
2. 用户 B 接受了你的请求
3. 你的联系人图标红色数字立即消失（如果没有其他待处理请求）
4. 收到桌面通知："好友请求已接受 - 用户 B 接受了你的好友请求"
5. 点击联系人图标，用户 B 已经出现在联系人列表中

### 场景 3：好友请求被拒绝

1. 你向用户 B 发送好友请求
2. 用户 B 拒绝了你的请求
3. 你的联系人图标红色数字立即消失（如果没有其他待处理请求）
4. 该请求从待处理列表中移除

## 🔔 桌面通知

### 通知类型

1. **新的好友请求**
   - 标题："新的好友请求"
   - 内容："[用户名] 想添加你为好友"

2. **好友请求已接受**
   - 标题："好友请求已接受"
   - 内容："[用户名] 接受了你的好友请求"

### 通知权限

首次使用时，应用会请求桌面通知权限：
- 点击"允许"即可接收通知
- 可以在浏览器设置中管理通知权限

## 🎨 界面说明

### 侧边栏联系人图标

```
┌─────────┐
│    👥   │  ← 联系人图标
│    ①    │  ← 红色数字徽章（待处理请求数量）
└─────────┘
```

- 没有待处理请求：不显示红色数字
- 1-99 个请求：显示具体数字
- 100+ 个请求：显示 "99+"

### 好友请求列表

点击联系人图标后，会看到：

```
┌─────────────────────────────┐
│ 新的好友请求 (1)            │  ← 可展开/收起
├─────────────────────────────┤
│ 👤 张三 (@zhangsan)         │
│    你好，我想加你为好友      │
│    [接受] [拒绝]            │
└─────────────────────────────┘
```

## ⚙️ 技术实现

### WebSocket 推送

- 订阅路径：`/user/queue/friend-requests`
- 推送类型：
  - `NEW_REQUEST` - 收到新的好友请求
  - `REQUEST_ACCEPTED` - 好友请求被接受
  - `REQUEST_REJECTED` - 好友请求被拒绝

### 自动刷新机制

1. 收到 WebSocket 推送
2. 自动调用 API 刷新数据
3. Pinia Store 更新状态
4. Vue 响应式系统自动更新 UI

### 离线处理

- 用户离线时，WebSocket 推送会失败
- 下次登录时，通过 API 加载最新数据
- 确保不会错过任何好友请求

## 📝 使用说明

### 发送好友请求

1. 点击联系人图标
2. 点击右上角的 "+" 按钮
3. 搜索用户名或昵称
4. 选择用户，输入验证消息
5. 点击"发送请求"

### 处理好友请求

1. 看到联系人图标有红色数字时，点击进入
2. 展开"新的好友请求"列表
3. 查看请求详情
4. 点击"接受"或"拒绝"

### 查看联系人

1. 点击联系人图标
2. 在联系人列表中查看所有好友
3. 点击好友头像，开始聊天

## 🐛 常见问题

### Q: 红色数字不更新？
A: 检查 WebSocket 连接状态，确保后端服务正常运行

### Q: 没有收到桌面通知？
A: 检查浏览器通知权限设置，确保已允许通知

### Q: 好友列表没有自动刷新？
A: 检查控制台日志，确认 WebSocket 推送是否正常接收

### Q: 离线时收到的好友请求会丢失吗？
A: 不会，下次登录时会自动加载所有待处理的好友请求
