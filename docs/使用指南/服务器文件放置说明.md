# 服务器文件放置说明（多项目版本）

## 📁 服务器目录结构（重要！）

你的服务器地址：`http://159.75.182.85/updates/`

### 多项目共存的文件结构

```
/var/www/updates/                    ← 服务器根目录
│
├── AI-IMG-1.0...                    ← 其他项目文件
├── latest.yml                       ← 其他项目的版本信息
│
├── quiet-latest.yml                 ← Quiet 项目 Windows 版本信息（必需）
├── quiet-latest-mac.yml             ← Quiet 项目 Mac 版本信息（必需）
│
├── quiet-1.0.0-setup.exe           ← Quiet Windows 安装包
├── quiet-1.0.0-mac.zip             ← Quiet Mac 更新包（必需）
├── quiet-1.0.0.dmg                 ← Quiet Mac 安装包（可选）
│
├── quiet-1.0.1-setup.exe           ← Quiet Windows 新版本
├── quiet-1.0.1-mac.zip             ← Quiet Mac 新版本（必需）
└── quiet-1.0.1.dmg                 ← Quiet Mac 新版本（可选）
```

---

## ⚠️ 关键规则

### 1. Quiet 项目 Windows 需要的文件
```
quiet-latest.yml              ← 指向最新版本（带项目名前缀）
quiet-{version}-setup.exe     ← 安装包
```

### 2. Quiet 项目 Mac 需要的文件
```
quiet-latest-mac.yml          ← 指向最新版本（带项目名前缀）
quiet-{version}-mac.zip       ← 更新包（必需！）
quiet-{version}.dmg           ← 安装包（可选，首次安装用）
```

### 3. 文件命名规则
- Windows 版本信息：`quiet-latest.yml`
- Mac 版本信息：`quiet-latest-mac.yml`
- Windows 安装包：`quiet-{version}-setup.exe`（例如：`quiet-1.0.0-setup.exe`）
- Mac ZIP：`quiet-{version}-mac.zip`（例如：`quiet-1.0.0-mac.zip`）
- Mac DMG：`quiet-{version}.dmg`（例如：`quiet-1.0.0.dmg`）

## 🔄 更新流程

### 发布 1.0.0 版本

1. 上传文件到服务器：
```bash
/var/www/updates/
├── AI-IMG-1.0...              ← 其他项目不受影响
├── latest.yml                 ← 其他项目不受影响
├── quiet-latest.yml           ← 内容：version: 1.0.0, path: quiet-1.0.0-setup.exe
├── quiet-latest-mac.yml       ← 内容：version: 1.0.0, path: quiet-1.0.0-mac.zip
├── quiet-1.0.0-setup.exe
├── quiet-1.0.0-mac.zip
└── quiet-1.0.0.dmg
```

2. 用户访问：
   - Quiet Windows 应用读取 `quiet-latest.yml` → 看到版本 1.0.0
   - Quiet Mac 应用读取 `quiet-latest-mac.yml` → 看到版本 1.0.0
   - 其他项目读取 `latest.yml` → 不受影响

### 发布 1.0.1 版本

1. 上传新文件到服务器：
```bash
/var/www/updates/
├── AI-IMG-1.0...              ← 其他项目不受影响
├── latest.yml                 ← 其他项目不受影响
├── quiet-latest.yml           ← 覆盖！内容：version: 1.0.1, path: quiet-1.0.1-setup.exe
├── quiet-latest-mac.yml       ← 覆盖！内容：version: 1.0.1, path: quiet-1.0.1-mac.zip
├── quiet-1.0.0-setup.exe      ← 保留旧版本（可选）
├── quiet-1.0.0-mac.zip
├── quiet-1.0.0.dmg
├── quiet-1.0.1-setup.exe      ← 新版本
├── quiet-1.0.1-mac.zip
└── quiet-1.0.1.dmg
```

2. 用户访问：
   - Quiet Windows 应用读取 `quiet-latest.yml` → 看到版本 1.0.1 → 下载 `quiet-1.0.1-setup.exe`
   - Quiet Mac 应用读取 `quiet-latest-mac.yml` → 看到版本 1.0.1 → 下载 `quiet-1.0.1-mac.zip`
   - 其他项目不受影响

## 🚨 常见错误

### ❌ 错误 1：文件名没有项目前缀

```
/var/www/updates/
├── latest.yml              ← 会和其他项目冲突！
└── latest-mac.yml          ← 会和其他项目冲突！
```

**正确做法：**
```
/var/www/updates/
├── quiet-latest.yml        ← 带项目名前缀
└── quiet-latest-mac.yml    ← 带项目名前缀
```

### ❌ 错误 2：Mac 只有 DMG 没有 ZIP

```
/var/www/updates/
├── quiet-latest-mac.yml
├── quiet-1.0.0.dmg        ← 只有 DMG
└── ❌ 缺少 quiet-1.0.0-mac.zip
```

**结果：** Mac 应用无法自动更新！

**正确做法：**
```
/var/www/updates/
├── quiet-latest-mac.yml
├── quiet-1.0.0-mac.zip    ← 必需
└── quiet-1.0.0.dmg        ← 可选
```

### ❌ 错误 3：文件名不匹配

```yaml
# quiet-latest-mac.yml 内容
path: quiet-1.0.0-mac.zip

# 但服务器上的文件名是
quiet-1.0.0.zip  ← 错误！缺少 -mac
```

**结果：** 下载失败，404 错误

## 💡 总结

### 必须记住的 4 点

1. **文件名带项目前缀**：`quiet-latest.yml` 和 `quiet-latest-mac.yml`
2. **Windows** 需要：`quiet-latest.yml` + `quiet-{version}-setup.exe`
3. **Mac** 需要：`quiet-latest-mac.yml` + `quiet-{version}-mac.zip`（不是 `.dmg`）
4. 每次发布新版本，覆盖 `quiet-latest.yml` 和 `quiet-latest-mac.yml`

### 文件放置位置

所有文件都放在同一个目录：`/var/www/updates/`

不需要子目录，不需要分平台文件夹，所有项目的文件平铺在一起，通过文件名前缀区分项目。
