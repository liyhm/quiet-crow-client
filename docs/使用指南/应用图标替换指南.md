# 应用图标替换指南

## 当前问题

打包后的应用在多个地方显示的都是默认的 Electron 图标，而不是我们的 logo：

1. ❌ 安装程序窗口的图标（左上角和大图标）
2. ❌ 桌面快捷方式图标
3. ❌ 任务栏图标
4. ❌ 开始菜单图标
5. ❌ 应用窗口图标

**这些图标都需要替换成我们的 logo！**

## 一次性解决方案

好消息：只需要替换一个文件，所有这些图标都会自动更新！

### 图标文件位置

```
resources/icon.png
```

**重要**：替换这一个文件后，electron-builder 会自动为所有位置生成正确的图标。

## 图标要求

### 1. 文件格式

- **Windows**: PNG 格式（会自动转换为 ICO）
- **macOS**: PNG 格式（会自动转换为 ICNS）
- **Linux**: PNG 格式

### 2. 图标尺寸

推荐使用以下尺寸的 PNG 图片：

- **最小尺寸**: 256x256 像素
- **推荐尺寸**: 512x512 像素
- **最佳尺寸**: 1024x1024 像素

### 3. 图标设计建议

- ✅ 使用正方形画布
- ✅ 图标内容居中
- ✅ 留有适当的内边距（约 10-15%）
- ✅ 使用透明背景（PNG）
- ✅ 高对比度，清晰可辨
- ✅ 简洁明了，避免过多细节

## 替换步骤

### 步骤 1: 准备图标文件

1. 准备你的 logo 图片
2. 确保图片是正方形（宽高相等）
3. **推荐尺寸：1024x1024 像素**（重要！）
4. 保存为 PNG 格式，**必须带透明背景**

**图标设计要求**：
- ✅ 正方形画布（1:1 比例）
- ✅ 透明背景（PNG 格式）
- ✅ 高分辨率（至少 512x512，推荐 1024x1024）
- ✅ 简洁清晰，避免过多细节
- ✅ 留有 10-15% 的内边距
- ✅ 高对比度，在深色和浅色背景下都清晰可见

### 步骤 2: 替换图标文件

**方法 1: 使用文件管理器**

1. 找到项目目录下的 `resources` 文件夹
2. 备份原图标：将 `icon.png` 重命名为 `icon.png.backup`
3. 将你的 logo 文件复制到 `resources` 文件夹
4. 将你的 logo 文件重命名为 `icon.png`

**方法 2: 使用命令行**

```powershell
# 进入项目目录
cd D:\workSpace\workIdea\lyh\quiet-crow-client

# 备份原图标
Copy-Item resources\icon.png resources\icon.png.backup

# 复制新图标（假设新图标在桌面，名为 my-logo.png）
Copy-Item ~\Desktop\my-logo.png resources\icon.png
```

### 步骤 3: 重新打包

替换图标后，需要重新打包应用：

```bash
npm run build:win
```

### 步骤 4: 验证图标

打包完成后，检查以下所有位置的图标是否都已更新：

#### 1. 安装程序图标 ✓
- 位置：`dist/quiet-1.0.0-setup.exe`
- 检查：右键点击安装包，查看图标
- 检查：运行安装程序，查看窗口左上角和中间的大图标

#### 2. 桌面快捷方式图标 ✓
- 位置：安装后的桌面快捷方式
- 检查：查看桌面上的 "quiet" 图标

#### 3. 任务栏图标 ✓
- 位置：应用运行时的任务栏
- 检查：启动应用，查看任务栏图标

#### 4. 开始菜单图标 ✓
- 位置：Windows 开始菜单
- 检查：打开开始菜单，搜索 "quiet"

#### 5. 应用窗口图标 ✓
- 位置：应用窗口左上角
- 检查：启动应用，查看窗口标题栏图标

**如果所有位置的图标都更新了，说明替换成功！** ✅

## 多平台图标配置

如果需要为不同平台使用不同的图标，可以在 `electron-builder.yml` 中配置：

### Windows 图标

```yaml
win:
  icon: resources/icon.png  # 或 resources/icon.ico
```

### macOS 图标

```yaml
mac:
  icon: resources/icon.png  # 或 resources/icon.icns
```

### Linux 图标

```yaml
linux:
  icon: resources/icon.png
```

## 高级配置：使用多尺寸图标

### Windows ICO 文件

如果你有专业的 ICO 文件（包含多个尺寸），可以直接使用：

1. 将 ICO 文件放到 `resources/` 目录
2. 修改 `electron-builder.yml`：

```yaml
win:
  icon: resources/icon.ico
```

### macOS ICNS 文件

如果你有专业的 ICNS 文件，可以直接使用：

1. 将 ICNS 文件放到 `resources/` 目录
2. 修改 `electron-builder.yml`：

```yaml
mac:
  icon: resources/icon.icns
```

## 常见问题

### Q: 替换图标后打包，图标没有变化？

A: 可能的原因：
1. 图标文件路径不对
2. 需要清理缓存后重新打包
3. Windows 系统缓存了旧图标

解决方案：
```powershell
# 清理构建目录
Remove-Item -Recurse -Force dist, out

# 重新打包
npm run build:win
```

### Q: 图标显示模糊或失真？

A: 原因：图标尺寸太小

解决方案：使用至少 512x512 像素的图标

### Q: 图标背景不透明？

A: 原因：PNG 文件没有透明通道

解决方案：使用图像编辑器添加透明背景

### Q: 不同平台需要不同的图标吗？

A: 不需要，一个 PNG 文件可以用于所有平台。electron-builder 会自动转换。

### Q: 安装后图标还是旧的？

A: Windows 系统缓存了图标

解决方案：
1. 卸载旧版本
2. 清理图标缓存：
   ```powershell
   ie4uinit.exe -show
   ```
3. 重启电脑
4. 安装新版本
