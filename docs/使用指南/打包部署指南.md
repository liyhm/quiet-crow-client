## 打包流程详解

### 步骤 1: 编译前端代码

```bash
npm run build
```

这个命令会：
1. 运行 TypeScript 类型检查
2. 使用 Vite 编译 Vue 代码
3. 编译 Electron 主进程和预加载脚本
4. 输出到 `out/` 目录

### 步骤 2: 打包成可执行文件

```bash
# Windows
electron-builder --win

# macOS
electron-builder --mac

# Linux
electron-builder --linux
```

这个命令会：
1. 读取 `electron-builder.yml` 配置
2. 将编译后的代码打包成应用
3. 创建安装程序
4. 输出到 `dist/` 目录

## 打包配置说明

### electron-builder.yml

```yaml
appId: com.electron.app          # 应用 ID
productName: quiet               # 应用名称
directories:
  buildResources: build          # 构建资源目录

win:
  executableName: quiet          # Windows 可执行文件名

nsis:
  artifactName: ${name}-${version}-setup.${ext}  # 安装包文件名
  shortcutName: ${productName}                    # 快捷方式名称
  createDesktopShortcut: always                   # 创建桌面快捷方式
```

### 修改应用信息（可选）

如果需要修改应用名称、图标等，编辑以下文件：

1. **应用名称**：修改 `electron-builder.yml` 中的 `productName`
2. **应用图标**：替换 `resources/icon.png`（至少 512x512 像素）
3. **应用 ID**：修改 `electron-builder.yml` 中的 `appId`

## 常见问题

### 1. 打包失败：网络问题

如果下载 Electron 二进制文件失败，已配置国内镜像：

```yaml
electronDownload:
  mirror: https://npmmirror.com/mirrors/electron/
```

### 2. 打包失败：权限问题

Windows 上可能需要管理员权限：
```bash
# 以管理员身份运行 PowerShell
npm run build:win
```

### 3. 打包后无法连接后端

检查 `.env.production` 文件是否正确配置：
```env
VITE_API_BASE_URL=http://159.75.182.85:8081
VITE_WS_URL=http://159.75.182.85:8081/ws
```

### 4. 打包体积过大

可以在 `electron-builder.yml` 中配置压缩：
```yaml
compression: maximum
```

### 5. macOS 打包需要签名

如果不需要签名，已配置：
```yaml
mac:
  notarize: false
```

## 测试打包后的应用

### 1. 安装应用

- Windows: 双击 `quiet-1.0.0-setup.exe`
- macOS: 打开 `quiet-1.0.0.dmg`，拖动到 Applications
- Linux: 运行 `chmod +x quiet-1.0.0.AppImage && ./quiet-1.0.0.AppImage`

### 2. 打开开发者工具

打包后的应用提供了多种方式打开开发者工具：

**方式 1: 3 次连击 Ctrl 键（推荐）⭐**
- 快速按 3 次 Ctrl 键（0.8 秒内）
- Windows/Linux: Ctrl 键
- macOS: Command 键

**方式 2: F12 快捷键**
- 按 F12 键

**方式 3: Ctrl+Shift+I**
- Windows/Linux: Ctrl+Shift+I
- macOS: Command+Shift+I

详细说明：`docs/使用指南/开发者工具使用说明.md`

### 3. 测试功能

1. 启动应用
2. 注册/登录账号
3. 测试消息发送和接收
4. 测试好友添加
5. 测试头像上传
6. 测试会话管理

### 3. 检查网络连接

打开开发者工具（F12），查看 Console 和 Network 标签：
- 确认 API 请求地址是否正确
- 确认 WebSocket 连接是否成功
- 查看是否有错误信息

将 `dist/quiet-1.0.0-setup.exe` 发送给用户，用户双击安装即可。

## 注意事项

1. **后端地址**：确保 `.env.production` 中的地址正确
2. **网络访问**：确保用户可以访问 `http://159.75.182.85:8081`
3. **防火墙**：确保后端服务器防火墙允许 8081 端口访问
4. **CORS**：确保后端配置了正确的 CORS 策略
5. **WebSocket**：确保后端支持 WebSocket 连接

## 后续优化

5. **代码签名**：Windows 和 macOS 应用签名，避免安全警告
