## 🔧 常见问题和解决方案

### 问题 1：服务器上没有文件

**症状：** curl 返回 404

**解决：** 上传 latest.yml 文件到服务器

```bash
# Windows
scp dist/latest.yml root@159.75.182.85:/var/www/updates/

# Mac
scp dist/latest-mac.yml root@159.75.182.85:/var/www/updates/
```

### 问题 2：文件名不匹配

**症状：** 服务器上有 `quiet-latest.yml`，但应用找不到

**原因：** electron-updater 可能在找 `latest.yml`（不带项目名前缀）

**解决方案 A：** 同时上传两个文件名

```bash
# 在服务器上
cd /var/www/updates/
cp quiet-latest.yml latest.yml
cp quiet-latest-mac.yml latest-mac.yml
```

**解决方案 B：** 修改 electron-builder.yml，去掉 channel

```yaml
publish:
  provider: generic
  url: http://159.75.182.85/updates/
  # channel: quiet  ← 注释掉这一行
```

### 问题 3：CORS 跨域问题

**症状：** 浏览器控制台显示 CORS 错误

**解决：** 在服务器 Nginx 配置中添加 CORS 头

```nginx
location /updates/ {
    add_header Access-Control-Allow-Origin *;
    add_header Access-Control-Allow-Methods 'GET, OPTIONS';
    add_header Access-Control-Allow-Headers 'Range';
    
    if ($request_method = 'OPTIONS') {
        return 204;
    }
}
```

### 问题 4：防火墙阻止

**症状：** 网络连接超时

**解决：** 
1. 检查服务器防火墙是否开放 80 端口
2. 检查本地防火墙是否阻止应用访问网络
3. 检查杀毒软件是否拦截

### 问题 5：文件内容格式错误

**症状：** 能访问文件，但应用报错

**解决：** 检查 yml 文件格式是否正确

```yaml
# 正确格式
version: 1.0.0
files:
  - url: quiet-1.0.0-setup.exe
    sha512: xxxxx...
    size: 99321087
path: quiet-1.0.0-setup.exe
sha512: xxxxx...
releaseDate: '2026-03-05T03:16:20.942Z'
```

**常见错误：**
- ❌ 缩进不正确（必须用空格，不能用 Tab）
- ❌ 版本号格式错误（必须是 x.y.z）
- ❌ 缺少必需字段

## 🎯 快速修复方案

### 最简单的方法：同时上传两个文件名

```bash
# 1. 在本地打包
npm run build:win  # 或 build:mac

# 2. 复制文件
cd dist
cp latest.yml quiet-latest.yml  # Windows
cp latest-mac.yml quiet-latest-mac.yml  # Mac

# 3. 上传所有文件
scp latest.yml root@159.75.182.85:/var/www/updates/
scp quiet-latest.yml root@159.75.182.85:/var/www/updates/
scp quiet-1.0.0-setup.exe root@159.75.182.85:/var/www/updates/
```

这样无论应用找哪个文件名都能找到。
