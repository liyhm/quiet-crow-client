# 跨平台打包指南

## 概述

本应用支持 Windows、macOS、Linux 三个平台。由于平台限制，需要在对应的操作系统上打包。

## 平台限制

| 当前系统 | 可以打包的平台 |
|---------|--------------|
| Windows | Windows, Linux |
| macOS   | macOS, Windows, Linux |
| Linux   | Linux, Windows |

**注意**：
- Windows 无法打包 macOS 应用（需要 macOS 系统和 Xcode）
- macOS 可以打包所有平台（推荐）
- Linux 无法打包 macOS 应用

---

## 方案 1：GitHub Actions 自动打包（推荐）

### 优点
- 自动化，无需手动操作
- 同时打包所有平台
- 自动发布到 GitHub Releases
- 完全免费

### 配置步骤

#### 1. 推送代码到 GitHub

```bash
git add .
git commit -m "准备发布 v1.0.0"
git push origin main
```

#### 2. 创建并推送标签

```bash
# 创建标签
git tag v1.0.0

# 推送标签到 GitHub
git push origin v1.0.0
```

#### 3. 自动打包

GitHub Actions 会自动：
1. 检测到新标签
2. 在 Windows、macOS、Linux 虚拟机上打包
3. 创建 GitHub Release
4. 上传所有安装包

#### 4. 查看结果

访问 GitHub 仓库：
- Actions 标签页：查看打包进度
- Releases 标签页：下载安装包

### 手动触发打包

如果不想创建标签，可以手动触发：

1. 访问 GitHub 仓库
2. 点击 "Actions" 标签页
3. 选择 "Build and Release" 工作流
4. 点击 "Run workflow"
5. 选择分支，点击 "Run workflow"

## 方案 2：本地打包

### Windows 打包

在 Windows 系统上：

```bash
# 打包 Windows 版本
npm run build:win

# 打包 Linux 版本（可选）
npm run build:linux
```

输出文件：
```
dist/
├── quiet-1.0.0-setup.exe          # Windows 安装包
├── latest.yml                      # Windows 更新元数据
├── quiet-1.0.0.AppImage           # Linux 安装包
└── latest-linux.yml                # Linux 更新元数据
```

### macOS 打包

在 macOS 系统上：

```bash
# 打包 macOS 版本
npm run build:mac

# 打包 Windows 版本（可选）
npm run build:win

# 打包 Linux 版本（可选）
npm run build:linux
```

输出文件：
```
dist/
├── quiet-1.0.0.dmg                # macOS 安装包
├── latest-mac.yml                  # macOS 更新元数据
├── quiet-1.0.0-setup.exe          # Windows 安装包
├── latest.yml                      # Windows 更新元数据
├── quiet-1.0.0.AppImage           # Linux 安装包
└── latest-linux.yml                # Linux 更新元数据
```

### Linux 打包

在 Linux 系统上：

```bash
# 打包 Linux 版本
npm run build:linux

# 打包 Windows 版本（可选）
npm run build:win
```

输出文件：
```
dist/
├── quiet-1.0.0.AppImage           # Linux 安装包
├── quiet-1.0.0.deb                # Debian/Ubuntu 安装包
├── latest-linux.yml                # Linux 更新元数据
├── quiet-1.0.0-setup.exe          # Windows 安装包
└── latest.yml                      # Windows 更新元数据
```

## 方案 3：使用云服务器打包

### 租用 macOS 云服务器

如果没有 Mac 电脑，可以租用云服务器：

#### MacStadium
- 专业的 macOS 云服务
- 按小时计费
- 网址：https://www.macstadium.com

#### AWS EC2 Mac
- 亚马逊提供的 macOS 实例
- 按小时计费
- 网址：https://aws.amazon.com/ec2/instance-types/mac/

#### 使用步骤

1. 租用 macOS 云服务器
2. SSH 连接到服务器
3. 克隆代码仓库
4. 安装依赖并打包

```bash
# SSH 连接
ssh user@mac-server-ip

# 克隆代码
git clone https://github.com/your-username/your-repo.git
cd your-repo

# 安装依赖
npm install

# 打包
npm run build:mac

# 下载安装包
scp user@mac-server-ip:~/your-repo/dist/*.dmg ./
```

## macOS 打包配置

### 1. 准备图标

macOS 需要 `.icns` 格式的图标：

```bash
# 使用在线工具转换
# https://cloudconvert.com/png-to-icns

# 或使用命令行工具
brew install imagemagick
convert icon.png -resize 512x512 icon.icns
```

将图标放到 `build/icon.icns`

### 2. 配置 electron-builder.yml

```yaml
mac:
  icon: build/icon.icns
  category: public.app-category.social-networking
  target:
    - dmg
    - zip
  hardenedRuntime: true
  gatekeeperAssess: false
  entitlements: build/entitlements.mac.plist
  entitlementsInherit: build/entitlements.mac.plist

dmg:
  artifactName: ${name}-${version}.${ext}
  background: build/dmg-background.png
  icon: build/icon.icns
  iconSize: 100
  contents:
    - x: 410
      y: 150
      type: link
      path: /Applications
    - x: 130
      y: 150
      type: file
```

### 3. 代码签名（可选）

如果需要发布到 Mac App Store 或避免"未验证的开发者"警告：

```bash
# 需要 Apple Developer 账号（$99/年）
# 配置签名证书
export CSC_LINK=/path/to/certificate.p12
export CSC_KEY_PASSWORD=your-password

# 打包时自动签名
npm run build:mac
```

## 常见问题

### 1. Windows 上能打包 macOS 吗？

不能。macOS 打包需要：
- macOS 系统
- Xcode 命令行工具
- Apple 开发者证书（可选）

### 2. 如何在 Windows 上测试 macOS 版本？

无法直接测试。建议：
- 使用 GitHub Actions 打包
- 在真实 Mac 上测试
- 或使用 macOS 虚拟机

### 3. 打包失败怎么办？

检查：
- Node.js 版本（推荐 20.x）
- 依赖是否完整安装
- 图标文件是否存在
- electron-builder 配置是否正确

### 4. 如何减小安装包体积？

```yaml
# electron-builder.yml
files:
  - '!**/.git'
  - '!**/.vscode'
  - '!**/node_modules/*/{CHANGELOG.md,README.md,README,readme.md,readme}'
  - '!**/node_modules/*/{test,__tests__,tests,powered-test,example,examples}'
  - '!**/node_modules/*.d.ts'
  - '!**/node_modules/.bin'
  - '!**/*.{iml,o,hprof,orig,pyc,pyo,rbc,swp,csproj,sln,xproj}'
  - '!.editorconfig'
  - '!**/._*'
  - '!**/{.DS_Store,.git,.hg,.svn,CVS,RCS,SCCS,.gitignore,.gitattributes}'
  - '!**/{__pycache__,thumbs.db,.flowconfig,.idea,.vs,.nyc_output}'
  - '!**/{appveyor.yml,.travis.yml,circle.yml}'
  - '!**/{npm-debug.log,yarn.lock,.yarn-integrity,.yarn-metadata.json}'
```

## 自动化脚本

### 发布脚本（release.sh）

```bash
#!/bin/bash

# 检查版本号
VERSION=$(node -p "require('./package.json').version")
echo "📦 准备发布版本: v$VERSION"

# 确认发布
read -p "确认发布 v$VERSION? (y/n) " -n 1 -r
echo
if [[ ! $REPLY =~ ^[Yy]$ ]]
then
    exit 1
fi

# 提交代码
git add .
git commit -m "Release v$VERSION"
git push origin main

# 创建并推送标签
git tag v$VERSION
git push origin v$VERSION

echo "✅ 发布完成！"
echo "🌐 GitHub Actions 正在自动打包..."
echo "📦 访问 https://github.com/your-username/your-repo/actions 查看进度"
```

### Windows 发布脚本（release.ps1）

```powershell
# 检查版本号
$version = (Get-Content package.json | ConvertFrom-Json).version
Write-Host "📦 准备发布版本: v$version"

# 确认发布
$confirm = Read-Host "确认发布 v$version? (y/n)"
if ($confirm -ne 'y') {
    exit
}

# 提交代码
git add .
git commit -m "Release v$version"
git push origin main

# 创建并推送标签
git tag "v$version"
git push origin "v$version"

Write-Host "✅ 发布完成！"
Write-Host "🌐 GitHub Actions 正在自动打包..."
Write-Host "📦 访问 https://github.com/your-username/your-repo/actions 查看进度"
```
