# 功能说明 - 删除会话

## 功能概述

用户可以删除不需要的会话，删除后会话将从列表中移除。

## 前端实现

### 1. UI 改进

**会话名称显示优化**:
- 优先显示用户昵称（`showName`）
- 如果没有昵称，显示 `用户 12743e2b...`（用户ID前8位）
- 如果都没有，显示"未知用户"

**删除按钮**:
- 鼠标悬停在会话上时显示删除按钮（垃圾桶图标）
- 点击删除按钮弹出确认对话框
- 确认后删除会话

## 后端需求

### 删除会话 API

**端点**: `DELETE /api/session/{sessionId}`

**功能**: 删除指定的会话

**请求**:
```
DELETE /api/session/12743e2b142fc6dc8f8642bf928cc636
Authorization: Bearer {token}
```

**响应**:
```json
{
  "code": 200,
  "message": "success",
  "data": null
}
```

### 实现建议

#### 方案1: 软删除（推荐）

只删除当前用户的 `session_user` 记录，不删除会话本身和消息。

```java
@DeleteMapping("/{sessionId}")
public Result<Void> deleteSession(@PathVariable String sessionId) {
    String userId = SecurityUtils.getCurrentUserId();
    
    // 删除当前用户的 session_user 记录
    sessionUserRepository.deleteBySessionIdAndUserId(sessionId, userId);
    
    return Result.success();
}
```

**优点**:
- 对方用户仍然可以看到会话和消息
- 可以恢复（如果需要）
- 不影响其他用户

**缺点**:
- 数据库中仍保留会话和消息记录

#### 方案2: 硬删除

删除会话、所有 `session_user` 记录和所有消息。

```java
@DeleteMapping("/{sessionId}")
@Transactional
public Result<Void> deleteSession(@PathVariable String sessionId) {
    String userId = SecurityUtils.getCurrentUserId();
    
    // 验证用户是否是会话成员
    SessionUser sessionUser = sessionUserRepository
        .findBySessionIdAndUserId(sessionId, userId)
        .orElseThrow(() -> new RuntimeException("会话不存在"));
    
    // 删除所有消息
    messageRepository.deleteBySessionId(sessionId);
    
    // 删除所有 session_user 记录
    sessionUserRepository.deleteBySessionId(sessionId);
    
    // 删除会话
    sessionRepository.deleteById(sessionId);
    
    return Result.success();
}
```

**优点**:
- 彻底清理数据
- 节省存储空间

**缺点**:
- 对方用户也会失去会话和消息
- 无法恢复
- 可能引起用户困惑

### 推荐方案

**使用软删除（方案1）**，原因：
1. 符合用户预期（只删除自己这边的会话）
2. 不影响其他用户
3. 可以在需要时恢复
4. 实现简单

### 数据库操作

```sql
-- 删除当前用户的 session_user 记录
DELETE FROM session_user 
WHERE session_id = ? AND user_id = ?;
```

## 用户体验

### 删除流程

1. 用户鼠标悬停在会话上
2. 显示删除按钮（垃圾桶图标）
3. 点击删除按钮
4. 弹出确认对话框："确定要删除这个会话吗？"
5. 用户点击"确定"
6. 会话从列表中消失
7. 如果是当前选中的会话，右侧显示"选择一个会话开始聊天"

### 注意事项

1. **确认对话框**: 防止误删
2. **清除选中状态**: 如果删除的是当前会话，需要清除选中
3. **视觉反馈**: 删除按钮只在悬停时显示，避免界面混乱
4. **错误处理**: 如果删除失败，显示错误提示
