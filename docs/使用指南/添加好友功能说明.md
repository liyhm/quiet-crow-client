# 添加好友功能说明

## ✅ 已更新的功能

### 前端更新

1. **搜索用户功能** ✅
   - 通过用户名或昵称搜索用户
   - 实时显示搜索结果
   - 支持选择用户

2. **添加好友流程优化** ✅
   - 不再需要输入用户 ID
   - 搜索 → 选择 → 发送请求
   - 更友好的用户体验

### 后端 API 对接

根据后端文档，已对接以下 API：

1. **用户搜索**: `GET /api/user/search?keyword={关键词}`
2. **通过用户名添加好友**: `POST /api/contact/request/by-username`
3. **获取好友列表**: `GET /api/contact/list`
4. **获取待处理请求**: `GET /api/contact/pending`
5. **接受好友请求**: `PUT /api/contact/accept/{requestId}`
6. **拒绝好友请求**: `PUT /api/contact/reject/{requestId}`

## 使用流程

### 1. 添加好友

1. 点击联系人标签
2. 点击右上角 "+" 按钮
3. 在搜索框输入用户名或昵称
4. 点击"搜索"或按 Enter
5. 从搜索结果中选择用户
6. 填写验证消息（可选）
7. 点击"发送请求"

### 2. 处理好友请求

1. 联系人页面顶部会显示"新的好友请求"通知
2. 点击展开查看请求列表
3. 每个请求显示：
   - 用户头像和昵称
   - 验证消息
   - 接受/拒绝按钮
4. 点击"接受"或"拒绝"处理请求

### 3. 开始聊天

1. 在联系人列表点击好友
2. 自动创建私聊会话
3. 切换到消息标签
4. 开始聊天

## 技术实现

### API 端点更新

```typescript
// API 配置
CONTACTS: {
  LIST: '/api/contact/list',
  REQUESTS: '/api/contact/request',
  REQUEST_BY_USERNAME: '/api/contact/request/by-username',  // 新增
  PENDING: '/api/contact/pending',
  ACCEPT: (id: string) => `/api/contact/accept/${id}`,
  REJECT: (id: string) => `/api/contact/reject/${id}`,
  DELETE: (userId: string) => `/api/contact/${userId}`
}
```

### 新增接口

```typescript
// 通过用户名发送好友请求
sendRequestByUsername(params: SendRequestByUsernameParams) {
  return request.post(API_ENDPOINTS.CONTACTS.REQUEST_BY_USERNAME, params)
}
```

### Store 方法

```typescript

const sendRequestByUsername = async (username: string, message: string) => {
  try {
    await contactsApi.sendRequestByUsername({ username, message })

  }

```

## 错误处理

1. **搜索失败**
   - 显示错误提示
   - 清空搜索结果

2. **未找到用户**
   - 显示"未找到用户"提示

3. **发送请求失败**
   - 显示具体错误信息
   - 保持弹窗打开，允许重试

4. **网络错误**
   - 显示友好的错误提示
   - 记录详细日志

## 注意事项

### 后端要求

1. 确保后端已启动在 `http://localhost:8080`
2. 确保 CORS 已正确配置
3. 确保数据库已初始化

### API 路径

根据后端文档，API 路径为：
- `/api/contact/*` （不是 `/api/contacts/*`）
- `/api/user/search` （不是 `/api/users/search`）
