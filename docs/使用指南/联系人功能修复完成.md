# 联系人功能修复完成

## 修复的问题

### 问题1: 联系人列表显示错误
**症状**: 所有联系人都显示为当前用户的信息

**原因**: `useContactStore.ts` 中的 `loadContacts()` 方法错误地为每个联系人调用 `userApi.getInfo()`，这个 API 返回的是当前登录用户的信息，而不是联系人的信息。

**修复**: 
- 更新了 `ContactRelation` 接口，匹配后端实际返回的数据结构
- 修改了 `loadContacts()` 方法，直接使用后端返回的联系人信息，不再额外调用 API
- 移除了未使用的导入

## 修改的文件

### 1. `src/renderer/src/shared/api/contacts.ts`

**修改前**:
```typescript
export interface ContactRelation {
  id: string
  fromUserId: string
  toUserId: string
  status: 'PENDING' | 'ACCEPTED' | 'REJECTED'
  remark: string | null
}
```

**修改后**:
```typescript
export interface ContactRelation {
  id: string
  userId: string
  username: string
  showName: string
  avatar: string
  remark: string | null
  addedTime: string
  publicKey: string
  status?: 'PENDING' | 'ACCEPTED' | 'REJECTED'
}
```

### 2. `src/renderer/src/entities/contact/model/useContactStore.ts`

**修改前**:
```typescript
const loadContacts = async () => {
  // ...
  const contactPromises = relations
    .filter((r) => r.status === 'ACCEPTED')
    .map(async (relation) => {
      try {
        const userInfo = await userApi.getInfo()  // ❌ 错误：返回当前用户信息
        return {
          userId: relation.toUserId,
          username: userInfo.username,
          displayName: userInfo.showName,
          // ...
        }
      } catch (error) {
        return null
      }
    })
  // ...
}
```

**修改后**:
```typescript
const loadContacts = async () => {
  // ...
  // 后端返回的联系人列表已经包含了用户信息
  // 直接映射到 Contact 接口
  contacts.value = relations
    .filter((r) => r.status === 'ACCEPTED')
    .map((relation: any) => ({
      userId: relation.userId || relation.toUserId,
      username: relation.username || 'unknown',
      displayName: relation.showName || relation.username || '未知用户',
      avatar: relation.avatar || '',
      status: 'ONLINE',
      statusMessage: ''
    }))
  // ...
}
```

## 技术要点

### 1. 数据映射
后端返回的字段名可能与前端不完全一致，需要做兼容处理：
```typescript
userId: relation.userId || relation.toUserId,
username: relation.username || 'unknown',
displayName: relation.showName || relation.username || '未知用户',
```

### 2. 性能优化
- 修复前：N+1 查询问题（每个联系人都调用一次 API）
- 修复后：只调用一次 API，直接使用返回的数据

### 3. 错误处理
- 提供默认值避免显示 undefined
- 使用 `||` 运算符提供降级方案

## 注意事项

1. **后端依赖**: 确保后端 `/api/contact/list` 返回完整的用户信息
2. **数据格式**: 如果后端返回的字段名不同，需要调整映射逻辑
3. **状态过滤**: 只显示 `status === 'ACCEPTED'` 的联系人
4. **空状态**: 如果没有好友，显示"暂无联系人"提示
