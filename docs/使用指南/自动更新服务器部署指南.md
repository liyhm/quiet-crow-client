# 自动更新服务器部署指南

## 概述

本应用使用 `electron-updater` 实现自动更新功能。更新服务器需要托管以下文件：
- 安装包文件（.exe、.dmg、.AppImage 等）
- `latest.yml`（Windows）或 `latest-mac.yml`（macOS）元数据文件

## 方案选择

### 方案 1：GitHub Releases（推荐，免费）

最简单的方案，完全免费，自动生成 `latest.yml`。

#### 配置步骤

1. **修改 `electron-builder.yml`**

```yaml
publish:
  provider: github
  owner: your-github-username
  repo: your-repo-name
  private: false  # 如果是私有仓库，设为 true
```

2. **修改 `package.json`**

添加 repository 字段：

```json
{
  "repository": {
    "type": "git",
    "url": "https://github.com/your-username/your-repo.git"
  }
}
```

3. **设置 GitHub Token**

```bash
# Windows PowerShell
$env:GH_TOKEN="your-github-token"

# Linux/Mac
export GH_TOKEN=your-github-token
```

获取 Token：
- 访问 https://github.com/settings/tokens
- 点击 "Generate new token (classic)"
- 勾选 `repo` 权限
- 生成并复制 token

4. **打包并发布**

```bash
npm run build:win
```

electron-builder 会自动：
- 打包应用
- 创建 GitHub Release
- 上传安装包和 `latest.yml`

5. **用户端自动更新**

应用会自动从 GitHub Releases 检查更新。

#### 优点
- 完全免费
- 自动生成元数据文件
- 支持私有仓库
- CDN 加速下载

#### 缺点
- 需要 GitHub 账号
- 国内访问可能较慢（可使用代理）

### 方案 2：自建静态文件服务器

适合需要完全控制的场景。

#### 1. 准备服务器

需要一台服务器（Linux/Windows）和域名，例如：
- 服务器：阿里云、腾讯云、AWS 等
- 域名：`updates.yourdomain.com`
- Web 服务器：Nginx、Apache、Caddy 等

#### 2. 配置 Nginx

```nginx
server {
    listen 80;
    server_name updates.yourdomain.com;
    
    # 自动更新文件目录
    root /var/www/updates;
    
    # 启用目录浏览（可选）
    autoindex on;
    
    # CORS 配置（允许跨域下载）
    add_header Access-Control-Allow-Origin *;
    add_header Access-Control-Allow-Methods 'GET, HEAD, OPTIONS';
    
    # 缓存配置
    location ~* \.(yml|yaml)$ {
        expires 5m;  # 元数据文件缓存 5 分钟
    }
    
    location ~* \.(exe|dmg|AppImage|deb)$ {
        expires 30d;  # 安装包缓存 30 天
    }
}
```

#### 3. 目录结构

```
/var/www/updates/
├── latest.yml              # Windows 元数据
├── latest-mac.yml          # macOS 元数据
├── quiet-1.0.0-setup.exe   # Windows 安装包
├── quiet-1.0.1-setup.exe   # 新版本
└── quiet-1.0.1.dmg         # macOS 安装包
```

#### 4. 修改配置文件

**electron-builder.yml**

```yaml
publish:
  provider: generic
  url: https://updates.yourdomain.com
```

**dev-app-update.yml**（开发环境）

```yaml
provider: generic
url: https://updates.yourdomain.com
updaterCacheDirName: quiet-updater
```

#### 5. 打包应用

```bash
npm run build:win
```

打包完成后，在 `dist` 目录找到：
- `quiet-1.0.0-setup.exe`（安装包）
- `latest.yml`（元数据文件）

#### 6. 上传文件到服务器

```bash
# 使用 scp 上传
scp dist/quiet-1.0.0-setup.exe root@your-server:/var/www/updates/
scp dist/latest.yml root@your-server:/var/www/updates/
```

#### 7. latest.yml 文件格式

```yaml
version: 1.0.0
files:
  - url: quiet-1.0.0-setup.exe
    sha512: [文件的 SHA512 哈希值]
    size: 123456789
path: quiet-1.0.0-setup.exe
sha512: [文件的 SHA512 哈希值]
releaseDate: '2026-03-05T12:00:00.000Z'
```

electron-builder 会自动生成这个文件，包含正确的哈希值。

### 方案 3：对象存储（OSS/S3）

适合大规模分发，成本低，速度快。

#### 使用阿里云 OSS

1. **创建 OSS Bucket**
   - 访问阿里云 OSS 控制台
   - 创建 Bucket，设置为"公共读"
   - 记录 Bucket 名称和 Endpoint

2. **配置 electron-builder.yml**

```yaml
publish:
  provider: generic
  url: https://your-bucket.oss-cn-hangzhou.aliyuncs.com/updates
```

3. **上传文件**

使用 OSS 控制台或 ossutil 工具上传：

```bash
# 安装 ossutil
# 配置访问密钥
ossutil config

# 上传文件
ossutil cp dist/quiet-1.0.0-setup.exe oss://your-bucket/updates/
ossutil cp dist/latest.yml oss://your-bucket/updates/
```

4. **配置 CDN 加速（可选）**
   - 在 OSS 控制台绑定自定义域名
   - 启用 CDN 加速
   - 使用自定义域名：`https://updates.yourdomain.com`

#### 使用 AWS S3

```yaml
publish:
  provider: s3
  bucket: your-bucket-name
  region: us-east-1
  path: updates
```

## 版本发布流程

### 1. 更新版本号

修改 `package.json`：

```json
{
  "version": "1.0.1"
}
```

### 2. 打包新版本

```bash
npm run build:win
```

### 3. 上传文件

根据选择的方案上传：
- GitHub Releases：自动上传
- 自建服务器：手动上传到 `/var/www/updates/`
- OSS/S3：使用工具上传

### 4. 用户端更新

用户打开应用后：
1. 应用自动检查更新（启动时 + 每小时）
2. 发现新版本，显示更新弹窗
3. 用户点击"立即更新"
4. 后台下载安装包
5. 下载完成，提示重启安装

## 测试自动更新

### 1. 本地测试

1. 打包当前版本（如 1.0.0）并安装
2. 修改版本号为 1.0.1
3. 重新打包
4. 上传 1.0.1 的文件到服务器
5. 打开已安装的 1.0.0 应用
6. 点击"设置" → "检查更新"
7. 应该显示更新弹窗

### 2. 验证更新服务器

访问以下 URL 检查文件是否可访问：

```
https://updates.yourdomain.com/latest.yml
https://updates.yourdomain.com/quiet-1.0.1-setup.exe
```

### 3. 查看日志

应用会输出更新日志：

```
🔄 检查更新...
✅ 发现新版本: 1.0.1
📥 开始下载更新...
✅ 下载完成
```

## 常见问题

### 1. 更新检查失败

**原因**：
- 服务器 URL 配置错误
- `latest.yml` 文件不存在或格式错误
- 网络连接问题
- CORS 配置错误

**解决**：
- 检查 `electron-builder.yml` 中的 `publish.url`
- 确保 `latest.yml` 可以通过浏览器访问
- 检查服务器 CORS 配置

### 2. 下载失败

**原因**：
- 安装包文件不存在
- 文件 SHA512 哈希值不匹配
- 网络中断

**解决**：
- 确保安装包文件已上传
- 不要手动修改 `latest.yml`，使用 electron-builder 生成
- 检查网络连接

### 3. 安装失败

**原因**：
- 安装包损坏
- 权限不足
- 杀毒软件拦截

**解决**：
- 重新下载安装包
- 以管理员身份运行
- 添加到杀毒软件白名单

### 4. 国内访问 GitHub 慢

**解决方案**：
- 使用 GitHub 代理服务（如 ghproxy.com）
- 使用国内 OSS 镜像
- 自建服务器

## 推荐配置

### 小型项目（< 1000 用户）

使用 **GitHub Releases**：
- 免费
- 配置简单
- 自动化程度高

### 中型项目（1000-10000 用户）

使用 **阿里云 OSS + CDN**：
- 成本低（约 ¥10-50/月）
- 速度快
- 稳定可靠

### 大型项目（> 10000 用户）

使用 **专业 CDN + 对象存储**：
- 全球加速
- 高可用性
- 支持大规模并发

---

## 安全建议

1. **使用 HTTPS**：确保更新服务器使用 HTTPS
2. **验证签名**：electron-updater 会自动验证文件哈希
3. **私有仓库**：敏感应用使用私有 GitHub 仓库或私有服务器
4. **访问控制**：限制更新服务器的访问权限
5. **备份文件**：保留历史版本的安装包

## 自动化脚本

### 发布脚本（release.sh）

```bash
#!/bin/bash

# 1. 检查版本号
VERSION=$(node -p "require('./package.json').version")
echo "📦 准备发布版本: $VERSION"

# 2. 打包
echo "🔨 开始打包..."
npm run build:win

# 3. 上传到服务器（自建服务器方案）
echo "📤 上传文件到服务器..."
scp dist/quiet-$VERSION-setup.exe root@your-server:/var/www/updates/
scp dist/latest.yml root@your-server:/var/www/updates/

echo "✅ 发布完成！"
echo "🌐 更新地址: https://updates.yourdomain.com/latest.yml"
```

### Windows 发布脚本（release.ps1）

```powershell
# 1. 检查版本号
$version = (Get-Content package.json | ConvertFrom-Json).version
Write-Host "📦 准备发布版本: $version"

# 2. 打包
Write-Host "🔨 开始打包..."
npm run build:win

# 3. 上传到 OSS（使用 ossutil）
Write-Host "📤 上传文件到 OSS..."
ossutil cp "dist/quiet-$version-setup.exe" "oss://your-bucket/updates/"
ossutil cp "dist/latest.yml" "oss://your-bucket/updates/"

Write-Host "✅ 发布完成！"
```
