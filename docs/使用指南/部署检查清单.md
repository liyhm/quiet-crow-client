## 🔧 配置步骤

### 步骤 1: 更新生产环境配置

编辑 `.env.production`：

```env
# 替换为实际的生产服务器地址
VITE_API_BASE_URL=https://api.your-domain.com
VITE_WS_URL=wss://api.your-domain.com/ws
```

**示例配置：**

```env
# 使用子域名（推荐）
VITE_API_BASE_URL=https://api.quietcrow.com
VITE_WS_URL=wss://api.quietcrow.com/ws

# 或使用路径
VITE_API_BASE_URL=https://quietcrow.com/api
VITE_WS_URL=wss://quietcrow.com/ws
```

### 步骤 2: 验证后端服务

```bash
# 测试 HTTP API
curl https://api.your-domain.com/api/auth/login

# 测试 WebSocket（使用 wscat 工具）
wscat -c wss://api.your-domain.com/ws
```

### 步骤 3: 构建生产版本

```bash
# 清理旧的构建文件
rm -rf out/
rm -rf dist/

# 构建前端
npm run build

# 构建 Electron 应用
npm run build:win   # Windows
npm run build:mac   # macOS
npm run build:linux # Linux
```

### 步骤 4: 测试构建产物

1. 安装构建的应用
2. 测试登录功能
3. 测试消息发送
4. 测试 WebSocket 连接
5. 检查网络请求地址

## 🌐 后端服务器要求

### 最低要求

1. **HTTPS 支持**
   - 有效的 SSL 证书
   - 强制 HTTPS 重定向
   - 支持 TLS 1.2+

2. **WebSocket 支持**
   - WSS (WebSocket Secure)
   - 支持 SockJS
   - 支持 STOMP 协议

3. **CORS 配置**

   ```java
   // Spring Boot 示例
   @Configuration
   public class CorsConfig {
       @Bean
       public WebMvcConfigurer corsConfigurer() {
           return new WebMvcConfigurer() {
               @Override
               public void addCorsMappings(CorsRegistry registry) {
                   registry.addMapping("/**")
                       .allowedOrigins("https://your-app-domain.com")
                       .allowedMethods("GET", "POST", "PUT", "DELETE")
                       .allowCredentials(true);
               }
           };
       }
   }
   ```

4. **性能优化**
   - 启用 Gzip 压缩
   - 配置缓存策略
   - 数据库连接池
   - Redis 缓存（可选）

### 推荐配置

1. **负载均衡**
   - Nginx 反向代理
   - 多实例部署
   - 健康检查

2. **监控告警**
   - 服务器监控
   - 应用性能监控
   - 错误日志收集

3. **安全加固**
   - 防火墙规则
   - DDoS 防护
   - 请求限流
   - SQL 注入防护

## 🐛 常见问题

### Q1: 打包后连接不上服务器

**检查：**

1. `.env.production` 配置是否正确
2. 后端服务器是否可访问
3. 防火墙是否开放端口
4. SSL 证书是否有效

### Q2: WebSocket 连接失败

**检查：**

1. 是否使用 `wss://` 而不是 `ws://`
2. 后端是否支持 WebSocket
3. Nginx 是否正确配置 WebSocket 代理
4. 防火墙是否允许 WebSocket 连接

### Q3: CORS 错误

**解决：**

1. 后端配置 CORS 允许的域名
2. 确认请求头正确
3. 检查预检请求（OPTIONS）

### Q4: 打包体积过大

**优化：**

1. 移除未使用的依赖
2. 使用代码分割
3. 压缩图片资源
4. 使用生产模式构建
