# 开发者工具使用说明

## 如何打开开发者工具（控制台）

打包后的应用提供了多种方式打开开发者工具，方便调试和查看日志。

### 方式 1: 3 次连击 Ctrl 键（推荐）⭐

**操作**: 快速按 3 次 Ctrl 键（800ms 内）

**说明**:
- Windows/Linux: 按 3 次 `Ctrl` 键
- macOS: 按 3 次 `Command` 键
- 时间限制: 必须在 800 毫秒（0.8 秒）内完成 3 次按键
- 效果: 打开或关闭开发者工具

**优点**:
- ✅ 隐蔽，不容易被普通用户误触
- ✅ 适合生产环境
- ✅ 跨平台支持

### 方式 2: F12 快捷键

**操作**: 按 `F12` 键

**说明**:
- 标准的开发者工具快捷键
- 打开或关闭开发者工具

### 方式 3: Ctrl+Shift+I

**操作**: 按 `Ctrl+Shift+I`（macOS: `Command+Shift+I`）

**说明**:
- Chrome 浏览器的标准快捷键
- 打开或关闭开发者工具

### 方式 4: Ctrl+Shift+C

**操作**: 按 `Ctrl+Shift+C`（macOS: `Command+Shift+C`）

**说明**:
- 打开开发者工具并激活元素选择器
- 开发者工具会以独立窗口打开

### Network（网络）标签

查看 API 请求和响应：

- 查看请求 URL
- 查看请求参数
- 查看响应数据
- 查看请求耗时
- 检查 WebSocket 连接

### Elements（元素）标签

查看和调试 UI 元素：

- 检查 HTML 结构
- 查看 CSS 样式
- 调试布局问题

### Application（应用）标签

查看应用数据：

- Local Storage（本地存储）
- Session Storage（会话存储）
- IndexedDB（数据库）
- Cookies

## 常见调试场景

### 场景 1: 检查 API 地址是否正确

1. 按 3 次 Ctrl 打开开发者工具

4. 确认 `BASE_URL` 和 `WS_URL` 是否正确

1. 打开开发者工具
2. 切换到 Console 标签

1. 打开开发者工具
2. 切换到 Console 标签
3. 查找 `🔌 WebSocket` 相关日志
4. 切换到 Network 标签，点击 WS 过滤器
5. 查看 WebSocket 连接状态

1. 打开开发者工具
2. 切换到 Console 标签
3. 发送一条消息
4. 查看 `📤 发送消息` 日志
5. 查看 `📥 收到消息` 日志

### 场景 5: 检查错误信息

1. 打开开发者工具
2. 切换到 Console 标签
3. 查找红色的错误信息（❌ 开头）
4. 查看错误详情和堆栈信息

## 生产环境调试

### 为什么需要隐藏的快捷键？

在生产环境中，我们不希望普通用户轻易打开开发者工具，但开发人员和技术支持人员需要能够调试问题。因此：

- ✅ 使用 3 次连击 Ctrl 这种隐蔽的方式
- ✅ 不在 UI 上显示任何开发者工具的入口
- ✅ 保留 F12 等标准快捷键供技术人员使用

### 技术支持场景

当用户报告问题时，技术支持人员可以：

1. 远程指导用户按 3 次 Ctrl 键
2. 打开开发者工具
3. 切换到 Console 标签
4. 截图或复制错误信息
5. 发送给技术支持团队分析

## 关闭开发者工具

### 方式 1: 再次按快捷键

- 3 次连击 Ctrl
- 或按 F12
- 或按 Ctrl+Shift+I

### 方式 2: 点击关闭按钮

点击开发者工具窗口右上角的 ✕ 按钮

## 注意事项

### 1. 时间限制

3 次连击 Ctrl 必须在 800ms 内完成，如果太慢会重置计数。

### 2. 键盘焦点

确保应用窗口有焦点（处于激活状态），否则快捷键可能不生效。

### 3. 跨平台差异

- Windows/Linux: 使用 `Ctrl` 键
- macOS: 使用 `Command` 键

### 4. 开发环境 vs 生产环境

- 开发环境: 启动时自动打开开发者工具
- 生产环境: 需要手动使用快捷键打开

## 实现原理

### 3 次连击检测

```typescript
let ctrlPressCount = 0
let ctrlPressTimer: number | null = null

const handleKeyDown = (event: KeyboardEvent) => {
  if (event.key === 'Control' || event.key === 'Meta') {
    ctrlPressCount++
    
    if (ctrlPressTimer) {
      clearTimeout(ctrlPressTimer)
    }
    
    if (ctrlPressCount === 3) {
      // 打开开发者工具
      window.api.devtools.toggle()
      ctrlPressCount = 0
    }
    
    // 800ms 后重置计数
    ctrlPressTimer = window.setTimeout(() => {
      ctrlPressCount = 0
    }, 800)
  }
}
```

### IPC 通信

渲染进程通过 IPC 通知主进程打开开发者工具：

```typescript
// 渲染进程
window.api.devtools.toggle()

// 主进程
ipcMain.handle('devtools:toggle', () => {
  const focusedWindow = BrowserWindow.getFocusedWindow()
  if (focusedWindow) {
    if (focusedWindow.webContents.isDevToolsOpened()) {
      focusedWindow.webContents.closeDevTools()
    } else {
      focusedWindow.webContents.openDevTools()
    }
  }
})
```

## 常见问题

### Q: 为什么按 3 次 Ctrl 没反应？

A: 可能的原因：
1. 按键速度太慢（超过 800ms）
2. 应用窗口没有焦点
3. 按错了键（应该是 Ctrl 或 Command）

### Q: 可以修改为其他快捷键吗？

A: 可以，修改 `src/renderer/src/App.vue` 中的 `handleKeyDown` 函数。

### Q: 可以修改连击次数吗？

A: 可以，修改 `if (ctrlPressCount === 3)` 中的数字。

### Q: 可以修改时间限制吗？

A: 可以，修改 `setTimeout` 中的 `800` 毫秒。

### Q: 生产环境可以禁用开发者工具吗？

A: 不建议完全禁用，因为技术支持需要调试。如果必须禁用，可以在主进程中添加环境判断。

## 总结

打包后的应用提供了多种方式打开开发者工具：

- ⭐ **3 次连击 Ctrl**（推荐，隐蔽）
- **F12**（标准快捷键）
- **Ctrl+Shift+I**（Chrome 标准）
- **Ctrl+Shift+C**（元素选择器）

推荐在生产环境使用 3 次连击 Ctrl 的方式，既方便技术支持调试，又不会被普通用户误触。
