# 环境配置说明

## 概述

项目支持多环境配置，通过环境变量区分开发环境和生产环境的后端地址。

## 环境变量文件

### `.env.development` - 开发环境

```env
VITE_API_BASE_URL=http://localhost:8080
VITE_WS_URL=ws://localhost:8080/ws
```

### `.env.production` - 生产环境

```env
VITE_API_BASE_URL=https://api.your-domain.com
VITE_WS_URL=wss://api.your-domain.com/ws
```

### `.env.example` - 示例文件

包含所有可配置的环境变量，用于参考。

## 配置说明

### 环境变量

| 变量名              | 说明              | 开发环境示例             | 生产环境示例                   |
| ------------------- | ----------------- | ------------------------ | ------------------------------ |
| `VITE_API_BASE_URL` | HTTP API 基础地址 | `http://localhost:8080`  | `https://api.your-domain.com`  |
| `VITE_WS_URL`       | WebSocket 地址    | `ws://localhost:8080/ws` | `wss://api.your-domain.com/ws` |

### 注意事项

1. **环境变量前缀**：Vite 要求环境变量必须以 `VITE_` 开头才能在客户端代码中访问
2. **WebSocket 协议**：
   - 开发环境使用 `ws://`（非加密）
   - 生产环境使用 `wss://`（加密，HTTPS 必须）
3. **HTTPS 要求**：生产环境建议使用 HTTPS，WebSocket 也必须使用 WSS

## 使用方式

### 在代码中访问环境变量

```typescript
// 直接访问
const apiUrl = import.meta.env.VITE_API_BASE_URL
const wsUrl = import.meta.env.VITE_WS_URL

// 通过配置对象访问（推荐）
import { API_CONFIG } from '@/shared/config/api'

console.log(API_CONFIG.BASE_URL) // 当前环境的 API 地址
console.log(API_CONFIG.WS_URL) // 当前环境的 WebSocket 地址
console.log(API_CONFIG.ENV) // 当前环境：development 或 production
console.log(API_CONFIG.IS_DEV) // 是否为开发环境
console.log(API_CONFIG.IS_PROD) // 是否为生产环境
```

### 配置文件

所有环境相关配置都在 `src/renderer/src/shared/config/api.ts`：

```typescript
export const API_CONFIG = {
  BASE_URL: import.meta.env.VITE_API_BASE_URL || 'http://localhost:8080',
  WS_URL: import.meta.env.VITE_WS_URL || 'ws://localhost:8080/ws',
  TOKEN_KEY: 'chat_token',
  TIMEOUT: 30000,
  ENV: import.meta.env.MODE,
  IS_PROD: import.meta.env.PROD,
  IS_DEV: import.meta.env.DEV
}
```

## 开发流程


## 部署前配置

### 1. 修改生产环境配置

编辑 `.env.production` 文件：

```env
# 替换为实际的生产服务器地址
VITE_API_BASE_URL=https://api.your-domain.com
VITE_WS_URL=wss://api.your-domain.com/ws
```

### 2. 后端服务器要求

生产环境后端服务器需要：

1. **HTTPS 支持**：配置 SSL 证书
2. **WebSocket 支持**：确保 WSS 可用
3. **CORS 配置**：允许前端域名访问
4. **防火墙规则**：开放相应端口

### 3. 域名配置示例

假设你的域名是 `example.com`：

```env
# 方案 1：使用子域名
VITE_API_BASE_URL=https://api.example.com
VITE_WS_URL=wss://api.example.com/ws

# 方案 2：使用路径
VITE_API_BASE_URL=https://example.com/api
VITE_WS_URL=wss://example.com/ws

# 方案 3：使用端口（不推荐生产环境）
VITE_API_BASE_URL=https://example.com:8080
VITE_WS_URL=wss://example.com:8080/ws
```

## 多环境支持

### 添加测试环境

1. 创建 `.env.test` 文件：

```env
VITE_API_BASE_URL=https://test-api.your-domain.com
VITE_WS_URL=wss://test-api.your-domain.com/ws
```

2. 在 `package.json` 添加脚本：

```json
{
  "scripts": {
    "dev:test": "vite --mode test",
    "build:test": "vite build --mode test"
  }
}
```

### 添加预发布环境

1. 创建 `.env.staging` 文件：

```env
VITE_API_BASE_URL=https://staging-api.your-domain.com
VITE_WS_URL=wss://staging-api.your-domain.com/ws
```

2. 在 `package.json` 添加脚本：

```json
{
  "scripts": {
    "dev:staging": "vite --mode staging",
    "build:staging": "vite build --mode staging"
  }
}
```

## 环境检测

### 在代码中检测当前环境

```typescript
import { API_CONFIG } from '@/shared/config/api'

// 根据环境执行不同逻辑
if (API_CONFIG.IS_DEV) {
  console.log('开发环境，启用调试功能')
  // 启用详细日志
  // 启用 Mock 数据
}

if (API_CONFIG.IS_PROD) {
  console.log('生产环境，禁用调试功能')
  // 禁用 console.log
  // 启用错误上报
}

// 根据环境显示不同信息
console.log(`当前环境: ${API_CONFIG.ENV}`)
console.log(`API 地址: ${API_CONFIG.BASE_URL}`)
console.log(`WebSocket 地址: ${API_CONFIG.WS_URL}`)
```

### 条件编译

```typescript
// 仅在开发环境执行
if (import.meta.env.DEV) {

  window.__DEV_TOOLS__ = true
}

// 仅在生产环境执行

  // 禁用开发者工具

```

## 安全建议

### 1. 不要提交敏感信息

`.env.production` 可能包含生产环境地址，建议：

```bash
# 添加到 .gitignore
.env.production
.env.local
```

### 2. 使用环境变量模板

提交 `.env.example` 作为模板，团队成员复制后修改。

### 3. 生产环境配置

生产环境配置应该：

- 使用 HTTPS/WSS
- 配置 CORS 白名单
- 启用请求限流
- 配置防火墙规则

## 常见问题

### Q1: 修改环境变量后不生效？

A: 需要重启开发服务器：

```bash
# 停止当前服务（Ctrl+C）
# 重新启动
npm run dev
```

### Q2: 打包后连接的还是开发环境？

A: 检查 `.env.production` 文件是否正确配置，确保使用 `npm run build` 而不是 `npm run dev`。

### Q3: WebSocket 连接失败？

A: 检查：

1. 生产环境是否使用 `wss://` 而不是 `ws://`
2. 后端是否支持 WebSocket
3. 防火墙是否开放端口
4. SSL 证书是否有效

### Q4: 如何在运行时切换环境？

A: 环境变量在构建时确定，无法在运行时切换。如需动态切换，可以：

1. 提供配置界面让用户输入服务器地址
2. 使用配置文件而不是环境变量
3. 构建多个版本的安装包
