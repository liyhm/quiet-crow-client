# 绿色按钮显示修复说明

## 问题描述

微信绿色 (`#95EC69`) 的按钮在某些情况下不显示，只有鼠标悬停时才出现。

## 根本原因

Tailwind CSS 的自定义颜色 `bg-wechat-green` 可能由于以下原因未正确应用：
1. Tailwind 编译缓存问题
2. 浏览器缓存
3. CSS 优先级问题

## 解决方案

使用内联样式 `style="background-color: #95EC69"` 替代 Tailwind 类名，确保颜色始终显示。

## 为什么使用内联样式

1. **确保显示**: 内联样式优先级最高，不会被其他 CSS 覆盖
2. **避免缓存**: 不依赖 Tailwind 编译，直接使用颜色值
3. **兼容性好**: 所有浏览器都支持内联样式
4. **立即生效**: 不需要重新编译或清除缓存

## 悬停效果

虽然移除了 `hover:bg-green-600`，但可以通过以下方式添加悬停效果：

### 方法 1: 使用 CSS 类（推荐）
```vue
<style scoped>
.btn-green {
  background-color: #95EC69;
}
.btn-green:hover {
  background-color: #7ED957;
}
</style>
```

### 方法 2: 使用 @apply（如果 Tailwind 正常工作）
```vue
<style scoped>
.btn-green {
  @apply bg-wechat-green hover:bg-green-600;
}
</style>
```

### 方法 3: JavaScript 动态样式
```vue
<button
  @mouseenter="hovered = true"
  @mouseleave="hovered = false"
  :style="{ backgroundColor: hovered ? '#7ED957' : '#95EC69' }"
>
```

## 其他保持不变的绿色元素

这些元素通常不会有显示问题，因为它们不是交互元素。

## 如果问题仍然存在

1. **清除浏览器缓存**: Ctrl+Shift+Delete
2. **重启开发服务器**: 
   ```bash
   # 停止当前服务
   Ctrl+C
   # 重新启动
   npm run dev
   ```
3. **清除 Tailwind 缓存**:
   ```bash
   rm -rf node_modules/.cache
   npm run dev
   ```
4. **硬刷新页面**: Ctrl+Shift+R
