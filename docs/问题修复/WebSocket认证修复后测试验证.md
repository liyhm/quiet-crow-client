## 问题排查

### 如果收不到消息

**检查 1：后端日志**
```
✅ WebSocket 用户认证成功: {userId}
```
- 如果看到 `anonymous`，说明认证失败
- 检查前端是否传递了 token

**检查 2：前端 token**
```javascript
console.log('Token:', authStore.token)
```
- 如果是 `null` 或 `undefined`，需要重新登录

**检查 3：消息转发**
```
📤 转发消息到: /user/{userId}/queue/messages
```
- 检查 userId 是否正确
- 检查是否转发给了正确的用户

**检查 4：前端订阅**
```
✅ 私有消息队列订阅成功
```
- 如果没有这条日志，说明订阅失败
- 检查 WebSocket 连接是否成功

---

### 如果消息显示位置错误

**检查控制台日志**：
```
📨 当前消息列表: {
  currentUserId: "xxx",
  messages: [
    { senderId: "yyy", isMine: true/false }
  ]
}
```

**验证**：
- `currentUserId` 应该是当前登录用户的 ID
- `senderId` 应该是消息发送者的 ID
- `isMine` 应该是 `senderId === currentUserId`

---

### 如果已读状态不持久

**检查后端日志**：
```
✅ 消息已标记为已读: {messageId}
```

**检查数据库**：
```sql
SELECT id, send_user, is_read 
FROM b_session_log 
WHERE session_id = 'xxx' 
ORDER BY send_time DESC;
```

**验证**：
- `is_read` 字段应该是 `true`
- 刷新后仍然是 `true`
