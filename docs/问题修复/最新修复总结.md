## 修复的问题

### 1. ✅ 页面刷新后加密初始化报错

**问题描述**:
```
Failed to load private key: TypeError: Cannot read properties of undefined (reading 'secureStorage')
at CryptoService.loadPrivateKeySecurely (crypto.ts:287:40)
at Proxy.initialize (useCryptoStore.ts:18:46)
at main.ts:37:21
```

**根本原因**:
页面刷新时，preload 脚本可能还未完全加载，导致 `window.api.secureStorage` 未定义。

**修复方案**:
在访问 `window.api.secureStorage` 之前添加检查，如果 API 不可用则跳过加密初始化，记录警告但不阻塞应用。

**修改的文件**:
- `src/renderer/src/entities/crypto/model/useCryptoStore.ts`
- `src/renderer/src/shared/lib/crypto.ts`

**代码变更**:
```typescript
// 添加 API 可用性检查
if (!window.api || !window.api.secureStorage) {
  console.warn('⚠️ Secure storage API not available')
  return // 或返回默认值
}
```

### 2. 🔍 接受好友申请后，好友没有出现在列表中

**问题描述**:
用户接受好友请求后，联系人列表中没有显示新添加的好友。

**可能原因**:
1. 后端返回的数据格式与前端期望不一致
2. 过滤条件过于严格（只显示 `status === 'ACCEPTED'` 的联系人）
3. 后端可能没有返回 `status` 字段
4. 字段名映射不正确

**修复方案**:
1. 添加详细的调试日志，输出后端返回的原始数据
2. 放宽过滤条件：如果没有 `status` 字段，也认为是已接受的好友
3. 增强字段映射的兼容性，支持多种字段名
4. 在接受请求后添加详细的日志输出

**修改的文件**:
- `src/renderer/src/entities/contact/model/useContactStore.ts`

**代码变更**:
```typescript
// 1. 详细的日志输出
console.log('✅ 后端返回的联系人关系:', JSON.stringify(relations, null, 2))

// 2. 放宽过滤条件
.filter((r) => {
  console.log('检查联系人状态:', r)
  return r.status === 'ACCEPTED' || !r.status // 兼容没有 status 字段的情况
})

// 3. 增强字段映射
const contact = {
  userId: relation.userId || relation.toUserId || relation.id,
  username: relation.username || 'unknown',
  displayName: relation.showName || relation.username || '未知用户',
  // ...
}
console.log('映射联系人:', contact)
```

## 调试指南

### 如果好友仍然不显示

查看控制台中 "后端返回的联系人关系" 的日志：

#### 情况1: 返回空数组 `[]`
**原因**: 后端没有返回好友关系数据
**检查**:
- 后端数据库中的 `contact_relation` 表
- 后端 API `/api/contact/list` 的实现
- 是否正确建立了好友关系

#### 情况2: 数据格式不正确
**检查字段名**:
- 前端期望：`userId`, `username`, `showName`
- 后端返回：可能是 `toUserId`, `userName`, `displayName` 等

**解决方案**: 调整前端的字段映射逻辑

#### 情况3: 有 `status` 字段且值不是 `'ACCEPTED'`
**检查**: 后端返回的 `status` 值是什么
**解决方案**: 
- 如果是 `'ACTIVE'` 或其他值，调整过滤条件
- 或者移除 `status` 字段的检查

public List<ContactRelationVO> getContactList(String userId) {
    List<ContactRelation> relations = contactRepository.findByUserId(userId);
