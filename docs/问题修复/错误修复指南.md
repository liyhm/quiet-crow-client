# 🔧 错误修复指南

## 最新错误：403 Token 验证失败 ⚠️

### 错误信息
```
Failed to load resource: the server responded with a status of 403
Token validation failed: AxiosError: Request failed with status code 403
```

### 原因分析
- localStorage 中有旧的或无效的 token
- 应用启动时尝试用这个 token 验证，但后端返回 403 禁止访问

### 解决方案（3种方法）

#### 方法 1: 清除浏览器缓存（最简单）✅

1. 打开浏览器开发者工具（F12）
2. 进入 Application/应用 标签
3. 找到 Local Storage
4. 删除 `auth_token` 键
5. 刷新页面

#### 方法 2: 控制台命令（最快）✅

在浏览器控制台执行：
```javascript
localStorage.clear()
location.reload()
```

#### 方法 3: 已自动修复 ✅

代码已更新，现在会自动处理：
- ✅ 403/401 错误时自动清除 token
- ✅ 自动跳转到登录页
- ✅ 改进日志输出

## 当前错误分析

根据你的截图，有以下错误：

### 1. ✅ Vue Router 警告（已修复）

```
[Vue Router warn]: The next() callback in navigation guards is deprecated
```

**原因**: Vue Router 4.x 废弃了 `next()` 回调方式

**解决**: 已更新路由守卫使用新的返回值方式

### 2. ❌ CORS 错误（需要启动后端）

```
Connecting to 'http://localhost:8080/api/session/my' violates Content Security Policy
```

**原因**: 后端服务未启动

**解决方案**:

#### 方案 A: 启动后端服务（推荐）

1. 确保后端已启动在 `http://localhost:8080`
2. 访问 `http://localhost:8080/swagger-ui.html` 验证后端正常
3. 重启前端应用

#### 方案 B: 临时使用 Mock 数据

如果暂时没有后端，可以使用 Mock 数据进行开发。

### 3. ❌ 网络错误（需要启动后端）

```
Failed to load sessions: AxiosError: Network Error
```

**原因**: 无法连接到后端 API

**解决**: 同上，启动后端服务

## 🚀 完整解决步骤

### 步骤 1: 清除旧数据 ⭐ 重要

```javascript
// 在浏览器控制台执行
localStorage.clear()
sessionStorage.clear()
location.reload()
```

### 步骤 2: 启动后端

```bash
# 进入后端项目目录
cd /path/to/backend

# 启动后端服务
# Spring Boot 项目通常是：
./mvnw spring-boot:run
# 或
java -jar target/your-app.jar
```

### 步骤 3: 验证后端

访问 Swagger UI：

```
http://localhost:8080/swagger-ui.html
```

应该能看到 API 文档界面。

### 步骤 4: 重启前端

```bash
# 停止当前前端（Ctrl+C）
# 重新启动
npm run dev
```

### 步骤 5: 注册新账号

1. 打开应用
2. 点击"没有账号？去注册"
3. 填写用户名、昵称、密码
4. 注册成功后自动登录

## 🛠️ 常见问题

### Q1: 后端启动失败

**检查**:

- 端口 8080 是否被占用
- 数据库是否启动
- 配置文件是否正确

**解决**:

```bash
# Windows 查看端口占用
netstat -ano | findstr :8080

# 杀死占用进程
taskkill /PID <进程ID> /F
```

### Q2: CORS 错误持续存在

**后端需要配置 CORS** - 参考 `后端CORS配置指南.md`

### Q3: 前端连接不上后端

**检查网络请求**:

1. 打开开发者工具（F12）
2. 切换到 Network 标签
3. 刷新页面
4. 查看失败的请求详情

**验证后端地址**:

```javascript
// 在浏览器控制台
console.log(import.meta.env.VITE_API_BASE_URL)
// 应该输出: http://localhost:8080
```

### Q4: 登录后立即退出

**可能原因**:

- Token 验证失败
- 用户信息获取失败

**检查**:

1. 查看控制台错误
2. 查看 Network 面板的 API 响应
3. 确认后端返回的数据格式正确

### Q5: 403 Token 验证失败 ⭐ 新增

**原因**: localStorage 有旧的无效 token

**解决**: 
```javascript
localStorage.clear()
location.reload()
```
