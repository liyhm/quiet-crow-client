# 消息立即显示完整解决方案

## 问题历史

### TASK 18：消息重复发送问题
- **问题**：群聊发一次消息，显示 2 次
- **原因**：乐观更新 + WebSocket 回调，导致消息添加两次
- **解决**：移除乐观更新，只通过 WebSocket 回调添加消息

### TASK 20：发送方消息不立即显示
- **问题**：发送方发送图片后，消息不会立即显示，需要手动点击会话
- **原因**：移除乐观更新后，必须等待 WebSocket 回调（100-500ms 延迟）
- **解决**：恢复乐观更新 + 改进去重逻辑

## 最终解决方案

### 核心思路

**乐观更新 + 智能去重 = 即时显示 + 无重复**

1. **发送时**：立即添加临时消息到本地（乐观更新）
2. **回调时**：查找临时消息并替换为真实消息（智能去重）
3. **接收时**：直接添加消息（无需去重）

### 实现细节

#### 1. 临时消息 ID 格式

```typescript
const tempMessageId = `temp-${Date.now()}-${Math.random()}`
```

- `temp-` 前缀：标识临时消息
- `Date.now()`：时间戳，确保唯一性
- `Math.random()`：随机数，避免冲突

#### 2. 发送消息（乐观更新）

**文本消息**（`useMessageStore.ts`）：


#### 3. 接收消息（智能去重）

```typescript
const receiveMessage = async (encryptedMessage): Promise<Message | null> => {
  const messageId = encryptedMessage.id || `msg-${Date.now()}`
  const existingMessages = messagesMap.value[sessionId] || []
  
  // 解密消息
  const content = backendCrypto.decryptWithSession(sessionId, encryptedMessage.content)
  
  // 步骤 1：检查是否有相同 ID 的消息（避免重复）
  const duplicateById = existingMessages.some(m => m.id === messageId)
  if (duplicateById) {
    console.warn('消息已存在（相同ID），跳过')
    return null
  }
  
  // 步骤 2：如果是自己发送的消息，查找临时消息并替换
  if (senderId === authStore.userId) {
    const tempMessage = existingMessages.find(m => 
      m.id.startsWith('temp-') &&           // 临时消息
      m.senderId === senderId &&            // 发送者相同
      m.content === content &&              // 内容相同
      Math.abs(timestamp1 - timestamp2) < 5000  // 时间相差 < 5秒
    )
    
    if (tempMessage) {
      // 替换临时消息
      tempMessage.id = messageId
      tempMessage.status = MessageStatus.DELIVERED
      tempMessage.timestamp = timestamp
      return tempMessage
    }
  }
  
  // 步骤 3：如果没有找到临时消息，添加新消息
  const message: Message = { ... }
  addMessage(sessionId, message)
  return message
}
```

### 匹配条件

临时消息和 WebSocket 回调消息的匹配条件：

| 条件 | 说明 |
|------|------|
| `m.id.startsWith('temp-')` | 临时消息标识 |
| `m.senderId === senderId` | 发送者相同 |
| `m.content === content` | 内容完全相同 |
| `Math.abs(t1 - t2) < 5000` | 时间戳相差小于 5 秒 |

### 消息状态流转

```
发送时：SENDING (发送中)
  ↓
WebSocket 发送成功：SENT (已发送)
  ↓
WebSocket 回调：DELIVERED (已送达) - 替换临时消息
```

## 技术亮点

1. **临时 ID 机制**：使用 `temp-` 前缀标识临时消息
2. **智能匹配**：通过发送者、内容、时间戳匹配临时消息
3. **原地替换**：直接更新临时消息的 ID 和状态，避免重新渲染
4. **时间容差**：5 秒的时间容差，处理网络延迟
5. **Vue 响应式**：利用 Vue 的响应式系统，自动更新 UI

## 注意事项

1. 临时 ID 必须使用 `temp-` 前缀
2. 时间戳匹配容差为 5 秒（可根据实际情况调整）
3. 只有自己发送的消息才会进行临时消息替换
4. 接收方的消息直接添加，不会触发临时消息替换逻辑
5. 消息内容必须完全相同才能匹配（包括 JSON 格式的媒体消息）
