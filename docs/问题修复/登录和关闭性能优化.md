# 登录和关闭性能优化

## 问题描述

用户反馈：
1. 登录偶尔要很久才能进入主页面
2. 点击右上角关闭按钮要等很久才关闭

## 问题分析

### 登录慢的原因

1. **串行等待**：所有操作都是串行执行
   - 登录 API → 密钥生成 → WebSocket 连接 → 等待 1 秒 → 跳转
2. **不必要的等待**：
   - `setTimeout(100)` - 无意义的等待
   - `setTimeout(1000)` - 等待 WebSocket 连接
3. **阻塞跳转**：WebSocket 连接完成前不跳转

### 关闭窗口慢的原因

1. **WebSocket 断开等待**：等待 WebSocket 优雅断开
2. **资源清理阻塞**：清理操作阻塞窗口关闭
3. **使用 close() 而不是 destroy()**：close() 会等待窗口事件处理完成

## 优化方案

### 1. 登录流程优化

**优化前：**
```javascript
// 串行执行
await authStore.login()
await cryptoStore.generateRSAKeyPair()
await setTimeout(100)
websocketService.connect()
await setTimeout(1000)
router.push('/')
```

**优化后：**
```javascript
// 并行执行
await authStore.login()

// 密钥生成和 WebSocket 连接并行
const tasks = []
if (!cryptoStore.hasKeyPair) {
  tasks.push(cryptoStore.generateRSAKeyPair().catch(...))
}

// WebSocket 在后台连接，不阻塞跳转
websocketService.initialize()
websocketService.connect()

// 只等待密钥生成（如果有）
if (tasks.length > 0) {
  await Promise.all(tasks)
}

// 立即跳转，不等待 WebSocket
router.push('/')
```

**优化效果：**
- 移除了 1100ms 的无意义等待
- WebSocket 在后台连接，不阻塞页面跳转
- 密钥生成失败不影响登录流程

### 2. 关闭窗口优化

**优化前：**
```javascript
// 前端
const closeWindow = async () => {
  await window.api.window.close()
}

// 主进程
ipcMain.handle('window:close', () => {
  mainWindow?.close() // 等待窗口事件处理完成
})
```

**优化后：**
```javascript
// 前端
const closeWindow = async () => {
  // 快速断开 WebSocket（不等待）
  try {
    websocketService.disconnect()
  } catch (error) {
    console.warn('⚠️ 断开 WebSocket 时出错（忽略）:', error)
  }
  
  // 立即关闭窗口
  await window.api.window.close()
}

// 主进程
ipcMain.handle('window:close', () => {
  // 使用 destroy() 强制立即关闭
  if (mainWindow) {
    mainWindow.destroy()
    mainWindow = null
  }
  
  // 在后台清理资源（不阻塞关闭）
  setTimeout(() => {
    console.log('🧹 后台清理资源...')
  }, 0)
})
```

**优化效果：**
- 使用 `destroy()` 强制立即关闭窗口
- WebSocket 断开不阻塞关闭流程
- 资源清理在后台异步执行

### 3. 登出流程优化

**优化前：**
```javascript
const logout = () => {
  token.value = null
  currentUser.value = null
  localStorage.removeItem(API_CONFIG.TOKEN_KEY)
  
  const { backendCrypto } = require('@/shared/lib/cryptoBackend')
  backendCrypto.clearAllKeys()
}
```

**优化后：**
```javascript
const logout = () => {
  // 快速清理本地数据
  token.value = null
  currentUser.value = null
  localStorage.removeItem(API_CONFIG.TOKEN_KEY)
  
  // 清除会话密钥（带错误处理）
  try {
    const { backendCrypto } = require('@/shared/lib/cryptoBackend')
    backendCrypto.clearAllKeys()
  } catch (error) {
    console.warn('⚠️ 清除密钥时出错（忽略）:', error)
  }
  
  // 断开 WebSocket（不等待）
  try {
    const { websocketService } = require('@/shared/api/websocket')
    websocketService.disconnect()
  } catch (error) {
    console.warn('⚠️ 断开 WebSocket 时出错（忽略）:', error)
  }
}
```

## 修改的文件

1. `src/renderer/src/pages/login/LoginPage.vue`
   - 移除登录后的 1100ms 等待
   - WebSocket 在后台连接，不阻塞跳转
   - 密钥生成和 WebSocket 连接并行处理
   - 关闭窗口前快速断开 WebSocket

2. `src/renderer/src/entities/user/model/useAuthStore.ts`
   - 登出时快速清理资源
   - 添加错误处理，防止清理失败阻塞登出
   - 断开 WebSocket 连接

3. `src/renderer/src/shared/api/websocket.ts`
   - 优化 disconnect() 方法
   - 清理所有订阅
   - 添加错误处理

4. `src/main/index.ts`
   - 使用 `destroy()` 强制立即关闭窗口
   - 资源清理在后台异步执行

5. `src/renderer/src/pages/main/MainPage.vue`
   - 关闭窗口前快速断开 WebSocket

## 测试建议

1. **登录速度测试**：
   - 测试正常登录，应该在 1-2 秒内完成
   - 测试网络慢的情况，不应该卡住

2. **关闭窗口测试**：
   - 点击关闭按钮，应该立即关闭
   - 不应该有明显的延迟

3. **WebSocket 测试**：
   - 登录后检查 WebSocket 是否正常连接
   - 关闭窗口后检查 WebSocket 是否正常断开

## 预期效果

- 登录速度提升约 1-2 秒（移除等待时间）
- 关闭窗口立即响应（<100ms）
- WebSocket 在后台连接，不影响用户体验
