# 实时消息推送问题排查

## 当前状态
- ✅ 消息能收到（点击会话后能看到历史消息）
- ❌ 没有实时推送（需要手动点击会话才显示新消息）
- ❌ 会话列表没有显示未读数

## 问题分析

### 可能的原因

#### 1. WebSocket 消息没有触发 UI 更新
消息到达后，store 更新了，但 Vue 的响应式系统没有触发重新渲染。

#### 2. 消息接收逻辑有问题
WebSocket 收到消息后，没有正确调用 store 的方法。

#### 3. 会话列表没有监听消息更新
会话列表组件没有响应消息 store 的变化。

## 调试步骤

### 步骤 1：检查 WebSocket 是否真的连接了

打开开发者工具（F12），查看控制台，应该看到：
```
✅ WebSocket 连接成功！
✅ 私有消息队列订阅成功
```

**如果没有这些日志**：
- WebSocket 没有连接
- 消息是从历史记录加载的，不是实时推送的

### 步骤 2：测试实时消息接收

1. 打开两个窗口（或一个应用 + 一个浏览器）
2. 用户A发送消息
3. 立即查看用户B的控制台

**应该看到的日志**：
```
📬 收到私有消息: {...}
📨 收到 WebSocket 消息: {...}
📩 处理接收到的消息: {...}
🔓 消息解密成功: 你好
✅ 消息已添加到 store
✅ 消息已处理，会话列表已更新
```

**如果没有这些日志**：
- WebSocket 没有收到消息
- 后端没有转发消息

### 步骤 3：检查会话列表是否更新

发送消息后，查看会话列表：
- 应该显示最后一条消息
- 应该显示未读数（红色数字）
- 会话应该排到最上面

**如果没有更新**：
- `sessionStore.updateLastMessage()` 没有被调用
- 或者调用了但 UI 没有响应

## 快速测试方法

### 测试 1：检查 WebSocket 连接状态

在控制台执行：
```javascript
// 查看连接状态
console.log('连接状态:', networkStore.connectionStatus)
```

应该返回：`"CONNECTED"`

### 测试 2：手动触发消息接收

在控制台执行：
```javascript
// 模拟收到消息
const messageStore = useMessageStore()
messageStore.receiveMessage({
  id: 'test-123',
  sessionId: '当前会话ID',
  sendUser: '对方用户ID',
  content: '测试消息（已加密）',
  sendTime: new Date().toISOString(),
  messageType: '1'
})
```

看看消息是否出现在聊天界面。

### 测试 3：检查会话列表响应式

在控制台执行：
```javascript
// 查看会话列表
console.log('会话列表:', sessionStore.sessions)

// 手动更新会话
sessionStore.updateLastMessage('会话ID', '测试消息', new Date().toISOString())
sessionStore.incrementUnread('会话ID')
```

看看会话列表是否更新。

## 可能的修复方案

### 方案 1：确保 WebSocket 正确连接

检查登录后是否初始化了 WebSocket：
```javascript
// 在 LoginPage.vue 的 handleLogin 中
websocketService.initialize()
websocketService.connect()
```

### 方案 2：确保订阅了正确的队列

检查 WebSocket 服务是否订阅了 `/user/queue/messages`：
```javascript
// 在 websocket.ts 的 subscribeToPrivateMessages 中
this.client.subscribe(`/user/queue/messages`, (message) => {
  this.handleIncomingMessage(JSON.parse(message.body))
})
```

### 方案 3：确保消息处理逻辑正确

检查 `handleIncomingMessage` 是否正确调用了 store：
```javascript
private handleIncomingMessage(message: any) {
  const messageStore = useMessageStore()
  messageStore.receiveMessage(message)
  
  const sessionStore = useSessionStore()
  sessionStore.updateLastMessage(...)
  sessionStore.incrementUnread(...)
}
```

### 方案 4：确保 Vue 响应式正常

检查 store 是否使用了 `ref()` 或 `reactive()`：
```javascript
// 在 useSessionStore 中
const sessions = ref<ChatSession[]>([])  // ✅ 正确

// 不要这样：
const sessions = []  // ❌ 错误，不是响应式的
```

## 后端需要检查的点

### 1. 消息是否正确转发

后端收到消息后，应该打印：
```
📨 收到消息 - 会话: xxx, 发送者: xxx
✅ 消息已保存，ID: xxx
📋 会话成员: [user1, user2]
📤 转发消息给用户: user2
✅ 消息已转发给: user2
```

### 2. 转发路径是否正确

```java
messagingTemplate.convertAndSendToUser(
    memberId,
    "/queue/messages",  // 必须是这个路径
    response
);
```

### 3. 消息格式是否正确

```java
ChatMessageResponse response = new ChatMessageResponse();
response.setId(message.getId());
response.setSessionId(sessionId);
response.setSendUser(senderId);  // 注意：是 sendUser 不是 senderId
response.setContent(request.getContent());
response.setSendTime(message.getSendTime().toString());
response.setMessageType(request.getMessageType());
```

## 临时解决方案

如果实时推送还没修复，可以使用轮询：

```javascript
// 每5秒刷新一次会话列表
setInterval(() => {
  sessionStore.loadSessions()
}, 5000)
```

但这不是最佳方案，应该修复 WebSocket 实时推送。
