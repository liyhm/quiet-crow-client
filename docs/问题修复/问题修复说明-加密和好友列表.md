# 问题修复说明 - 加密初始化和好友列表

## 修复的问题

### 问题1: 页面刷新后加密初始化报错 ✅

**错误信息**:
```
Failed to load private key: TypeError: Cannot read properties of undefined (reading 'secureStorage')
```

**原因**: 
页面刷新时，`window.api` 对象可能还未完全初始化，导致访问 `window.api.secureStorage` 时出错。

**修复方案**:
1. 在 `useCryptoStore.ts` 的 `initialize()` 方法中添加检查
2. 在 `crypto.ts` 的 `savePrivateKeySecurely()` 和 `loadPrivateKeySecurely()` 方法中添加检查
3. 如果 API 不可用，记录警告但不阻塞应用启动

**修改的文件**:
- `src/renderer/src/entities/crypto/model/useCryptoStore.ts`
- `src/renderer/src/shared/lib/crypto.ts`

**修改内容**:


### 问题2: 接受好友申请后，好友没有出现在列表中 🔍

**可能的原因**:
1. 后端返回的数据格式与前端期望不一致
2. 数据过滤条件过于严格（只显示 `status === 'ACCEPTED'` 的联系人）
3. 后端可能没有返回 `status` 字段

**修复方案**:
1. 添加详细的调试日志，查看后端返回的实际数据
2. 放宽过滤条件：如果没有 `status` 字段，也认为是已接受的好友
3. 添加多个字段的兼容性处理

**修改的文件**:
- `src/renderer/src/entities/contact/model/useContactStore.ts`

**修改内容**:

```typescript
const loadContacts = async () => {
  try {
    loading.value = true
    console.log('📇 加载联系人列表...')
    
    const relations = await contactsApi.getList()
    console.log('✅ 后端返回的联系人关系:', JSON.stringify(relations, null, 2))

    contacts.value = relations
      .filter((r) => {
        console.log('检查联系人状态:', r)
        // 如果没有 status 字段，也认为是已接受的
        return r.status === 'ACCEPTED' || !r.status
      })
      .map((relation: any) => {
        const contact = {
          // 兼容多种字段名
          userId: relation.userId || relation.toUserId || relation.id,
          username: relation.username || 'unknown',
          displayName: relation.showName || relation.username || '未知用户',
          avatar: relation.avatar || '',
          status: 'ONLINE',
          statusMessage: ''
        }
        console.log('映射联系人:', contact)
        return contact
      })
    
    console.log('✅ 联系人列表加载完成:', contacts.value.length, '个联系人')
    return { success: true }
  }
}

const acceptRequest = async (requestId: string) => {

    console.log('✅ 接受请求响应:', response)
```

## 调试步骤

### 1. 测试加密初始化修复

1. 刷新页面（F5 或 Ctrl+R）
2. 打开浏览器控制台
3. 应该看到以下日志之一：
   - `⚠️ Secure storage API not available, skipping crypto initialization`（API 未就绪）
   - `✅ Crypto initialized`（初始化成功）
4. 不应该再看到 "Cannot read properties of undefined" 错误

### 2. 测试好友列表更新

1. 用两个账号测试（账号A 和 账号B）
2. 账号A 发送好友请求给账号B
3. 账号B 登录，查看好友请求
4. 点击"接受"按钮
5. 查看控制台日志，应该看到：


6. 联系人列表应该显示新添加的好友

### 3. 检查后端返回的数据格式

如果好友仍然没有出现，查看控制台中 "后端返回的联系人关系" 的日志，检查：

1. **是否有数据返回**？
   - 如果是空数组 `[]`，说明后端没有返回好友关系
   - 检查后端数据库中的 `contact_relation` 表

2. **数据格式是否正确**？
   ```json
   [
     {
       "id": "relation-uuid",
       "userId": "user-uuid",
       "username": "testuser",
       "showName": "测试用户",
       "avatar": "",
       "remark": null,
       "addedTime": "2024-02-27T10:00:00",
       "publicKey": "...",
       "status": "ACCEPTED"  // 可能没有这个字段
     }
   ]
   ```

3. **字段名是否匹配**？
   - 前端期望：`userId`, `username`, `showName`
   - 如果后端返回的字段名不同，需要调整映射逻辑

## 可能需要的后端修复

如果前端日志显示后端返回的数据为空或格式不正确，可能需要检查后端：

### 1. 检查 ContactController.getList() 方法

确保返回的数据包含用户信息：

```java
@GetMapping("/list")
public Result<List<ContactRelationVO>> getContactList() {
    String userId = SecurityUtils.getCurrentUserId();
    List<ContactRelationVO> contacts = contactService.getContactList(userId);
    return Result.success(contacts);
}
```

### 2. 检查 ContactRelationVO 类

确保包含所有必要的字段：

```java
public class ContactRelationVO {
    private String id;
    private String userId;      // 好友的用户ID
    private String username;    // 好友的用户名
    private String showName;    // 好友的昵称
    private String avatar;      // 好友的头像
    private String remark;      // 备注
    private LocalDateTime addedTime;
    private String publicKey;
    // 可选：private String status;
}
```

### 3. 检查 ContactService.getContactList() 方法

确保正确关联用户表：

```java
public List<ContactRelationVO> getContactList(String userId) {
    // 查询好友关系
    List<ContactRelation> relations = contactRepository.findByUserId(userId);
    
    // 关联用户信息
    return relations.stream()
        .map(relation -> {
            User friend = userRepository.findById(relation.getFriendUserId());
            ContactRelationVO vo = new ContactRelationVO();
            vo.setId(relation.getId());
            vo.setUserId(friend.getId());
            vo.setUsername(friend.getUsername());
            vo.setShowName(friend.getShowName());
            vo.setAvatar(friend.getAvatar());
            vo.setRemark(relation.getRemark());
            vo.setAddedTime(relation.getAddedTime());
            vo.setPublicKey(friend.getPublicKey());
            return vo;
        })
        .collect(Collectors.toList());
}
```
