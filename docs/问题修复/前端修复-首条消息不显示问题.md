# 前端修复 - 首条消息不显示问题

## 问题描述

**现象**:
- 用户 A 给用户 B 发送第一条消息
- 用户 B 的会话列表能看到消息（显示"刚刚"）✅
- 但右侧聊天面板没有显示消息 ❌
- 用户 B 点击会话后，消息才显示
- 后续消息能正常显示

**根本原因**:
1. WebSocket 收到消息，调用 `addMessage()` 添加到 `messagesMap`
2. 但是 `messagesMap[sessionId]` 不存在时，直接赋值 `messagesMap.value[sessionId] = []` 
3. 这种方式可能不会触发 Vue 的响应式更新
4. 导致 `currentMessages` 计算属性没有重新计算
5. UI 没有更新

---

## 解决方案

### 修改 addMessage 方法

**文件**: `src/renderer/src/entities/message/model/useMessageStore.ts`

**修改前**:
```typescript
const addMessage = (sessionId: string, message: Message): void => {
  if (!messagesMap.value[sessionId]) {
    messagesMap.value[sessionId] = []  // ← 可能不触发响应式
  }
  messagesMap.value[sessionId].push(message)
}
```

**修改后**:
```typescript
const addMessage = (sessionId: string, message: Message): void => {
  if (!messagesMap.value[sessionId]) {
    // 使用 Vue 的响应式方式创建新数组
    messagesMap.value = {
      ...messagesMap.value,
      [sessionId]: []
    }
  }
  messagesMap.value[sessionId].push(message)
  console.log('✅ 消息已添加到 messagesMap:', {
    sessionId,
    messageId: message.id,
    totalMessages: messagesMap.value[sessionId].length
  })
}
```

**关键改进**:
- 使用对象展开运算符 `{ ...messagesMap.value, [sessionId]: [] }`
- 创建新对象，触发 Vue 的响应式更新
- 添加日志，方便调试

---

## 响应式原理

### Vue 3 响应式系统

Vue 3 使用 Proxy 来追踪对象的变化：

```typescript
// ✅ 正确：创建新对象，触发响应式
messagesMap.value = {
  ...messagesMap.value,
  [sessionId]: []
}

// ❌ 可能有问题：直接赋值属性
messagesMap.value[sessionId] = []
```

**为什么直接赋值可能有问题**:
- 虽然 Vue 3 的 Proxy 理论上能追踪属性添加
- 但在某些情况下（特别是嵌套对象），可能不会立即触发更新
- 使用对象展开更安全，确保触发响应式

## 其他可能的问题

### 1. 计算属性缓存

如果修改后仍然有问题，可能是计算属性缓存导致的。可以尝试强制刷新：

**文件**: `src/renderer/src/pages/main/MainPage.vue`

```typescript
const currentMessages = computed(() => {
  if (!activeSessionId.value) return []
  
  // 强制读取 messagesMap，确保响应式追踪
  const map = messageStore.messagesMap
  const messages = map[activeSessionId.value] || []
  
  console.log('📨 当前消息列表:', {
    sessionId: activeSessionId.value,
    messageCount: messages.length
  })
  
  return messages
})
```

---

### 2. 使用 watch 监听消息变化

如果计算属性仍然有问题，可以使用 `watch` 监听：

```typescript
import { watch } from 'vue'

watch(
  () => messageStore.messagesMap[activeSessionId.value],
  (newMessages) => {
    console.log('📨 消息列表变化:', newMessages?.length)
    // 触发滚动到底部
    nextTick(() => {
      scrollToBottom()
    })
  },
  { deep: true }
)
```

---

### 3. 使用 nextTick 确保 DOM 更新

在 `handleIncomingMessage` 中添加 `nextTick`：

**文件**: `src/renderer/src/shared/api/websocket.ts`

```typescript
import { nextTick } from 'vue'

private async handleIncomingMessage(message: any) {
  // ... 处理消息
  
  if (receivedMessage) {
    // 更新会话列表
    sessionStore.updateLastMessage(...)
    sessionStore.incrementUnread(...)
    
    // 如果当前正在查看该会话，确保 DOM 更新
    if (sessionStore.activeSessionId === message.sessionId) {
      await nextTick()
      console.log('💡 DOM 已更新，消息应该可见')
    }
  }
}
```
