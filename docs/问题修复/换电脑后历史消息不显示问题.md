# 换电脑后历史消息不显示问题

## 🔍 问题描述

用户反馈：换了台电脑后，之前在另一台电脑上发送的历史消息看不到了。

## 📊 问题分析

### 当前实现机制

1. **消息存储位置**：
   - 所有消息都存储在后端服务器（MySQL 数据库）
   - 前端不保存任何消息到本地存储

2. **消息加载流程**：
   ```
   用户登录 → 加载会话列表 → 点击会话 → 加载该会话的历史消息
   ```

3. **会话密钥管理**：
   - 每个会话有独立的 AES 加密密钥
   - 密钥存储在后端服务器
   - 前端通过 API 获取密钥后临时保存在内存中

### 可能的原因

#### ✅ 正常情况（不是问题）

如果你看到的是这样的界面：
- 左侧会话列表显示正常
- 但右侧聊天区域是空白的

**这是正常的！** 需要点击左侧的会话才会加载历史消息。

#### ❌ 异常情况（需要修复）

1. **点击会话后仍然没有消息**
   - 可能是会话密钥获取失败
   - 可能是消息解密失败
   - 可能是 API 请求失败

2. **会话列表为空**
   - Token 可能过期
   - 后端服务未启动
   - 网络连接问题

## 🔧 解决方案

### 方案 1: 正常使用流程（推荐）

1. **登录应用**
   - 使用相同的账号密码登录

2. **查看会话列表**
   - 左侧应该显示所有历史会话

3. **点击会话**
   - 点击任意会话
   - 系统会自动：
     - 获取会话密钥
     - 加载历史消息
     - 解密并显示

### 方案 2: 检查和调试

#### 1. 打开浏览器开发者工具

按 `F12` 或右键 → 检查

#### 2. 查看控制台日志

点击会话后，应该看到类似的日志：

```
📥 开始加载消息，会话ID: xxx
📨 获取到 10 条历史消息
🔓 解密消息: ...
✅ 解密成功: ...
✅ 消息加载完成，共 10 条
```

#### 3. 检查网络请求

切换到 Network 标签，查看：

- `GET /api/session/my` - 获取会话列表
- `GET /api/session/{sessionId}/messages` - 获取历史消息
- `GET /api/encryption/session/{sessionId}/key` - 获取会话密钥

#### 4. 常见错误和解决方法

**错误 1: 401 Unauthorized**
```
原因：Token 过期或无效
解决：重新登录
```

**错误 2: 解密失败**
```
原因：会话密钥不匹配
解决：检查后端密钥是否正确
```

**错误 3: 网络错误**
```
原因：后端服务未启动或网络不通
解决：检查后端服务状态
```

### 方案 3: 优化建议（开发者）

#### 自动加载最近会话的消息

修改 `MainPage.vue`，在加载会话列表后自动选择第一个会话：

```typescript
// 在 onMounted 中添加
onMounted(async () => {
  // ... 现有代码 ...
  
  // 加载会话列表
  await sessionStore.loadSessions()
  
  // 自动选择第一个会话
  const sessions = sessionStore.sessions
  if (sessions.length > 0) {
    await selectSession(sessions[0].id)
  }
})
```

#### 添加消息预加载

在会话列表中显示最后一条消息的预览：

```typescript
// 在 SessionList.vue 中

  const messages = await sessionApi.getMessages(sessionId, { limit: 1 })
```

## 💡 常见问题

### Q1: 为什么换电脑后需要重新登录？

**答**：Token 存储在浏览器的 localStorage 中，换电脑后需要重新登录获取新的 Token。

### Q2: 历史消息存储在哪里？

**答**：所有消息都存储在后端服务器的 MySQL 数据库中，不会因为换电脑而丢失。

### Q3: 为什么点击会话后才显示消息？

**答**：这是性能优化的设计，避免一次性加载所有会话的所有消息。只有在需要时才加载。

### Q4: 消息加密密钥会丢失吗？

**答**：不会。密钥存储在后端服务器，前端只是临时获取使用。

### Q5: 如何确认消息真的存在？

**答**：
1. 使用另一台电脑登录相同账号
2. 或者直接查询数据库
3. 或者使用 Swagger UI 测试 API
