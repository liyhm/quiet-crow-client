# 消息不显示问题调试指南

## 🔍 问题描述

重启应用后，左侧会话列表显示"共X条消息"，但右侧消息列表为空。

## 🎯 问题定位

### 情况 1: API 返回空数组

**日志特征**:
```
✅ [sessionApi.getMessages] API 响应成功
✅ [sessionApi.getMessages] 数据长度: 0
⚠️ [loadMessages] API 返回的消息列表为空！
```

**结论**: 后端问题，后端没有返回消息数据

**解决方法**:
1. 检查后端数据库是否有消息记录
2. 检查后端 API 是否正确查询
3. 检查后端是否有过滤逻辑（如水位线）

### 情况 2: API 请求失败

**日志特征**:
```
❌ [sessionApi.getMessages] API 请求失败
❌ [sessionApi.getMessages] 状态码: 401/500
```

**结论**: 后端问题，API 请求失败

**解决方法**:
1. 检查 Token 是否有效
2. 检查后端服务是否正常运行
3. 检查网络连接

### 情况 3: API 返回数据但解密失败

**日志特征**:
```
📨 [loadMessages] 返回消息数量: 3
🔓 [loadMessages] 解密消息 1/3: ...
❌ [loadMessages] 解密消息失败 1/3: ...
```

**结论**: 加密密钥问题

**解决方法**:
1. 检查会话密钥是否正确
2. 检查加密算法是否一致
3. 清除缓存重新获取密钥

### 情况 4: 消息加载成功但不显示

**日志特征**:
```
✅ [loadMessages] 消息加载完成，共 3 条
📋 [loadMessages] 最终消息列表: [...]
🔄 [currentMessages] computed 被触发
📊 [currentMessages] 获取到的消息数量: 0  ← 注意这里是 0
```

**结论**: 前端状态管理问题

**解决方法**:
1. 检查 messagesMap 是否正确更新
2. 检查 sessionId 是否匹配
3. 检查 Vue 响应式是否正常

### 情况 5: 消息存在但 computed 未触发

**日志特征**:
```
✅ [loadMessages] 消息加载完成，共 3 条
（没有看到 [currentMessages] 日志）
```

**结论**: Vue computed 未触发

**解决方法**:
1. 检查 activeSessionId 是否正确设置
2. 检查 Vue 响应式依赖
3. 强制刷新页面

## 💡 常见原因总结

### 后端问题（80%）

1. **数据库没有消息** - 消息被删除或未保存
2. **水位线过滤** - 后端根据水位线过滤了消息
3. **权限问题** - 用户没有权限查看消息
4. **API 逻辑错误** - 后端查询逻辑有问题

### 前端问题（20%）

1. **状态管理错误** - messagesMap 未正确更新
2. **sessionId 不匹配** - 活动会话 ID 与加载的不一致
3. **Vue 响应式失效** - computed 未触发
4. **缓存问题** - 旧数据缓存导致
