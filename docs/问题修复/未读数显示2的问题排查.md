# 未读数显示 2 的问题排查

## 🔍 问题描述

**场景**:
- lyh1 和 lyh2 刚加为好友
- lyh1 发送了 1 条消息："lyh2"
- lyh2 那边显示未读数是 2（而不是 1）

## 📊 可能的原因

### 原因 1: 好友请求接受时产生了系统消息

**假设流程**:
1. lyh2 接受好友请求
2. 后端创建私聊会话
3. 后端可能自动发送了一条系统消息（如"你们已成为好友"）
4. lyh1 发送消息："lyh2"
5. 所以 lyh2 实际收到了 2 条消息

**排查方法**:
```javascript
// 在浏览器控制台执行
const messageStore = useMessageStore()
const sessionStore = useSessionStore()

// 查看当前会话的所有消息
console.log('当前会话消息:', messageStore.getMessages(sessionStore.activeSessionId))
```

### 原因 2: 后端未读数计算错误

**问题**: 后端在计算未读数时，可能没有正确排除用户自己发送的消息

**后端应该使用的 SQL**:
```sql
-- 正确的未读数计算
SELECT COUNT(*) as unread_count
FROM message
WHERE session_id = ?
AND id > (SELECT watermark FROM session_user WHERE session_id = ? AND user_id = ?)
AND send_user != ?  -- 关键：排除自己发送的消息
```

**错误的 SQL**:
```sql
-- 错误：没有排除自己的消息
SELECT COUNT(*) as unread_count
FROM message
WHERE session_id = ?
AND id > (SELECT watermark FROM session_user WHERE session_id = ? AND user_id = ?)
-- 缺少：AND send_user != ?
```

### 原因 3: 水位线初始化错误

**问题**: 创建会话时，水位线可能被初始化为错误的值

**正确的初始化**:
```java
// 创建会话时
SessionUser sessionUser = new SessionUser();
sessionUser.setSessionId(sessionId);
sessionUser.setUserId(userId);
sessionUser.setWatermark(0L);  // 初始化为 0
```

**错误的初始化**:
```java
// 如果初始化为 null 或负数，可能导致计算错误
sessionUser.setWatermark(null);  // 错误
sessionUser.setWatermark(-1L);   // 错误
```

### 原因 4: 前端重复增加未读数

**问题**: 前端在接收消息时，可能重复调用了 `incrementUnread`

**检查代码**:
```typescript
// src/renderer/src/shared/api/websocket.ts
private async handleIncomingMessage(message: any) {
  // ...
  
  // 4. 增加未读计数（如果不是当前活动会话）
  sessionStore.incrementUnread(message.sessionId)  // 这里可能被调用了多次？
}
```

**排查方法**:
在 `incrementUnread` 方法中添加日志：
```typescript
const incrementUnread = (sessionId: string) => {
  console.log('🔔 [incrementUnread] 被调用')
  console.log('🔔 [incrementUnread] sessionId:', sessionId)
  console.log('🔔 [incrementUnread] 调用堆栈:', new Error().stack)
  
  const session = sessions.value.find((s) => s.id === sessionId)

    console.log('📊 [incrementUnread] 当前未读数:', session.unreadCount)
    session.unreadCount++
    console.log('📊 [incrementUnread] 新的未读数:', session.unreadCount)
  }
}
```

### 方案 1: 前端重新计算未读数

// src/renderer/src/entities/session/model/useSessionStore.ts

const loadSessions = async () => {
  try {

## 🎯 最终解决方案

**需要后端修复**:

1. **修复未读数计算逻辑**:
   ```java
   public int calculateUnreadCount(String sessionId, String userId) {
       SessionUser sessionUser = sessionUserRepository
           .findBySessionIdAndUserId(sessionId, userId)
           .orElse(null);
       
       if (sessionUser == null) {
           return 0;
       }
       
       // 关键：排除自己发送的消息
       return messageRepository.countBySessionIdAndIdGreaterThanAndSendUserNot(
           sessionId,
           sessionUser.getWatermark(),
           userId  // 这是关键！
       );
   }
   ```

2. **发送消息时立即更新发送者水位线**:
   ```java
   public void sendMessage(String sessionId, String senderId, String content) {
       // 保存消息
       Message message = messageRepository.save(newMessage);
       
       // 立即更新发送者的水位线
       SessionUser senderSessionUser = sessionUserRepository
           .findBySessionIdAndUserId(sessionId, senderId)
           .orElseThrow();
       
       senderSessionUser.setWatermark(message.getId());
       sessionUserRepository.save(senderSessionUser);
   }
   ```

3. **不要在接受好友请求时发送系统消息**:
   - 或者如果必须发送，确保系统消息不计入未读数
