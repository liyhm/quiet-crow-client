## 说明

这些错误主要是 TypeScript 严格类型检查导致的：

1. **未使用的变量/参数**: 使用下划线前缀 `_` 或直接删除
2. **unknown 类型**: 使用类型断言 `as Error` 明确类型
3. **属性名错误**: 使用正确的属性名
4. **null vs 空字符串**: 根据函数签名使用正确的类型

这些修复不会影响应用的功能，只是让代码符合 TypeScript 的严格类型检查规则。
