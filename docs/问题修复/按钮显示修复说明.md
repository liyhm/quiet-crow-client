# 按钮显示修复说明

## 问题描述

之前所有按钮只有在鼠标悬停时才显示背景色，导致用户不知道哪里可以点击。

## 修复内容

### 1. 窗口控制按钮 ✅

**位置**: 标题栏右侧

**修复前**:
```css
class="hover:bg-gray-300 p-1 rounded"
```

**修复后**:
```css
class="bg-gray-200 hover:bg-gray-300 p-1 rounded"  /* 最小化、最大化 */
class="bg-red-500 hover:bg-red-600 text-white p-1 rounded"  /* 关闭 */
```

### 2. 侧边栏设置按钮 ✅

**位置**: 左侧边栏底部

**修复前**:
```css
class="hover:bg-gray-700 text-gray-400 hover:text-white"
```

**修复后**:
```css
class="bg-gray-700 hover:bg-gray-600 text-gray-300 hover:text-white"
```

### 3. 添加好友按钮 ✅

**位置**: 联系人页面右上角

**修复前**:
```css
class="hover:bg-gray-200"
```

**修复后**:
```css
class="bg-gray-200 hover:bg-gray-300"
```

### 4. 聊天输入区域按钮 ✅

**位置**: 聊天输入框上方（表情、文件上传）

**修复前**:
```css
class="hover:bg-gray-100 p-2 rounded"
```

**修复后**:
```css
class="bg-gray-100 hover:bg-gray-200 p-2 rounded"
```

### 5. 聊天头部菜单按钮 ✅

**位置**: 聊天窗口右上角

**修复前**:
```css
class="hover:bg-gray-100 p-2 rounded"
```

**修复后**:
```css
class="bg-gray-100 hover:bg-gray-200 p-2 rounded"
```

## 修复原则

1. **始终可见**: 所有按钮都有默认背景色，用户一眼就能看到
2. **悬停反馈**: 鼠标悬停时背景色加深，提供交互反馈
3. **颜色区分**: 
   - 普通按钮: 灰色系 (`bg-gray-100`, `bg-gray-200`)
   - 危险操作: 红色系 (`bg-red-500`)
   - 侧边栏: 深色系 (`bg-gray-700`)

## 保持不变的按钮

以下按钮保持原样，因为它们本身就有明显的背景色：

1. **主要操作按钮**: 
   - 登录/注册按钮 (`bg-wechat-green`)
   - 发送消息按钮 (`bg-wechat-green`)
   - 接受好友请求 (`bg-wechat-green`)

2. **次要操作按钮**:
   - 取消按钮 (`bg-gray-200`)
   - 拒绝按钮 (`bg-gray-200`)

3. **列表项**:
   - 会话列表项 (hover 时显示背景)
   - 联系人列表项 (hover 时显示背景)
   - 这些保持 hover 效果是合理的
