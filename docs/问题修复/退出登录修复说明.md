# 退出登录功能修复说明

## 问题描述

点击退出登录后，个人信息弹窗显示异常：
1. 头像背景色不显示（绿色背景消失）
2. 控制台出现 Pinia store 错误

## 问题原因

### 1. 绿色背景不显示
使用 Tailwind 类名 `bg-wechat-green` 可能未正确编译或被缓存

### 2. Store 错误
调用了 `sessionStore.$reset()` 和 `messageStore.$reset()`，但这些 store 可能没有正确实现 reset 方法

### 2. 优化退出登录逻辑

**修复前**:
```typescript
const handleLogout = (): void => {
  console.log('👋 退出登录...')
  
  // 清理 WebSocket 连接
  websocketService.disconnect()
  
  // 清理 Store 数据
  authStore.logout()
  sessionStore.$reset()  // ❌ 可能导致错误
  messageStore.$reset()  // ❌ 可能导致错误
  
  // 跳转到登录页
  router.push('/login')
  
  console.log('✅ 已退出登录')
}
```

**修复后**:
```typescript
const handleLogout = (): void => {
  console.log('👋 退出登录...')

  // 清理 WebSocket 连接
  try {
    websocketService.disconnect()
  } catch (error) {
    console.warn('WebSocket 断开失败:', error)
  }

  // 清理认证信息
  authStore.logout()

  // 关闭弹窗
  showProfileModal.value = false
  showSettingsModal.value = false

  // 跳转到登录页
  router.push('/login')

  console.log('✅ 已退出登录')
}
```

## 改进点

### 1. 移除 Store Reset
- 不再调用 `sessionStore.$reset()` 和 `messageStore.$reset()`
- 只清理认证信息 `authStore.logout()`
- 路由守卫会自动处理未登录状态

### 2. 添加错误处理
- WebSocket 断开使用 try-catch 包裹
- 避免断开失败影响退出流程

### 3. 关闭弹窗
- 退出前关闭所有打开的弹窗
- 避免跳转后弹窗残留

### 4. 用户名显示优化
- 在用户名前添加 `@` 符号
- 更符合社交应用习惯

## 退出登录流程

1. 用户点击"退出登录"按钮
2. 尝试断开 WebSocket 连接
3. 清除 localStorage 中的 token
4. 清空 authStore 的用户信息
5. 关闭所有打开的弹窗
6. 跳转到登录页面
7. 路由守卫检测到未登录状态
8. 保持在登录页面

## 为什么不需要手动清理其他 Store

1. **路由守卫保护**: 
   - 未登录用户无法访问主页面
   - 重新登录时会重新加载数据

2. **数据隔离**:
   - 每次登录都会重新获取会话和消息
   - 旧数据不会影响新用户

3. **简化逻辑**:
   - 减少出错可能
   - 提高代码可维护性
