# 前端问题总结和修复计划

## 当前问题

### 🔴 问题 1：接收端收不到实时消息
**现象**：
- 发送方发送消息成功
- 接收方控制台没有任何日志
- 接收方界面没有更新

**原因**：
- 后端没有转发消息
- 后端的消息处理器没有实现或有问题

**解决方案**：
- 后端需要实现消息转发功能
- 参考文档：`后端需求-实时消息转发功能.md`

### 🔴 问题 2：所有消息都显示在左边
**现象**：
- 自己发送的消息也显示在左边
- 无法区分是谁发送的

**可能原因**：
1. `message.senderId` 和 `currentUserId` 不匹配
2. 数据库中的 `send_user` 字段有问题
3. 前端比较逻辑有问题

**已添加调试**：
- 在控制台打印详细的消息信息
- 包括 `senderId`、`currentUserId`、`isMine` 等

**下一步**：
- 查看控制台日志
- 确认 `senderId` 和 `currentUserId` 的值
- 如果不匹配，需要检查后端返回的数据

### 🔴 问题 3：最新消息在最上面
**现象**：
- 消息列表倒序显示
- 最新消息在顶部，需要向上滚动查看

**已修复**：
- 消息按时间升序排序（最旧的在上面，最新的在下面）
- 符合聊天工具的标准设计

### 🔴 问题 4：消息内容不显示
**现象**：
- 消息气泡中只显示时间
- 看不到消息内容

**可能原因**：
1. 消息解密失败
2. `message.content` 是空字符串
3. 后端返回的数据有问题

**已添加调试**：
- 打印解密过程
- 打印消息内容
- 即使解密失败也显示 "[解密失败]"

## 调试步骤

### 步骤 1：检查消息加载

刷新页面，点击会话，查看控制台：

```
📥 开始加载消息，会话ID: xxx
📨 获取到 X 条历史消息
🔓 解密消息: { id: xxx, sendUser: xxx, encrypted: xxx }
✅ 解密成功: 你好
✅ 消息加载完成，共 X 条
📋 消息列表（按时间升序）: [...]
📨 当前消息列表: {
  currentUserId: "xxx",
  messages: [
    { senderId: "xxx", isMine: true/false, content: "xxx" }
  ]
}
```

**关键检查点**：
1. `senderId` 和 `currentUserId` 是否不同
2. `isMine` 是否正确（自己发的是 true，对方发的是 false）
3. `content` 是否有内容

### 步骤 2：检查消息发送

发送消息后，查看控制台：

```
📤 发送消息到 WebSocket: {
  destination: "/app/chat/xxx",
  sessionId: "xxx",
  encrypted: "xxx..."
}
✅ 消息已发送
```

### 步骤 3：检查消息接收

对方应该看到：

```
📬 收到私有消息: {...}
📨 收到 WebSocket 消息: {...}
📩 处理接收到的消息: {...}
🔓 消息解密成功: xxx
✅ 消息已添加到 store
✅ 消息已处理，会话列表已更新
```

**如果没有这些日志**：
- 后端没有转发消息
- 需要检查后端代码

## 聊天界面设计规范

### 1. 消息排序
- ✅ 最旧的消息在上面
- ✅ 最新的消息在下面
- ✅ 发送新消息后自动滚动到底部

### 2. 消息布局
- ✅ 自己发送的消息：右对齐，绿色气泡
- ✅ 对方发送的消息：左对齐，白色气泡
- ✅ 头像显示：对方在左边，自己在右边

### 3. 消息内容
- ✅ 显示消息文本
- ✅ 显示发送时间（相对时间：刚刚、X分钟前、昨天等）
- ✅ 显示发送者标识（头像或昵称首字母）

### 4. 实时更新
- ❌ 收到新消息立即显示（需要后端支持）
- ❌ 会话列表显示最后一条消息
- ❌ 会话列表显示未读数

### 5. 交互体验
- ✅ 点击会话加载历史消息
- ✅ 输入框支持 Enter 发送，Shift+Enter 换行
- ✅ 发送按钮在有内容时高亮
- ❌ 滚动到顶部加载更多历史消息（待实现）
