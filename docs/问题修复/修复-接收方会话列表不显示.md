# 修复：接收方会话列表不显示

## 问题描述

✅ 消息已经能实时接收（控制台有日志）  
❌ 但接收方的会话列表中没有显示这个会话

### 控制台日志

```
📬 收到 WebSocket 消息
📩 处理接收到的消息
🔓 消息解密成功
✅ 消息已添加到 store
📊 消息数量更新: 222
```

---

## 根本原因

1. 接收方的会话列表中没有这个会话
2. `updateLastMessage()` 和 `incrementUnread()` 只更新已存在的会话
3. 如果会话不存在，就不会显示

### 为什么会话不存在？

**场景**：
- 用户 A 和用户 B 是好友
- 用户 A 先登录，创建了与用户 B 的会话
- 用户 B 后登录，此时会话列表中没有这个会话
- 用户 A 发送消息，用户 B 收到消息但看不到会话

---

## 解决方案

### 修改内容

**文件**：`src/renderer/src/shared/api/websocket.ts`

**修改**：在 `handleIncomingMessage` 方法中添加会话检查

**逻辑**：
1. 收到消息时，检查会话是否存在
2. 如果不存在，重新加载会话列表
3. 然后再更新消息和未读数

### 修改后的代码逻辑

```typescript
private async handleIncomingMessage(message: any) {
  // 1. 检查会话是否存在
  const sessionExists = sessionStore.sessions.find(s => s.id === message.sessionId)
  
  // 2. 如果不存在，重新加载会话列表
  if (!sessionExists) {
    await sessionStore.loadSessions()
  }
  
  // 3. 接收并解密消息（等待 Promise 完成）
  const receivedMessage = await messageStore.receiveMessage(message)
  
  // 4. 更新会话列表
  if (receivedMessage) {
    sessionStore.updateLastMessage(...)
    sessionStore.incrementUnread(...)
  }
}
```

### 关键改进

1. **添加会话检查**：`sessionExists` 检查会话是否在列表中
2. **重新加载会话列表**：如果不存在，调用 `loadSessions()`
3. **等待 Promise**：使用 `await` 等待消息解密完成
4. **使用解密后的内容**：`receivedMessage.content` 是解密后的文本

## 性能考虑

### 会话列表重新加载的开销

**场景**：接收到新会话的第一条消息时

**开销**：
- 一次 API 请求：`GET /api/session/my`
- 响应时间：通常 < 100ms
- 频率：每个新会话只加载一次

**优化**：
- 只在会话不存在时加载
- 已存在的会话不会重复加载
- 对用户体验影响很小

## 常见问题

### Q1: 为什么不在创建会话时就同步？

**原因**：
- 用户 A 创建会话时，用户 B 可能不在线
- 用户 B 登录时，会话还没有消息，不会显示在列表中
- 只有收到第一条消息时，才会显示

### Q2: 会不会频繁重新加载会话列表？

**不会**：
- 只在会话不存在时加载
- 加载后会话就存在了，不会再次加载
- 每个新会话只加载一次

### Q3: 如果加载失败怎么办？

**处理**：
- 消息仍然会保存到 store
- 用户刷新页面后会看到
- 不会丢失消息
