# 组件拆分后的问题修复

## 问题 1: 语法错误 - Unexpected token

### 错误信息
```
Error parsing JavaScript expression: Unexpected token, expected "," (3:8)
MainPage.vue:75:21
```

### 原因
在 SettingsModal 组件的 `@showProfile` 事件处理中，使用了多行代码但没有正确的语法：

```vue
<!-- ❌ 错误写法 -->
<SettingsModal
  @showProfile="
    showProfileModal = true
    showSettingsModal = false
  "
/>
```

Vue 模板中的事件处理器如果是多行代码，需要：
1. 使用分号分隔
2. 或者改成函数调用

### 解决方案

创建一个专门的处理函数：

```vue
<!-- ✅ 正确写法 -->
<SettingsModal
  @showProfile="handleShowProfileFromSettings"
/>
```

```typescript
const handleShowProfileFromSettings = (): void => {
  showProfileModal.value = true
  showSettingsModal.value = false
}
```

### 修复位置
- **文件**: `src/renderer/src/pages/main/MainPage.vue`
- **行数**: 75-78

## 问题 2: 登录验证很久

### 现象
用户登录后，页面需要等待很长时间才能进入主页面。

### 原因分析

1. **路由守卫重复验证**: 每次路由导航都会调用 `checkAuth()`
2. **没有缓存**: 即使已经有用户信息，仍然会重新请求 API
3. **没有超时处理**: 如果后端响应慢，会一直等待
4. **网络请求阻塞**: `userApi.getInfo()` 阻塞了路由导航

### 解决方案

#### 1. 添加用户信息缓存

```typescript
const checkAuth = async () => {
  const savedToken = localStorage.getItem(API_CONFIG.TOKEN_KEY)
  if (savedToken) {
    token.value = savedToken
    
    // ✅ 如果已经有用户信息，直接使用
    if (currentUser.value) {
      console.log('✅ 使用缓存的用户信息')
      isInitialized.value = true
      return
    }
    
    // ... 继续验证
  }
}
```

#### 2. 添加请求超时

```typescript
// ✅ 添加 5 秒超时
const userInfo = await Promise.race([
  userApi.getInfo(),
  new Promise((_, reject) => 
    setTimeout(() => reject(new Error('请求超时')), 5000)
  )
]) as any
```

#### 3. 优化路由守卫

```typescript
// 只在未初始化时才调用 checkAuth
if (!authStore.isInitialized) {
  await authStore.checkAuth()
}
```

### 修复位置
- **文件**: `src/renderer/src/entities/user/model/useAuthStore.ts`
- **函数**: `checkAuth()`

### 效果
- ✅ 首次登录：正常验证（最多 5 秒超时）
- ✅ 已登录状态：立即使用缓存，无需等待
- ✅ 路由切换：不会重复验证

## 问题 3: 登录进去后报错

### 可能的原因

根据截图中的错误信息，可能是以下几个原因：

1. **组件导入失败**: 新拆分的组件路径不正确
2. **Props 类型不匹配**: 传递给子组件的 props 类型错误
3. **缺少必需的 props**: 子组件需要的 props 没有传递
4. **API 请求失败**: 后端接口返回错误

### 调试步骤

#### 1. 检查浏览器控制台

打开开发者工具（F12），查看 Console 标签页的错误信息：
- 红色错误：组件加载失败、API 错误
- 黄色警告：Props 验证失败、性能警告

#### 2. 检查 Network 标签

查看 API 请求是否成功：
- 状态码 200：成功
- 状态码 401：未授权（token 失效）
- 状态码 500：服务器错误

#### 3. 检查组件导入

确认所有新组件都正确导入：

### 常见错误及解决方案

#### 错误 1: Cannot read property 'xxx' of undefined

**原因**: 某个对象为 undefined 或 null

**解决方案**: 添加可选链和默认值

```typescript
// ❌ 错误
const name = activeSession.name

// ✅ 正确
const name = activeSession?.name || '未知会话'
```

#### 错误 2: Component is missing template or render function

**原因**: 组件文件不存在或路径错误

**解决方案**: 检查文件路径和文件是否存在

```bash
# 检查文件是否存在
ls src/renderer/src/widgets/titleBar/TitleBar.vue
```

#### 错误 3: Invalid prop type

**原因**: 传递的 prop 类型与定义不匹配

**解决方案**: 检查 prop 类型定义

```typescript
// 组件定义
defineProps<{
  messages: Message[]  // 期望数组
}>()

// 使用时确保传递数组
<MessageList :messages="currentMessages || []" />
```
