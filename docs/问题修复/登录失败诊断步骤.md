# 登录失败诊断步骤

## 当前症状

从截图分析：
- 请求状态：红色 X（失败）
- 请求 URL：`http://159.75.182.85:8081/api/auth/login`
- 传输大小：`0 B transferred`
- 时间：显示但请求未完成

## 诊断步骤

### 步骤 1: 检查是否使用了新打包的版本

**问题**: 你可能还在使用修改 CSP 配置之前打包的旧版本。

**检查方法**:
1. 查看你当前安装的应用是什么时候打包的
2. 确认是否在修改 CSP 配置后重新打包了

**解决方案**:
```powershell
# 1. 卸载旧版本
# 通过 Windows 设置 → 应用 → 卸载 quiet

# 2. 清理构建目录
cd D:\workSpace\workIdea\lyh\quiet-crow-client
Remove-Item -Recurse -Force dist, out

# 3. 重新打包
npm run build:win

# 4. 安装新版本
# 运行 dist\quiet-1.0.0-setup.exe
```

### 步骤 2: 查看 Console 日志

**操作**: 按 3 次 Ctrl 键打开开发者工具，切换到 Console 标签

**查找以下信息**:

#### 2.1 查看 API 配置
应该看到：
```
🔧 API 配置: {
  BASE_URL: "http://159.75.182.85:8081",
  WS_URL: "http://159.75.182.85:8081/ws",
  ENV: "production",
  IS_DEV: false,
  IS_PROD: true
}
```

**如果看到的是**:
```
BASE_URL: "http://localhost:8080"
```
说明使用的是开发环境配置，不是生产环境。

#### 2.2 查看 CSP 错误
如果有 CSP 错误，会看到：
```
Refused to connect to 'http://159.75.182.85:8081/api/auth/login' 
because it violates the following Content Security Policy directive: 
"connect-src 'self' https://api.your-domain.com"
```

**如果看到这个错误**，说明：
- 使用的是旧版本的应用
- CSP 配置没有更新
- 需要重新打包

#### 2.3 查看网络错误
可能的错误信息：
```
❌ 登录失败: Network Error
❌ 登录失败: Request failed with status code 0
❌ 登录失败: timeout of 30000ms exceeded
```

### 步骤 3: 检查后端服务

**操作**: 在浏览器中访问后端 API

```
http://159.75.182.85:8081/swagger-ui/index.html
```

**预期结果**: 应该能看到 Swagger 文档页面

**如果无法访问**:
- 后端服务可能没有启动
- 防火墙可能阻止了访问
- IP 地址或端口可能不正确

### 步骤 4: 测试网络连接

**操作**: 在开发者工具的 Console 中执行

```javascript
fetch('http://159.75.182.85:8081/api/auth/login', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    username: 'test',
    password: 'test'
  })
})
.then(res => res.json())
.then(data => console.log('✅ 连接成功:', data))
.catch(err => console.error('❌ 连接失败:', err))
```

**预期结果**:
- 如果后端正常，应该返回错误信息（用户名或密码错误）
- 如果 CSP 阻止，会看到 CSP 错误
- 如果网络问题，会看到 Network Error

### 步骤 5: 检查 Network 标签详细信息

**操作**: 在 Network 标签中点击失败的 `login` 请求

**查看以下信息**:

#### 5.1 General 部分
- Request URL: 应该是 `http://159.75.182.85:8081/api/auth/login`
- Request Method: 应该是 `POST`
- Status Code: 如果是 `(blocked:csp)` 说明被 CSP 阻止

#### 5.2 Request Headers 部分
- Content-Type: 应该是 `application/json`
- 如果看到 "Provisional headers are shown"，说明请求被阻止了

#### 5.3 Response Headers 部分
- 如果是空的，说明请求没有发送出去
- 如果有内容，查看是否有 CORS 相关的错误

### 步骤 6: 对比开发环境和生产环境

**开发环境测试**:
```powershell
# 启动开发环境
npm run dev
```

在开发环境中测试登录：
- 如果开发环境可以登录，说明代码逻辑没问题
- 如果开发环境也不能登录，说明是代码或后端问题

## 常见问题和解决方案

### 问题 1: 使用了旧版本的应用

**症状**:
- Console 显示 CSP 错误
- 错误信息中包含 `https://api.your-domain.com`

**解决方案**:
1. 卸载旧版本
2. 重新打包
3. 安装新版本

### 问题 2: 后端服务未启动

**症状**:
- Network Error
- 无法访问 Swagger 文档

**解决方案**:
1. 启动后端服务
2. 确认服务运行在 8081 端口
3. 检查防火墙设置

### 问题 3: 网络连接问题

**症状**:
- timeout 错误
- 连接超时

**解决方案**:
1. 检查网络连接
2. 检查防火墙
3. 尝试 ping 后端服务器

### 问题 4: CORS 配置问题

**症状**:
- CORS policy 错误
- Access-Control-Allow-Origin 错误

**解决方案**:
1. 检查后端 CORS 配置
2. 确认后端允许来自 Electron 应用的请求

## 完整的重新打包流程

如果以上步骤都检查过了，建议完整重新打包：

