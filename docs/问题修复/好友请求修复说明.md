# 好友请求提醒功能修复说明

## 问题描述

用户反馈的两个问题：
1. 如果有新朋友加好友邀请，没有提示，需要点击到联系人列表才可以看到
2. 当别人同意好友请求后，联系人图标上的红色数字没有自动刷新
3. 当别人同意好友请求后，联系人列表没有自动刷新，需要手动刷新才能看到新好友

## 解决方案

### 第一阶段：基础提醒功能（已完成）

在侧边栏的联系人图标上显示未读的好友请求数量（红色圆圈），类似消息未读数的显示方式。

### 第二阶段：实时推送功能（需要后端配合）

通过 WebSocket 实时推送好友请求相关的通知，实现自动刷新。

## 实现细节

### 1. 前端修改

#### Sidebar.vue
- 导入 `useContactStore`
- 添加 `pendingRequestCount` 计算属性，统计待处理的好友请求数量
- 在联系人按钮上添加红色圆圈显示未读数

```vue
<script setup lang="ts">
import { useContactStore } from '@/entities/contact/model/useContactStore'

const contactStore = useContactStore()

const pendingRequestCount = computed(() => contactStore.pendingRequests.length)
</script>

<template>
  <!-- 联系人按钮 -->
  <button>
    <!-- 好友请求未读数 -->
    <span
      v-if="pendingRequestCount > 0"
      class="absolute -top-1 -right-1 w-5 h-5 rounded-full bg-red-500 text-white text-xs flex items-center justify-center"
    >
      {{ pendingRequestCount > 99 ? '99+' : pendingRequestCount }}
    </span>
  </button>
</template>
```

#### MainPage.vue
- 在 `onMounted` 钩子中添加加载好友请求的逻辑
- 确保应用启动时自动加载好友请求列表

```typescript
onMounted(async () => {
  // 加载会话列表
  await sessionStore.loadSessions()
  
  // 加载好友请求
  await contactStore.loadPendingRequests()
  console.log('✅ 好友请求加载完成，数量:', contactStore.pendingRequests.length)
})
```

### 2. 数据流

1. 应用启动时，`MainPage.vue` 的 `onMounted` 调用 `contactStore.loadPendingRequests()`
2. `contactStore` 从后端 API 获取待处理的好友请求列表
3. `Sidebar.vue` 通过 `pendingRequestCount` 计算属性实时显示未读数
4. 用户接受或拒绝好友请求后，`contactStore` 会重新加载列表，未读数自动更新

### 3. 后端 API

使用现有的好友请求 API：

```
GET /api/contacts/requests/pending
```

返回待处理的好友请求列表。

## 测试步骤

### 基础功能测试

1. 启动应用，登录账号 A
2. 使用另一个账号 B 向账号 A 发送好友请求
3. 账号 A 的侧边栏联系人图标应显示红色圆圈，数字为 1
4. 点击联系人图标，查看好友请求列表
5. 接受或拒绝请求后，红色圆圈应消失（如果没有其他待处理请求）

### 实时推送测试（需要后端实现 WebSocket 推送）

#### 测试场景 1：收到新的好友请求
1. 用户 A 登录应用
2. 用户 B 向用户 A 发送好友请求
3. 验证：用户 A 的侧边栏联系人图标上应立即显示红色数字（无需刷新）

#### 测试场景 2：好友请求被接受
1. 用户 A 向用户 B 发送好友请求
2. 用户 B 接受好友请求
3. 验证：
   - 用户 A 的侧边栏联系人图标上的红色数字应立即消失
   - 用户 A 点击联系人图标，应立即看到用户 B 出现在联系人列表中（无需刷新）

## 注意事项

1. 未读数显示规则：
   - 0 个请求：不显示红色圆圈
   - 1-99 个请求：显示具体数字
   - 100+ 个请求：显示 "99+"

2. 自动刷新：
   - 应用启动时自动加载
   - 接受/拒绝请求后自动刷新
   - 暂不支持 WebSocket 实时推送（可后续添加）

3. 性能考虑：
   - 使用 computed 属性，自动缓存计算结果
   - 只在必要时重新加载列表
