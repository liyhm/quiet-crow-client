# 未读数显示错误 - 前端修复说明

## 🔍 问题描述

用户反馈：每次收到第一条消息时，未读数显示为 2，但实际只有 1 条消息。

## 📊 问题分析

### 问题根源

前端在接收 WebSocket 消息时，没有检查消息是否已经存在，导致：

1. **后端返回未读数**: 后端在会话列表接口中返回 `unreadCount: 1`
2. **WebSocket 推送消息**: 前端收到 WebSocket 推送的新消息
3. **前端增加未读数**: `incrementUnread()` 方法执行 `unreadCount++`
4. **结果**: 显示 `unreadCount: 2`（1 + 1 = 2）

### 代码流程

```
1. 用户 A 发送消息
   ↓
2. 后端保存消息，更新会话列表
   ↓
3. 后端通过 WebSocket 推送给用户 B
   ↓
4. 用户 B 的前端收到消息
   ↓
5. 前端调用 receiveMessage() 添加消息
   ↓
6. 前端调用 incrementUnread() 增加未读数
   ↓
7. 用户 B 刷新页面
   ↓
8. 前端加载会话列表，后端返回 unreadCount: 1
   ↓
9. 前端显示 unreadCount: 1
   ↓
10. 用户 A 再发一条消息
   ↓
11. 前端收到 WebSocket 推送
   ↓
12. 前端 incrementUnread()，unreadCount 变成 2
   ↓
13. 但实际只有 1 条新消息！
```

## ✅ 修复方案

### 修复 1: 消息去重

在 `receiveMessage` 方法中添加去重逻辑：

**文件**: `src/renderer/src/entities/message/model/useMessageStore.ts`

**修改前**:
```typescript
const receiveMessage = async (encryptedMessage) => {
  // 直接添加消息，没有检查是否已存在
  const message = { ... }
  addMessage(message.sessionId, message)
  return message
}
```

**修改后**:
```typescript
const receiveMessage = async (encryptedMessage) => {
  // 检查消息是否已存在（去重）
  const messageId = encryptedMessage.id || `msg-${Date.now()}`
  const existingMessages = messagesMap.value[encryptedMessage.sessionId] || []
  const isDuplicate = existingMessages.some(m => m.id === messageId)
  
  if (isDuplicate) {
    console.warn('⚠️ 消息已存在，跳过重复添加:', messageId)
    return null  // 返回 null，不增加未读数
  }
  
  // 添加新消息
  const message = { ... }
  addMessage(message.sessionId, message)
  return message
}
```

### 修复 2: 增加日志

在 `incrementUnread` 方法中添加详细日志：

**文件**: `src/renderer/src/entities/session/model/useSessionStore.ts`

```typescript
const incrementUnread = (sessionId: string) => {
  const session = sessions.value.find((s) => s.id === sessionId)
  if (session && session.id !== activeSessionId.value) {
    console.log('📊 [incrementUnread] 增加未读数')
    console.log('📊 [incrementUnread] sessionId:', sessionId)
    console.log('📊 [incrementUnread] 当前未读数:', session.unreadCount)
    
    session.unreadCount++
    
    console.log('📊 [incrementUnread] 新的未读数:', session.unreadCount)
  }
}
```

## ⚠️ 注意事项

### 1. 消息 ID 的重要性

消息去重依赖于消息 ID，确保：
- 后端返回的消息必须有唯一的 `id` 字段
- WebSocket 推送的消息也必须包含相同的 `id`

### 2. 时间戳问题

如果消息没有 ID，会使用时间戳生成：
```typescript
const messageId = encryptedMessage.id || `msg-${Date.now()}`
```

这可能导致同一条消息在不同时间生成不同的 ID，无法去重。

**建议**: 后端必须返回消息 ID

### 3. 并发问题

如果同时收到多条消息，可能会有竞态条件。但由于 JavaScript 是单线程的，这个问题不大。
