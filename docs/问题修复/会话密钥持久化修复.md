# 会话密钥持久化修复

## 📋 问题描述

**现象**：
- 昨天发送的消息正常显示
- 今天重新登录后，昨天的消息全部显示"加载失败"
- 新发送的消息可以正常显示

**影响**：
- 用户无法查看历史消息
- 每次重新登录都会丢失所有会话密钥
- 严重影响用户体验

---

## 🔍 问题分析

### 根本原因

会话密钥存储在**内存**中（`Map`），而不是持久化存储：

```typescript
// ❌ 之前的实现
export class BackendCryptoUtil {
  private sessionKeys: Map<string, string> = new Map()
  // ...
}
```

### 问题流程

1. **第一天**：
   - 用户登录
   - 发送消息时，生成会话密钥并存储在内存中
   - 消息加密后发送到后端
   - 接收消息时，使用内存中的密钥解密
   - ✅ 一切正常

2. **第二天**：
   - 用户重新登录（或刷新页面）
   - **内存被清空，所有会话密钥丢失**
   - 尝试加载历史消息
   - 调用 `ensureSessionKey` 从后端获取密钥
   - ❌ 但是后端返回的密钥可能不是原来的密钥
   - ❌ 或者后端没有保存密钥
   - 解密失败，显示"加载失败"

### 为什么新消息可以正常显示？

- 新消息使用的是**当前会话的密钥**
- 这个密钥在内存中，可以正常加密和解密
- 但是关闭应用后，这个密钥也会丢失

---

## ✅ 解决方案

### 1. 持久化会话密钥到 localStorage

```typescript
const SESSION_KEYS_STORAGE_KEY = 'chat_session_keys'

export class BackendCryptoUtil {
  private sessionKeys: Map<string, string> = new Map()

  constructor() {
    // 从 localStorage 恢复会话密钥
    this.loadKeysFromStorage()
  }

  // 从 localStorage 加载密钥
  private loadKeysFromStorage() {
    try {
      const stored = localStorage.getItem(SESSION_KEYS_STORAGE_KEY)
      if (stored) {
        const keys = JSON.parse(stored)
        this.sessionKeys = new Map(Object.entries(keys))
        console.log('✅ 从 localStorage 恢复了', this.sessionKeys.size, '个会话密钥')
      }
    } catch (error) {
      console.error('❌ 加载会话密钥失败:', error)
    }
  }

  // 保存密钥到 localStorage
  private saveKeysToStorage() {
    try {
      const keys = Object.fromEntries(this.sessionKeys)
      localStorage.setItem(SESSION_KEYS_STORAGE_KEY, JSON.stringify(keys))
    } catch (error) {
      console.error('❌ 保存会话密钥失败:', error)
    }
  }

  // 设置会话密钥（Base64 格式）
  setSessionKey(sessionId: string, aesKeyBase64: string) {
    this.sessionKeys.set(sessionId, aesKeyBase64)
    this.saveKeysToStorage() // ✅ 持久化
  }
}
```

### 2. 退出登录时清除密钥

```typescript
const logout = () => {
  token.value = null
  currentUser.value = null
  localStorage.removeItem(API_CONFIG.TOKEN_KEY)
  
  // ✅ 清除会话密钥
  backendCrypto.clearAllKeys()
  console.log('✅ 已清除所有会话密钥')
}
```

### 修复前
- ❌ 重新登录后无法查看历史消息
- ❌ 刷新页面后无法查看历史消息

### 修复后

- ✅ 刷新页面后可以查看历史消息

- ✅ 退出登录时自动清除密钥（安全）

---

## 🔐 安全考虑

### 优点
- ✅ 用户体验好，历史消息始终可见
- ✅ 减少后端请求（不需要频繁获取密钥）
- ✅ 退出登录时自动清除（安全）

### 风险
- ⚠️ 密钥存储在 localStorage（明文）
- ⚠️ 如果设备被盗，密钥可能泄露

### 缓解措施
1. **设备锁定**：建议用户设置设备密码
2. **自动登出**：长时间不活动自动退出
3. **密钥加密**：可以考虑用设备指纹加密密钥（未来优化）

## 📝 后端建议

### 当前方案的问题

前端持久化密钥只是临时方案，更好的方案是：

### 方案 1: 后端持久化密钥（推荐）

```java
// 后端数据库表
CREATE TABLE session_keys (
  session_id VARCHAR(64) PRIMARY KEY,
  aes_key VARCHAR(256) NOT NULL,
  created_at DATETIME NOT NULL,
  updated_at DATETIME NOT NULL
);

// 后端 API
GET /api/encryption/session/{sessionId}/key
- 返回该会话的密钥
- 如果不存在，自动生成并保存
```

### 方案 2: 密钥派生（最安全）

```java
// 使用用户密码派生会话密钥
sessionKey = PBKDF2(userPassword, sessionId, iterations)
```

优点：
- 不需要存储密钥
- 用户密码就是主密钥
- 更安全

缺点：
- 忘记密码后无法恢复历史消息
- 需要用户每次输入密码
