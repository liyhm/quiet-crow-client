# 发送方图片消息立即显示修复

## 问题描述

发送方发送图片后，图片消息不会立即显示在消息列表中，需要手动点击会话才能显示。

## 问题原因

在之前的修复中（TASK 18），为了解决消息重复发送的问题，我们移除了乐观更新（optimistic update）。这导致：

1. 发送方发送消息后，不会立即添加到本地消息列表
2. 必须等待 WebSocket 回调（`handleIncomingMessage`）才能看到消息
3. WebSocket 回调可能有 100-500ms 的延迟，导致用户体验不佳

## 解决方案

### 1. 恢复乐观更新

在发送消息时，立即添加到本地消息列表，使用临时 ID：

```typescript
// 文本消息（useMessageStore.ts）
const tempMessageId = `temp-${Date.now()}-${Math.random()}`
const message: Message = {
  id: tempMessageId,
  sessionId,
  senderId: authStore.userId || 'current-user',
  content,
  timestamp: new Date().toISOString(),
  status: MessageStatus.SENDING,
  isRead: false,
  type: MessageType.TEXT
}
addMessage(sessionId, message)

// 媒体消息（MainPage.vue）
const tempMessageId = `temp-${Date.now()}-${Math.random()}`
const localMessage: Message = {
  id: tempMessageId,
  sessionId: activeSessionId.value,
  senderId: authStore.userId || 'current-user',
  content: messageContent,
  timestamp: new Date().toISOString(),
  status: MessageStatus.SENDING,
  isRead: false,
  type: messageType === 'IMAGE' ? MessageType.IMAGE : MessageType.FILE
}
messageStore.addMessage(activeSessionId.value, localMessage)
```

### 2. 改进去重逻辑

在 `receiveMessage` 方法中，改进去重逻辑，支持临时消息替换：

```typescript
// 1. 检查是否有相同 ID 的消息（避免重复）
const duplicateById = existingMessages.some(m => m.id === messageId)
if (duplicateById) {
  return null
}

// 2. 如果是自己发送的消息，查找临时消息并替换
if (senderId === authStore.userId) {
  const tempMessage = existingMessages.find(m => 
    m.id.startsWith('temp-') && 
    m.senderId === senderId &&
    m.content === content &&
    Math.abs(new Date(m.timestamp).getTime() - new Date(timestamp).getTime()) < 5000
  )
  
  if (tempMessage) {
    // 更新临时消息的 ID 和状态
    tempMessage.id = messageId
    tempMessage.status = MessageStatus.DELIVERED
    tempMessage.timestamp = timestamp
    return tempMessage
  }
}

// 3. 如果没有找到临时消息，添加新消息
addMessage(message.sessionId, message)
```

### 3. 匹配条件

临时消息和 WebSocket 回调消息的匹配条件：

- 发送者 ID 相同（`senderId === authStore.userId`）
- 消息内容相同（`m.content === content`）
- 时间戳相差小于 5 秒（`Math.abs(timestamp1 - timestamp2) < 5000`）
- 临时 ID 格式（`m.id.startsWith('temp-')`）

## 技术细节

### 临时 ID 格式

```typescript
`temp-${Date.now()}-${Math.random()}`
```

- `temp-` 前缀：标识这是临时消息
- `Date.now()`：时间戳，确保唯一性
- `Math.random()`：随机数，避免同一毫秒内的冲突

### 消息状态流转

```
SENDING (发送中) → SENT (已发送) → DELIVERED (已送达)
```

1. 乐观更新时：`SENDING`
2. WebSocket 发送成功后：`SENT`
3. WebSocket 回调时：`DELIVERED`（替换临时消息）

### 去重策略

1. **按 ID 去重**：如果消息 ID 已存在，跳过
2. **临时消息替换**：如果是自己发送的消息，查找临时消息并替换
3. **内容匹配**：通过发送者、内容、时间戳匹配临时消息

## 优势

1. **即时反馈**：用户发送消息后立即看到，体验流畅
2. **无重复**：通过改进的去重逻辑，避免消息重复
3. **状态可见**：显示消息发送状态（发送中、已发送、已送达）
4. **容错性强**：即使 WebSocket 回调失败，消息也会显示（状态为"发送中"）

## 注意事项

1. 临时 ID 必须使用 `temp-` 前缀
2. 时间戳匹配容差为 5 秒（可根据实际情况调整）
3. 只有自己发送的消息才会进行临时消息替换
4. 接收方的消息直接添加，不会触发临时消息替换逻辑
