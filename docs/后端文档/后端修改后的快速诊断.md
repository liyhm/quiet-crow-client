# 后端修改后的快速诊断指南

## 问题现象
后端修改后，消息接收可能出现问题。

## 快速诊断步骤

### 步骤 1：检查 WebSocket 连接状态

打开浏览器控制台（F12），查看是否有以下日志：

```
✅ WebSocket 连接成功！
✅ 私有消息队列订阅成功
```

**如果没有看到这些日志**：
- WebSocket 连接失败
- 可能是后端 WebSocket 配置有问题

### 步骤 2：发送测试消息

1. 在聊天界面发送一条消息
2. 查看控制台是否有：

```
📤 发送消息到 WebSocket: {...}
✅ 消息已发送
```

**如果看到这些日志**：
- 前端发送成功
- 问题可能在后端

### 步骤 3：检查后端是否收到消息

查看后端控制台，应该有类似的日志：
- 收到消息
- 处理消息
- 转发消息

**如果后端没有日志**：
- WebSocket 连接可能有问题
- 检查后端 WebSocket 配置

**如果后端有收到但没有转发**：
- 后端消息转发逻辑有问题
- 需要检查后端代码

### 步骤 4：检查接收端

在接收端（另一个用户）的控制台，应该看到：

```
📬 收到私有消息: {...}
📨 收到 WebSocket 消息: {...}
📩 处理接收到的消息: {...}
🔓 消息解密成功: xxx
✅ 消息已添加到 store
```

**如果没有这些日志**：
- 后端没有转发消息
- 或者转发的目标地址不对

## 常见问题和解决方案

### 问题 1：WebSocket 连接失败

**可能原因**：
- 后端 WebSocket 端点改变了
- 后端 CORS 配置有问题
- 后端 WebSocket 服务没启动

**解决方案**：
1. 检查 `.env.development` 中的 `VITE_WS_URL`
2. 确认后端 WebSocket 端点是否是 `http://localhost:8080/ws`
3. 测试后端 `/ws/info` 端点是否可访问

### 问题 2：消息发送成功但接收端收不到

**可能原因**：
- 后端没有实现消息转发
- 后端转发逻辑有 bug
- 订阅的队列地址不对

**解决方案**：
1. 检查后端是否实现了 WebSocket 消息处理器
2. 检查后端是否正确转发到 `/user/queue/messages`
3. 参考文档：`后端需求-实时消息转发功能.md`

### 问题 3：消息显示问题

**可能原因**：
- 后端返回的数据格式改变了
- `sendUser` 字段名称或格式改变了
- 消息内容字段改变了

**解决方案**：
1. 在控制台查看接收到的原始消息数据
2. 对比后端返回的字段名称
3. 如果字段名改变了，需要更新前端代码

## 后端需要检查的关键点

### 1. WebSocket 配置

```java
// WebSocket 端点应该是
@Configuration
@EnableWebSocketMessageBroker
public class WebSocketConfig implements WebSocketMessageBrokerConfigurer {
    
    @Override
    public void registerStompEndpoints(StompEndpointRegistry registry) {
        registry.addEndpoint("/ws")
                .setAllowedOrigins("*")
                .withSockJS();
    }
    
    @Override
    public void configureMessageBroker(MessageBrokerRegistry registry) {
        registry.enableSimpleBroker("/topic", "/queue");
        registry.setApplicationDestinationPrefixes("/app");
        registry.setUserDestinationPrefix("/user");
    }
}
```

### 2. 消息处理器

```java
@MessageMapping("/chat/{sessionId}")
public void handleMessage(
    @DestinationVariable String sessionId,
    @Payload MessageDto message,
    SimpMessageHeaderAccessor headerAccessor
) {
    // 1. 保存消息到数据库
    Message savedMessage = messageService.saveMessage(message);
    
    // 2. 获取会话的所有用户
    List<String> userIds = sessionService.getSessionUsers(sessionId);
    
    // 3. 转发给每个用户（除了发送者）
    String senderId = headerAccessor.getUser().getName();
    for (String userId : userIds) {
        if (!userId.equals(senderId)) {
            messagingTemplate.convertAndSendToUser(
                userId,
                "/queue/messages",
                savedMessage
            );
        }
    }
}
```

### 3. 消息格式

后端转发的消息应该包含：
```json
{
  "id": "message-uuid",
  "sessionId": "session-uuid",
  "sendUser": "sender-user-uuid",
  "content": "encrypted-content",
  "messageType": "1",
  "sendTime": "2024-02-27T10:00:00"
}
```

## 下一步行动

### 如果是前端问题
1. 检查 WebSocket 连接配置
2. 检查消息处理逻辑
3. 更新字段映射

### 如果是后端问题
1. 实现消息转发功能（参考 `后端需求-实时消息转发功能.md`）
2. 确保消息格式正确
3. 测试消息转发是否工作

### 如果不确定
1. 同时查看前端和后端控制台
2. 对比发送和接收的数据
3. 逐步排查每个环节
