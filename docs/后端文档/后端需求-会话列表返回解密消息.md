# 后端需求：会话列表返回解密消息预览

## 问题描述

当前会话列表返回的 `lastMessage` 是加密的内容，前端无法直接显示，只能显示 `[加密消息]` 或 `新消息`。

### 当前返回数据

```json
{
  "sessionId": "xxx",
  "showName": "lyh2",
  "lastMessage": "/wdaXLeFHIq2H2pwij...",  // 加密的 Base64 字符串
  "lastMessageTime": "2024-02-27T16:00:00",
  "unreadCount": 1
}
```

### 用户体验问题

- 用户看不到消息预览
- 只能看到 `新消息` 或 `[加密消息]`
- 无法快速了解对话内容

---

## 需求

### 目标

会话列表返回解密后的消息内容，方便用户预览。

### 输入

- 会话 ID
- 当前用户 ID

### 输出

```json
{
  "sessionId": "xxx",
  "showName": "lyh2",
  "lastMessage": "lyh2 发测试消息2",  // 解密后的文本
  "lastMessageTime": "2024-02-27T16:00:00",
  "unreadCount": 1
}
```

### 业务规则

1. 从数据库获取最后一条消息
2. 使用会话密钥解密消息内容
3. 截取前 50 个字符作为预览
4. 如果解密失败，返回 `[消息]` 或空字符串

---

## 实现要点

### 1. 获取最后一条消息

**查询**：
```sql
SELECT message_content 
FROM b_session_log 
WHERE session_id = ? 
ORDER BY send_time DESC 
LIMIT 1
```

### 2. 解密消息

**步骤**：
1. 获取会话密钥（从 `b_session_key` 表）
2. 使用 AES-ECB 解密消息内容
3. 如果解密失败，返回默认文本

### 3. 截取预览

**规则**：
- 最多显示 50 个字符
- 超过 50 个字符，添加 `...`
- 换行符替换为空格

**示例**：
```
原文: "这是一条很长的消息内容，包含了很多信息..."
预览: "这是一条很长的消息内容，包含了很多信息..."

原文: "短消息"
预览: "短消息"
```

## 性能考虑

### 解密开销

**场景**：用户有 50 个会话

**开销**：
- 50 次数据库查询（获取最后一条消息）
- 50 次解密操作
- 总耗时：约 50-100ms

**优化方案**：
1. 批量查询最后一条消息
2. 缓存会话密钥
3. 异步解密（如果性能有问题）

### SQL 优化

**批量查询**：
```sql
SELECT session_id, message_content, send_time
FROM b_session_log
WHERE (session_id, send_time) IN (
  SELECT session_id, MAX(send_time)
  FROM b_session_log
  WHERE session_id IN (?, ?, ?, ...)
  GROUP BY session_id
)
```

## 错误处理

### 解密失败

**场景**：
- 会话密钥不存在
- 消息内容损坏
- 加密算法不匹配

**处理**：
- 返回 `[消息]` 或空字符串
- 记录错误日志
- 不影响其他会话的加载

### 消息不存在

**场景**：
- 新创建的会话，还没有消息

**处理**：
- `lastMessage` 返回空字符串
- `lastMessageTime` 返回会话创建时间
