# 后端需求：群组和会话管理

## 概述

实现微信模式的会话删除和群组管理功能。

---

## 核心设计原则

### 私聊
- 删除会话 = 软删除（只删除当前用户的记录）
- 对方不受影响
- 对方发消息后，会话自动恢复

### 群聊
- 删除会话 = 软删除（只删除当前用户的记录）
- 退出群聊 = 不再接收消息
- 解散群聊 = 所有成员的会话都消失（只有群主可以）

  - `ADMIN`：管理员
  - `MEMBER`：普通成员

## API 需求

### 1. 删除会话（已实现，需修改）

**接口**：`DELETE /api/session/{sessionId}`

**当前实现**：硬删除 `session_user` 记录

**需要修改为**：软删除

**逻辑**：
1. 查找当前用户的 `session_user` 记录
2. 设置 `is_deleted = true`
3. 设置 `deleted_time = 当前时间`
4. 不删除 `session` 和 `message` 记录

**SQL**：
```sql
UPDATE b_session_user 
SET is_deleted = true, 
    deleted_time = NOW(),
    updated_time = NOW()
WHERE session_id = ? 
  AND user_id = ?
```

**响应**：
```json
{
  "code": 200,
  "message": "会话已删除",
  "data": "会话已删除"
}
```

### 3. 接收消息时自动恢复会话

**触发时机**：用户收到新消息时

**逻辑**：
1. 检查接收者的 `session_user` 记录
2. 如果 `is_deleted = true`，自动恢复：
   - 设置 `is_deleted = false`
   - 清空 `deleted_time`
   - 更新 `updated_time`

**SQL**：
```sql
UPDATE b_session_user 
SET is_deleted = false, 
    deleted_time = NULL,
    updated_time = NOW()
WHERE session_id = ? 
  AND user_id = ?
  AND is_deleted = true
```

**实现位置**：在消息转发逻辑中添加

```java
// 伪代码
public void forwardMessage(String sessionId, Message message) {
    // 获取会话成员
    List<String> userIds = getSessionUsers(sessionId);
    
    // 转发给每个成员
    for (String userId : userIds) {
        // 检查并恢复已删除的会话
        restoreDeletedSession(sessionId, userId);
        
        // 转发消息
        messagingTemplate.convertAndSendToUser(userId, "/queue/messages", message);
    }
}

private void restoreDeletedSession(String sessionId, String userId) {
    // 如果会话被删除，自动恢复
    sessionUserRepository.restoreIfDeleted(sessionId, userId);
}
```

### 4. 创建群聊

**接口**：`POST /api/session/group`

**请求**：
```json
{
  "name": "我的群聊",
  "memberIds": ["user1", "user2", "user3"]
}
```

**逻辑**：
1. 验证成员是否存在
2. 创建 `session` 记录（type = GROUP）
3. 创建群主的 `session_user` 记录（role = OWNER）
4. 创建成员的 `session_user` 记录（role = MEMBER）
5. 生成群聊密钥
6. 通知所有成员加入群聊

**SQL**：
```sql
-- 1. 创建会话
INSERT INTO b_session (id, type, name, created_by, created_time)
VALUES (?, 'GROUP', ?, ?, NOW());

-- 2. 创建群主记录
INSERT INTO b_session_user (id, session_id, user_id, session_user_type, role, created_time)
VALUES (?, ?, ?, 'GROUP', 'OWNER', NOW());

-- 3. 创建成员记录
INSERT INTO b_session_user (id, session_id, user_id, session_user_type, role, created_time)
VALUES (?, ?, ?, 'GROUP', 'MEMBER', NOW());
```

**响应**：
```json
{
  "code": 200,
  "data": {
    "sessionId": "group-session-uuid",
    "type": "GROUP",
    "name": "我的群聊",
    "memberCount": 4
  }
}
```

**业务规则**：
- 群名称不能为空
- 至少需要 2 个成员（不包括创建者）
- 最多 500 个成员
- 创建者自动成为群主

### 5. 退出群聊

**接口**：`POST /api/session/{sessionId}/leave`

**权限**：任何群成员

**逻辑**：
1. 验证用户是群成员
2. 验证不是群主（群主不能退出，只能解散）
3. 设置 `is_deleted = true`
4. 设置 `deleted_time = 当前时间`
5. 通知其他成员（可选）

**SQL**：
```sql
-- 检查是否是群主
SELECT role FROM b_session_user 
WHERE session_id = ? AND user_id = ?;

-- 如果不是群主，设置为已删除
UPDATE b_session_user 
SET is_deleted = true, 
    deleted_time = NOW()
WHERE session_id = ? 
  AND user_id = ?
  AND role != 'OWNER';
```

**响应**：
```json
{
  "code": 200,
  "message": "已退出群聊",
  "data": "已退出群聊"
}
```

**错误处理**：
- 如果是群主，返回错误：`群主不能退出群聊，请先转让群主或解散群聊`
- 如果不是群成员，返回错误：`您不是该群的成员`

---

### 6. 解散群聊

**接口**：`DELETE /api/session/{sessionId}/disband`

**权限**：只有群主

**逻辑**：
1. 验证用户是群主
2. 删除 `session` 记录（或标记为已解散）
3. 将所有成员的 `session_user` 记录设置为 `is_deleted = true`
4. 通知所有成员群聊已解散

**SQL**：
```sql
-- 1. 验证是否是群主
SELECT role FROM b_session_user 
WHERE session_id = ? AND user_id = ?;

-- 2. 标记会话为已解散（可选）
UPDATE b_session 
SET is_disbanded = true, disbanded_time = NOW()
WHERE id = ?;

-- 3. 删除所有成员的记录
UPDATE b_session_user 
SET is_deleted = true, deleted_time = NOW()
WHERE session_id = ?;
```

**响应**：
```json
{
  "code": 200,
  "message": "群聊已解散",
  "data": "群聊已解散"
}
```

**错误处理**：
- 如果不是群主，返回错误：`只有群主可以解散群聊`

**权限**：群成员

1. 验证用户是群成员
2. 查询所有未删除的成员
3. 返回成员列表（包含角色信息）

**SQL**：
```sql
SELECT su.user_id, su.role, su.created_time as joined_time,
       u.username, u.show_name, u.avatar
FROM b_session_user su
JOIN b_user u ON su.user_id = u.id
WHERE su.session_id = ? 
  AND su.is_deleted = false
ORDER BY 
  CASE su.role 
    WHEN 'OWNER' THEN 1 
    WHEN 'ADMIN' THEN 2
```

### 8. 添加群成员

**接口**：`POST /api/session/{sessionId}/members`

**权限**：群主或管理员

**请求**：
```json
{
  "userIds": ["user4", "user5"]
}
```

**逻辑**：
1. 验证当前用户是群主或管理员
2. 验证要添加的用户存在
3. 验证用户不是已有成员
4. 创建 `session_user` 记录（role = MEMBER）
5. 通知新成员加入群聊
6. 通知其他成员有新成员加入

**SQL**：
```sql
-- 1. 验证权限
SELECT role FROM b_session_user 
WHERE session_id = ? AND user_id = ?;

-- 2. 检查用户是否已是成员
SELECT user_id FROM b_session_user 
WHERE session_id = ? AND user_id IN (?, ?) AND is_deleted = false;

-- 3. 添加新成员
INSERT INTO b_session_user (id, session_id, user_id, session_user_type, role, created_time)
VALUES (?, ?, ?, 'GROUP', 'MEMBER', NOW());
```

**响应**：
```json
{
  "code": 200,
  "message": "成员添加成功",
  "data": {
    "addedCount": 2,
    "memberCount": 6
  }
}
```

**业务规则**：
- 只有群主和管理员可以添加成员
- 不能添加已是成员的用户
- 群成员数量不能超过 500

---

### 9. 移除群成员

**接口**：`DELETE /api/session/{sessionId}/members/{userId}`

**权限**：群主或管理员

**逻辑**：
1. 验证当前用户是群主或管理员
2. 验证要移除的用户是群成员
3. 验证不能移除群主
4. 设置该成员的 `is_deleted = true`
5. 通知被移除的成员
6. 通知其他成员

**SQL**：
```sql
-- 1. 验证权限
SELECT role FROM b_session_user 
WHERE session_id = ? AND user_id = ?;

-- 2. 检查被移除用户的角色
SELECT role FROM b_session_user 
WHERE session_id = ? AND user_id = ?;

-- 3. 移除成员
UPDATE b_session_user 
SET is_deleted = true, deleted_time = NOW()
WHERE session_id = ? AND user_id = ?;
```

**响应**：
```json
{
  "code": 200,
  "message": "成员已移除",
  "data": "成员已移除"
}
```

**业务规则**：
- 只有群主和管理员可以移除成员
- 不能移除群主
- 管理员不能移除其他管理员（只有群主可以）

## 通知机制

### WebSocket 通知

**群聊事件通知**：

```json
{
  "type": "GROUP_EVENT",
  "sessionId": "group-session-uuid",
  "eventType": "MEMBER_JOINED",  // MEMBER_JOINED/MEMBER_LEFT/MEMBER_REMOVED/GROUP_DISBANDED
  "data": {
    "userId": "user-uuid",
    "username": "张三",
    "timestamp": "2024-02-27T16:00:00"
  }
}
```

**事件类型**：
- `MEMBER_JOINED`：新成员加入
- `MEMBER_LEFT`：成员退出
- `MEMBER_REMOVED`：成员被移除
- `GROUP_DISBANDED`：群聊已解散
- `ROLE_CHANGED`：角色变更
