**方法**: `DELETE`  
**路径**: `/api/session/{sessionId}`  
**认证**: 需要 Bearer Token  
**功能**: 软删除会话（只删除当前用户的 session_user 记录）

## 实现代码

### 1. Controller 层

```java
package com.example.chat.controller;

import com.example.chat.common.Result;
import com.example.chat.service.SessionService;
import com.example.chat.util.SecurityUtils;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.web.bind.annotation.*;

@RestController
@RequestMapping("/api/session")
public class SessionController {

    @Autowired
    private SessionService sessionService;

    /**
     * 删除会话（软删除）
     * 只删除当前用户的 session_user 记录，不影响其他用户
     */
    @DeleteMapping("/{sessionId}")
    public Result<Void> deleteSession(@PathVariable String sessionId) {
        String userId = SecurityUtils.getCurrentUserId();
        sessionService.deleteSessionForUser(sessionId, userId);
        return Result.success();
    }
}
```

### 2. Service 层

```java
package com.example.chat.service;

import com.example.chat.entity.SessionUser;
import com.example.chat.repository.SessionUserRepository;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

@Service
public class SessionService {

    @Autowired
    private SessionUserRepository sessionUserRepository;

    /**
     * 删除用户的会话（软删除）
     * 只删除当前用户的 session_user 记录
     * 
     * @param sessionId 会话ID
     * @param userId 用户ID
     */
    @Transactional
    public void deleteSessionForUser(String sessionId, String userId) {
        // 查找用户的 session_user 记录
        SessionUser sessionUser = sessionUserRepository
            .findBySessionIdAndUserId(sessionId, userId)
            .orElseThrow(() -> new RuntimeException("会话不存在或无权访问"));
        
        // 删除记录
        sessionUserRepository.delete(sessionUser);
        
        // 可选：记录日志
        log.info("用户 {} 删除了会话 {}", userId, sessionId);
    }
}
```

### 3. Repository 层

## 数据库操作

### SQL 语句

```sql
-- 删除用户的会话记录
DELETE FROM session_user 
WHERE session_id = ? AND user_id = ?;
```

### 注意事项

1. **不删除 session 表记录** - 其他用户可能还在使用这个会话
2. **不删除 message 表记录** - 保留消息历史
3. **只删除 session_user 表记录** - 只影响当前用户

## 软删除 vs 硬删除

### 软删除（推荐）✅

**实现**: 只删除 `session_user` 记录

**优点**:
- 不影响其他用户
- 对方仍能看到会话和消息
- 可以恢复（如果需要）
- 符合用户预期

**缺点**:
- 会话和消息仍占用存储空间

### 硬删除（不推荐）❌

**实现**: 删除 `session`、`session_user`、`message` 记录

**优点**:
- 彻底清理数据
- 节省存储空间

**缺点**:
- 影响其他用户（对方也会失去会话）
- 无法恢复
- 可能引起用户困惑

### 2. 恢复删除

可以添加一个 `deleted` 字段，实现逻辑删除：

```java
// SessionUser 实体添加字段
private Boolean deleted = false;
private LocalDateTime deletedTime;

// 删除时只标记为已删除
public void deleteSessionForUser(String sessionId, String userId) {
    SessionUser sessionUser = sessionUserRepository
        .findBySessionIdAndUserId(sessionId, userId)
        .orElseThrow(() -> new RuntimeException("会话不存在"));
    
    sessionUser.setDeleted(true);
    sessionUser.setDeletedTime(LocalDateTime.now());
    sessionUserRepository.save(sessionUser);
}

// 查询时过滤已删除的记录
public List<SessionUserVO> getMySessions(String userId) {
    return sessionUserRepository
        .findByUserIdAndDeleted(userId, false)
        .stream()
        .map(this::convertToVO)
        .collect(Collectors.toList());
}
```

### 3. 定期清理

可以添加定时任务，清理长期未使用的已删除记录：

```java
@Scheduled(cron = "0 0 2 * * ?")  // 每天凌晨2点执行
public void cleanupDeletedSessions() {
    LocalDateTime threshold = LocalDateTime.now().minusDays(30);
    sessionUserRepository.deleteByDeletedTrueAndDeletedTimeBefore(threshold);
}
```
