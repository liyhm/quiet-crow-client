# 后端需求 - 实时消息转发功能

## 需求背景

**当前问题**：
- 用户发送消息后，消息已保存到数据库
- 但对方用户收不到实时推送
- 需要手动刷新或点击会话才能看到新消息

**期望效果**：
- 用户A发送消息
- 用户B立即收到消息推送
- 消息实时显示在聊天界面

## 核心需求

### 需求 1：实现 WebSocket 消息处理器

**功能描述**：
接收前端通过 WebSocket 发送的聊天消息，并进行处理。

**接收路径**：
- 前端发送到：`/app/chat/{sessionId}`
- 需要使用 `@MessageMapping` 注解

**接收参数**：
- `sessionId`：会话ID（从路径获取）
- 请求体包含：
  - `sessionId`：会话ID
  - `content`：加密的消息内容
  - `messageType`：消息类型（"1" 表示文本）
  - `encrypted`：是否加密（true）
- `Principal`：当前登录用户（从 WebSocket 连接获取）

**处理流程**：
1. 获取发送者ID（从 Principal 获取）
2. 保存消息到数据库
3. 查询会话的所有成员
4. 遍历成员，排除发送者自己
5. 将消息转发给其他成员

**日志要求**：
- 打印收到的消息信息（会话ID、发送者、内容前20个字符）
- 打印消息保存结果（消息ID）
- 打印会话成员列表
- 打印每个转发操作的结果

### 需求 2：消息转发给特定用户

**功能描述**：
将消息通过 WebSocket 推送给指定用户。

**转发方式**：
- 使用 `SimpMessagingTemplate.convertAndSendToUser()`
- 目标路径：`/queue/messages`（前端订阅 `/user/queue/messages`）
- 注意：不要包含 `/user` 前缀，框架会自动添加

**转发内容**：
需要包含以下字段：
- `id`：消息ID（数据库生成的）
- `sessionId`：会话ID
- `sendUser`：发送者用户ID（注意字段名是 `sendUser`）
- `content`：加密的消息内容（原样转发）
- `sendTime`：发送时间（ISO 8601 格式字符串）
- `messageType`：消息类型

### 需求 3：查询会话成员

**功能描述**：
根据会话ID查询该会话的所有成员用户ID。

**查询来源**：
- 从 `session_user` 表查询
- 条件：`session_id = ?`

**返回结果**：
- 返回用户ID列表（List<String>）
- 私聊会话应该返回2个用户ID
- 群聊会话返回所有成员的用户ID

**注意事项**：
- 确保返回的是用户ID（`user_id` 字段）
- 不是关系ID或其他ID

### 需求 4：保存消息到数据库

**功能描述**：
将接收到的消息保存到 `message` 表。

**保存字段**：
- `id`：消息ID（UUID）
- `session_id`：会话ID
- `send_user`：发送者用户ID
- `message_content`：加密的消息内容
- `message_type`：消息类型
- `send_time`：发送时间（当前时间）
- `is_read`：是否已读（false）

**返回结果**：
- 返回保存后的消息对象（包含生成的ID和时间）

## 数据格式

### 前端发送的消息格式

```json
{
  "sessionId": "session-uuid",
  "content": "加密的消息内容（Base64）",
  "messageType": "1",
  "encrypted": true
}
```

### 后端转发的消息格式

```json
{
  "id": "message-uuid",
  "sessionId": "session-uuid",
  "sendUser": "sender-user-id",
  "content": "加密的消息内容（Base64）",
  "sendTime": "2024-02-27T10:30:00",
  "messageType": "1"
}
```

### 常见问题排查

**问题 1：后端没有收到消息**
- 检查 `@MessageMapping` 路径是否是 `/chat/{sessionId}`
- 检查 WebSocket 配置是否正确

**问题 2：消息保存失败**
- 检查数据库连接
- 检查字段映射是否正确
- 查看异常日志

**问题 3：查询不到会话成员**
- 检查 `session_user` 表是否有数据
- 检查查询条件是否正确
- 确认返回的是用户ID而不是其他ID

**问题 4：消息转发失败**
- 检查用户ID是否正确
- 检查转发路径是否是 `/queue/messages`
- 检查是否使用了 `convertAndSendToUser` 方法
- 确认字段名是 `sendUser` 而不是 `senderId`

**问题 5：前端收不到消息**
- 检查后端是否打印了"消息已转发"日志
- 检查转发的用户ID是否正确
- 检查消息格式是否符合要求

## 补充说明

### 关于用户ID获取

- 从 `Principal.getName()` 获取当前登录用户ID
- 这个ID应该在用户登录时设置到 WebSocket 连接中

### 关于消息加密

- 前端发送的消息内容已加密
- 后端不需要解密，原样转发即可
- 接收方前端会自动解密

### 关于消息类型

- `"1"` 表示文本消息
- 后续可能支持其他类型（图片、文件等）

### 关于异常处理

- 建议添加 try-catch 捕获异常
- 打印详细的错误日志
- 避免异常导致消息丢失
