# 后端需求：媒体文件上传问题修复

## 📋 问题描述

根据前端测试，发现后端媒体文件上传接口存在以下问题：

### 问题 1: `created_at` 字段不能为 null

**错误信息**:
```
Error updating database. Cause:
java.sql.SQLIntegrityConstraintViolationException:
Column 'created_at' cannot be null
```

**原因分析**:
- 数据库表 `b_media_file` 的 `created_at` 字段没有设置默认值
- 或者后端代码在插入数据时没有设置 `created_at` 字段

### 问题 2: `mimeType` 字段验证过于严格

**错误信息**:
```json
{
  "code": 500,
  "message": "缺少必需字段: mimeType",
  "data": null
}
```

**原因分析**:
- 某些文件（如没有扩展名的文件）的 `file.type` 为空
- 前端已修复（为空时使用 `application/octet-stream`）
- 但后端也应该容错处理

## 🔧 修复方案

### 修复 1: 数据库表结构

确保 `b_media_file` 表的 `created_at` 和 `updated_at` 字段有默认值：

```sql
ALTER TABLE b_media_file 
MODIFY COLUMN created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间';

ALTER TABLE b_media_file 
MODIFY COLUMN updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间';
```

**完整的表结构**:
```sql
CREATE TABLE IF NOT EXISTS b_media_file (
  id VARCHAR(64) PRIMARY KEY COMMENT '文件ID',
  session_id VARCHAR(64) NOT NULL COMMENT '会话ID',
  uploader_id VARCHAR(64) NOT NULL COMMENT '上传者用户ID',
  file_name VARCHAR(512) NOT NULL COMMENT '加密的文件名（Base64编码）',
  file_size BIGINT NOT NULL COMMENT '文件大小（字节）',
  mime_type VARCHAR(128) NOT NULL COMMENT 'MIME类型',
  message_type VARCHAR(32) NOT NULL COMMENT '消息类型：IMAGE, VIDEO',
  storage_path VARCHAR(512) NOT NULL COMMENT '文件存储路径',
  access_url VARCHAR(512) COMMENT '访问URL',
  upload_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '上传时间',
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  
  INDEX idx_session_id (session_id),
  INDEX idx_uploader_id (uploader_id),
  INDEX idx_upload_time (upload_time)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='媒体文件表';
```

### 修复 3: 后端代码 - 字段自动填充

添加 MyBatis Plus 的字段自动填充处理器：

```java
@Component
public class MyMetaObjectHandler implements MetaObjectHandler {
    
    @Override
    public void insertFill(MetaObject metaObject) {
        LocalDateTime now = LocalDateTime.now();
        
        // 自动填充创建时间
        this.strictInsertFill(metaObject, "createdAt", LocalDateTime.class, now);
        this.strictInsertFill(metaObject, "uploadTime", LocalDateTime.class, now);
        
        // 自动填充更新时间
        this.strictInsertFill(metaObject, "updatedAt", LocalDateTime.class, now);
    }
    
    @Override
    public void updateFill(MetaObject metaObject) {
        // 自动填充更新时间
        this.strictUpdateFill(metaObject, "updatedAt", LocalDateTime.class, LocalDateTime.now());
    }
}
```

### 修复 4: 后端代码 - mimeType 容错处理

在上传接口中添加 mimeType 的容错处理：

```java
@PostMapping("/upload")
public Result<UploadResponse> uploadMedia(
    @RequestParam("file") MultipartFile file,
    @RequestParam("metadata") String metadataJson
) {
    try {
        // 解析 metadata
        MediaMetadata metadata = JSON.parseObject(metadataJson, MediaMetadata.class);
        
        // 验证必需字段
        if (StringUtils.isBlank(metadata.getSessionId())) {
            return Result.error("缺少必需字段: sessionId");
        }
        if (StringUtils.isBlank(metadata.getFileName())) {
            return Result.error("缺少必需字段: fileName");
        }
        if (metadata.getFileSize() == null || metadata.getFileSize() <= 0) {
            return Result.error("缺少必需字段: fileSize");
        }
        
        // mimeType 容错处理
        if (StringUtils.isBlank(metadata.getMimeType())) {
            log.warn("mimeType 为空，使用默认值: application/octet-stream");
            metadata.setMimeType("application/octet-stream");
        }
        
        // messageType 容错处理
        if (StringUtils.isBlank(metadata.getMessageType())) {
            // 根据 mimeType 推断 messageType
            if (metadata.getMimeType().startsWith("image/")) {
                metadata.setMessageType("IMAGE");
            } else if (metadata.getMimeType().startsWith("video/")) {
                metadata.setMessageType("VIDEO");
            } else {
                metadata.setMessageType("FILE");
            }
        }
        
        // 验证文件大小
        if ("IMAGE".equals(metadata.getMessageType()) && metadata.getFileSize() > 5 * 1024 * 1024) {
            return Result.error("图片文件大小不能超过 5MB");
        }
        if ("VIDEO".equals(metadata.getMessageType()) && metadata.getFileSize() > 50 * 1024 * 1024) {
            return Result.error("视频文件大小不能超过 50MB");
        }
        
        // 验证用户权限
        String userId = StpUtil.getLoginIdAsString();
        if (!sessionService.isMember(metadata.getSessionId(), userId)) {
            return Result.error("无权限上传文件到该会话");
        }
        
        // 保存文件
        String fileId = IdUtil.fastSimpleUUID();
        String storagePath = buildStoragePath(fileId, metadata.getMessageType());
        
        // 保存到磁盘
        File destFile = new File(storagePath);
        destFile.getParentFile().mkdirs();
        file.transferTo(destFile);
        
        // 保存到数据库
        MediaFile mediaFile = new MediaFile();
        mediaFile.setId(fileId);
        mediaFile.setSessionId(metadata.getSessionId());
        mediaFile.setUploaderId(userId);
        mediaFile.setFileName(metadata.getFileName());
        mediaFile.setFileSize(metadata.getFileSize());
        mediaFile.setMimeType(metadata.getMimeType());
        mediaFile.setMessageType(metadata.getMessageType());
        mediaFile.setStoragePath(storagePath);
        // createdAt 和 updatedAt 会自动填充
        
        mediaFileService.save(mediaFile);
        
        // 返回结果
        UploadResponse response = new UploadResponse();
        response.setFileId(fileId);
        response.setUploadTime(LocalDateTime.now().toString());
        response.setFileSize(metadata.getFileSize());
        response.setThumbnailUrl(null);
        
        return Result.success(response);
        
    } catch (Exception e) {
        log.error("文件上传失败", e);
        return Result.error("文件上传失败: " + e.getMessage());
    }
}

/**
 * 构建存储路径
 * 格式: /data/media/{messageType}/{year}/{month}/{day}/{fileId}.enc
 */
private String buildStoragePath(String fileId, String messageType) {
    LocalDateTime now = LocalDateTime.now();
    String type = messageType.toLowerCase() + "s"; // images, videos
    
    return String.format("%s/%s/%d/%02d/%02d/%s.enc",
        mediaStorageBasePath,
        type,
        now.getYear(),
        now.getMonthValue(),
        now.getDayOfMonth(),
        fileId
    );
}
```
