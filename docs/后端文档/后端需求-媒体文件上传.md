# 后端需求：媒体文件上传与下载

## 📋 需求概述

前端需要支持在聊天中发送图片和视频文件，需要后端提供：
1. 文件上传接口（支持图片和视频）
2. 文件下载接口
3. 文件元数据查询接口（可选）

**文件大小限制**：
- 图片：最大 5MB
- 视频：最大 50MB

**支持的文件格式**：
- 图片：JPEG, PNG, GIF, WebP
- 视频：MP4, MOV, AVI, WebM

---

## 🔌 接口定义

### 1. 文件上传接口

**接口路径**: `POST /api/media/upload`

**请求头**:
```
Authorization: Bearer {token}
Content-Type: multipart/form-data
```

**请求参数** (multipart/form-data):
- `file` (必需): 加密后的文件二进制数据
- `metadata` (必需): JSON 字符串，包含文件元数据

**metadata 字段说明**:
```json
{
  "sessionId": "会话ID",
  "fileName": "加密后的文件名（Base64编码）",
  "fileSize": 12345678,
  "mimeType": "image/jpeg",
  "messageType": "IMAGE"
}
```

**字段说明**:
- `sessionId`: 文件所属的会话 ID
- `fileName`: 原始文件名（已加密，Base64 编码）
- `fileSize`: 原始文件大小（字节）
- `mimeType`: 文件的 MIME 类型（image/jpeg, video/mp4 等）
- `messageType`: 消息类型，`IMAGE` 或 `VIDEO`

**响应格式** (200 OK):
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "fileId": "file_1234567890abcdef",
    "uploadTime": "2024-02-28T10:30:00Z",
    "fileSize": 12345678,
    "thumbnailUrl": null
  }
}
```

**响应字段说明**:
- `fileId`: 服务器生成的唯一文件 ID（用于后续下载）
- `uploadTime`: 文件上传时间（ISO 8601 格式）
- `fileSize`: 文件大小（字节）
- `thumbnailUrl`: 缩略图 URL（可选，暂不实现）

---

### 2. 文件下载接口

**接口路径**: `GET /api/media/{fileId}`

**请求参数**:
- `fileId` (路径参数): 文件的唯一标识符

**请求头**:
```
Authorization: Bearer {token}
```

**响应格式** (200 OK):
- **Content-Type**: `application/octet-stream`
- **Content-Disposition**: `attachment; filename="encrypted-file"`
- **Body**: 加密的文件二进制数据

**响应头**:
```
X-File-Name: {加密的文件名，Base64编码}
X-File-Size: {文件大小}
X-Mime-Type: {原始MIME类型}
X-Upload-Time: {上传时间}
```

**响应头字段说明**:
- `X-File-Name`: 加密的文件名（Base64 编码）
- `X-File-Size`: 文件大小（字节）
- `X-Mime-Type`: 原始 MIME 类型
- `X-Upload-Time`: 上传时间（ISO 8601 格式）

---

## 🗄️ 数据库设计

### 文件表 (b_media_file)

```sql
CREATE TABLE b_media_file (
  id VARCHAR(64) PRIMARY KEY COMMENT '文件ID',
  session_id VARCHAR(64) NOT NULL COMMENT '会话ID',
  uploader_id VARCHAR(64) NOT NULL COMMENT '上传者用户ID',
  file_name VARCHAR(512) NOT NULL COMMENT '加密的文件名（Base64编码）',
  file_size BIGINT NOT NULL COMMENT '文件大小（字节）',
  mime_type VARCHAR(128) NOT NULL COMMENT 'MIME类型',
  message_type VARCHAR(32) NOT NULL COMMENT '消息类型：IMAGE, VIDEO',
  storage_path VARCHAR(512) NOT NULL COMMENT '文件存储路径',
  thumbnail_path VARCHAR(512) COMMENT '缩略图存储路径（可选）',
  upload_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '上传时间',
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  deleted_at DATETIME COMMENT '软删除时间',
  
  INDEX idx_session_id (session_id),
  INDEX idx_uploader_id (uploader_id),
  INDEX idx_upload_time (upload_time),
  INDEX idx_deleted_at (deleted_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='媒体文件表';
```

**字段说明**:
- `id`: 文件唯一标识符（建议使用 UUID 或雪花算法生成）
- `session_id`: 文件所属的会话 ID
- `uploader_id`: 上传者的用户 ID
- `file_name`: 加密后的文件名（Base64 编码）
- `file_size`: 文件大小（字节）
- `mime_type`: 文件的 MIME 类型
- `message_type`: 消息类型（IMAGE 或 VIDEO）
- `storage_path`: 文件在服务器上的存储路径
- `thumbnail_path`: 缩略图存储路径（可选，暂不实现）
- `upload_time`: 文件上传时间
- `deleted_at`: 软删除标记

## 🔐 权限控制

### 文件上传
- ✅ 用户必须已登录（验证 Token）
- ✅ 用户必须是会话的成员（验证 session_user 表）
- ✅ 文件大小必须在限制范围内
- ✅ 文件类型必须在允许列表中
- ❌ 非会话成员无法上传文件

### 文件下载
- ✅ 用户必须已登录（验证 Token）
- ✅ 用户必须是文件所属会话的成员
- ✅ 文件必须存在且未被删除
- ❌ 非会话成员无法下载文件

## 📝 错误码

### 文件上传接口

| 状态码 | 错误信息 | 说明 |
|--------|---------|------|
| 200 | success | 上传成功 |
| 400 | 文件不能为空 | 未上传文件 |
| 400 | 文件类型不支持 | MIME 类型不在允许列表中 |
| 400 | 图片文件大小不能超过 5MB | 图片文件过大 |
| 400 | 视频文件大小不能超过 50MB | 视频文件过大 |
| 400 | 元数据格式错误 | metadata 不是有效的 JSON |
| 400 | 缺少必需字段: {fieldName} | metadata 缺少必需字段 |
| 401 | 未认证 | Token 无效或过期 |
| 403 | 无权限上传文件到该会话 | 用户不是会话成员 |
| 404 | 会话不存在 | sessionId 无效 |
| 413 | 文件过大 | 文件超过服务器限制 |
| 500 | 文件保存失败 | 服务器存储错误 |

| 状态码 | 错误信息 | 说明 |
|--------|---------|------|
| 200 | success | 下载成功 |
| 401 | 未认证 | Token 无效或过期 |
| 403 | 无权限访问该文件 | 用户不是会话成员 |
| 404 | 文件不存在 | fileId 无效或文件已删除 |
| 500 | 文件读取失败 | 服务器存储错误 |

## 💾 文件存储方案

### 推荐方案：本地文件系统存储

**存储路径结构**:
```
/data/media/
  ├── images/
  │   ├── 2024/
  │   │   ├── 02/
  │   │   │   ├── 28/
  │   │   │   │   ├── file_1234567890abcdef.enc
  │   │   │   │   └── file_abcdef1234567890.enc
  ├── videos/
  │   ├── 2024/
  │   │   ├── 02/
  │   │   │   ├── 28/
  │   │   │   │   └── file_xyz123456789.enc
```

**路径生成规则**:
```
/{messageType}/{year}/{month}/{day}/{fileId}.enc
```

**示例**:
- 图片: `/data/media/images/2024/02/28/file_1234567890abcdef.enc`
- 视频: `/data/media/videos/2024/02/28/file_xyz123456789.enc`

**优点**:
- 简单易实现
- 性能好
- 成本低
- 易于备份

**注意事项**:
1. 确保存储目录有足够的磁盘空间
2. 定期清理已删除的文件（软删除后 30 天）
3. 考虑文件权限设置（仅应用可读写）
4. 建议配置文件存储路径为环境变量

### 可选方案：对象存储（OSS/S3）

- 阿里云 OSS

**优点**:
- 高可用性
- 自动扩展
- CDN 加速

**缺点**:
- 成本较高
- 需要额外配置

## 🔄 业务流程

### 文件上传流程

```
1. 前端选择文件
   ↓
2. 前端验证文件类型和大小
   ↓
3. 前端使用会话密钥加密文件
   ↓
4. 前端加密文件名
   ↓
5. 前端发送 multipart/form-data 请求
   ↓
6. 后端验证 Token 和权限
   ↓
7. 后端验证文件大小和类型
   ↓
8. 后端生成唯一 fileId
   ↓
9. 后端保存文件到存储路径
   ↓
10. 后端保存文件元数据到数据库
    ↓
11. 后端返回 fileId 和上传时间
    ↓
12. 前端创建媒体消息
    ↓
13. 前端通过 WebSocket 发送消息
```

### 文件下载流程

```
1. 前端接收到媒体消息
   ↓
2. 前端显示文件占位符
   ↓
3. 用户点击查看文件
   ↓
4. 前端发送下载请求（GET /api/media/{fileId}）
   ↓
5. 后端验证 Token 和权限
   ↓
6. 后端从数据库查询文件元数据
   ↓
7. 后端读取文件内容
   ↓
8. 后端返回文件流和响应头
   ↓
9. 前端接收加密文件
   ↓
10. 前端使用会话密钥解密文件
    ↓
11. 前端显示图片或播放视频
    ↓
12. 前端缓存解密后的文件到 IndexedDB
```

## 💻 前端调用示例

### 上传文件

```typescript
// 文件上传
async function uploadFile(file: File, sessionId: string): Promise<string> {
  // 1. 加密文件
  const encryptedFile = await cryptoModule.encryptFile(file, sessionKey)
  
  // 2. 加密文件名
  const encryptedFileName = await cryptoModule.encryptText(file.name, sessionKey)
  const base64FileName = btoa(encryptedFileName)
  
  // 3. 准备表单数据
  const formData = new FormData()
  formData.append('file', new Blob([encryptedFile]), 'encrypted.bin')
  formData.append('metadata', JSON.stringify({
    sessionId,
    fileName: base64FileName,
    fileSize: file.size,
    mimeType: file.type,
    messageType: file.type.startsWith('image/') ? 'IMAGE' : 'VIDEO'
  }))
  
  // 4. 上传文件
  const response = await fetch(`${API_BASE_URL}/api/media/upload`, {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${token}`
    },
    body: formData
  })
  
  const result = await response.json()
  return result.data.fileId
}
```

### 下载文件

```typescript
// 文件下载
async function downloadFile(fileId: string): Promise<Blob> {
  // 1. 下载加密文件
  const response = await fetch(`${API_BASE_URL}/api/media/${fileId}`, {
    headers: {
      'Authorization': `Bearer ${token}`
    }
  })
  
  // 2. 获取文件元数据
  const fileName = response.headers.get('X-File-Name')
  const mimeType = response.headers.get('X-Mime-Type')
  
  // 3. 读取加密数据
  const encryptedData = await response.arrayBuffer()
  
  // 4. 解密文件
  const decryptedData = await cryptoModule.decryptFile(encryptedData, sessionKey)
  
  // 5. 创建 Blob
  return new Blob([decryptedData], { type: mimeType })
}
```

## 🎯 业务规则

### 文件上传
1. **文件验证**: 必须验证文件类型和大小
2. **权限验证**: 必须验证用户是会话成员
3. **文件加密**: 前端负责加密，后端存储加密文件
4. **文件名加密**: 文件名也需要加密（Base64 编码）
5. **原子操作**: 文件保存和数据库记录插入应在事务中执行
6. **错误回滚**: 如果数据库插入失败，应删除已保存的文件

### 文件下载
1. **权限验证**: 必须验证用户是会话成员
2. **流式传输**: 大文件应使用流式传输，避免内存溢出
3. **缓存控制**: 可以设置适当的缓存头（Cache-Control）
4. **断点续传**: 可选支持 Range 请求（优先级低）

### 文件删除
1. **软删除**: 建议使用软删除，保留文件一段时间
2. **定期清理**: 定期清理软删除超过 30 天的文件
3. **级联删除**: 删除会话时，应标记该会话的所有文件为已删除

## 📌 注意事项

### 安全性
1. **文件加密**: 所有文件必须在客户端加密后上传
2. **权限控制**: 严格验证用户权限，防止越权访问
3. **文件扫描**: 建议对上传的文件进行病毒扫描（可选）
4. **路径遍历**: 防止路径遍历攻击，验证 fileId 格式

### 性能优化
1. **流式处理**: 使用流式处理大文件，避免一次性加载到内存
2. **异步处理**: 文件保存可以异步处理，快速返回响应
3. **CDN 加速**: 如果使用对象存储，可以配置 CDN 加速下载
4. **压缩传输**: 可以考虑使用 gzip 压缩传输（对已加密文件效果有限）

### 存储管理
1. **磁盘空间**: 监控磁盘空间，及时清理过期文件
2. **备份策略**: 定期备份重要文件
3. **存储路径**: 配置为环境变量，便于迁移
4. **文件权限**: 确保应用有读写权限，但其他用户无权访问

### 错误处理
1. **详细日志**: 记录所有上传和下载操作的日志
2. **友好提示**: 返回清晰的错误信息
3. **重试机制**: 前端应实现上传失败重试
4. **事务回滚**: 确保数据一致性
