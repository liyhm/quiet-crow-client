# 后端需求：群成员列表接口

## 📋 需求概述

前端需要一个接口来获取群聊的成员列表，用于在"潜伏者名单"弹窗中显示群成员信息。

---

## 🔌 接口定义

### 获取群成员列表

**接口路径**: `GET /api/session/{sessionId}/members`

**请求参数**:
- `sessionId` (路径参数): 会话ID（群聊的 session ID）

**请求头**:
```
Authorization: Bearer {token}
```

**响应格式**:
```json
[
  {
    "userId": "user123",
    "showName": "张三",
    "userType": "OWNER",
    "isOnline": true
  },
  {
    "userId": "user456",
    "showName": "李四",
    "userType": "MEMBER",
    "isOnline": false
  }
]
```

**字段说明**:
- `userId`: 用户ID
- `showName`: 显示名称（优先使用 displayName，如果没有则使用 username）
- `userType`: 用户类型
  - `OWNER`: 群主
  - `MEMBER`: 普通成员
- `isOnline`: 是否在线（可以根据 WebSocket 连接状态判断）

---

## 🗄️ 数据库查询逻辑

### 1. 验证会话类型
```sql
SELECT session_type FROM b_session WHERE id = {sessionId}
```
- 如果 `session_type != 'GROUP'`，返回 400 错误："该会话不是群聊"

### 2. 验证用户权限
```sql
SELECT COUNT(*) FROM b_session_user 
WHERE session_id = {sessionId} 
  AND user_id = {currentUserId}
  AND deleted_at IS NULL
```
- 如果用户不在群聊中，返回 403 错误："无权查看该群聊成员"

### 3. 查询群成员列表
```sql
SELECT 
  su.user_id,
  u.display_name,
  u.username,
  su.session_user_type,
  u.status
FROM b_session_user su
JOIN b_user u ON su.user_id = u.id
WHERE su.session_id = {sessionId}
  AND su.deleted_at IS NULL
ORDER BY 
  CASE WHEN su.session_user_type = 'OWNER' THEN 0 ELSE 1 END,
  su.created_at ASC
```

**排序规则**:
1. 群主排在最前面
2. 其他成员按加入时间排序

---

## 🔐 权限控制

- 只有群成员才能查看群成员列表
- 已退出群聊的用户（deleted_at 不为空）无法查看
