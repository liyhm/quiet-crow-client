# 后端优化建议 - 头像显示问题

## 问题描述

前端上传头像成功，但无法显示。

**原因**: 后端返回的头像 URL 是加密文件（`.enc` 结尾）
```
http://127.0.0.1:9005/img-rtm/image/2026/03/04/xxx.enc
```

浏览器的 `<img>` 标签无法直接显示加密图片。

---

## 解决方案

### 方案 1: 头像不加密（推荐）⭐

**理由**:
- 头像是公开信息，不需要加密
- 需要在多处展示（会话列表、联系人、群成员等）
- 加密会导致性能开销和用户体验差

**修改点**:
- 头像上传接口：保存明文图片（`.jpg`、`.png`）
- 返回明文 URL，不带 `.enc` 后缀

---

### 方案 2: 提供解密接口

如果必须加密头像，需要提供解密接口：

**新增接口**:
```
GET /api/media/avatar/{fileId}
```

**功能**:
- 接收加密文件 ID
- 后端解密
- 返回明文图片流（`image/jpeg` 或 `image/png`）

**前端调整**:
- 头像 URL 改为：`http://127.0.0.1:9005/api/media/avatar/{fileId}`
- 浏览器可以直接显示

---

## 推荐方案

**使用方案 1**，因为：
1. 实现简单，无需额外接口
2. 性能最优，无解密开销
3. 符合业界惯例（头像通常不加密）
4. 用户体验最好

## 其他优化建议

### 1. 未读数计算
确保 SQL 查询排除发送者自己的消息：
```sql
WHERE session_id = ? 
AND id > watermark 
AND send_user != current_user  -- 关键
```

### 2. 发送消息时更新水位线
发送消息后立即更新发送者的水位线，避免自己的消息被计入未读数。

### 3. 好友请求系统消息
如果接受好友请求时自动发送系统消息，确保系统消息不计入未读数。

## 📡 WebSocket 优化

### 1. 认证优化
`java
@Override
public void configureClientInboundChannel(ChannelRegistration registration) {
    registration.interceptors(new ChannelInterceptor() {
        @Override
        public Message<?> preSend(Message<?> message, MessageChannel channel) {
            StompHeaderAccessor accessor = StompHeaderAccessor.wrap(message);
            if (StompCommand.CONNECT.equals(accessor.getCommand())) {
                String token = accessor.getFirstNativeHeader("Authorization");
                // 验证 token
                User user = jwtService.validateToken(token);
                accessor.setUser(new UserPrincipal(user));
            }
            return message;
        }
    });
}
`
