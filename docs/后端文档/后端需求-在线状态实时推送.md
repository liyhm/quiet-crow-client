# 后端需求：在线状态实时推送

## 📋 需求背景

**用户反馈**：
- "前端好友列表要手动刷新才能更新在线状态，不好"
- "如果有一方退出了，前端要能快速更新这个状态"

**当前问题**：
- 用户登录/退出时，其他在线用户的好友列表不会自动更新
- 需要手动刷新（点击联系人标签）才能看到最新的在线状态
- 用户体验不好，不够实时

**期望效果**：
- 用户 A 登录时，所有在线好友（B、C、D...）的好友列表自动显示 A 在线
- 用户 A 退出时，所有在线好友的好友列表自动显示 A 离线
- 无需手动刷新，实时更新

## ✅ 解决方案

### 核心思路

通过 WebSocket 推送在线状态变化通知给所有相关的在线好友。

### 推送时机

1. **用户登录时**
   - 查询该用户的所有好友
   - 向所有在线好友推送：该用户已上线

2. **用户退出时**
   - 查询该用户的所有好友
   - 向所有在线好友推送：该用户已离线

3. **WebSocket 连接时**
   - 查询该用户的所有好友
   - 向所有在线好友推送：该用户已上线

4. **WebSocket 断开时**
   - 查询该用户的所有好友
   - 向所有在线好友推送：该用户已离线

## 🔧 技术实现

### 1. 创建在线状态通知服务

```java
@Service
@Slf4j
public class OnlineStatusNotificationService {
    
    @Autowired
    private SimpMessagingTemplate messagingTemplate;
    
    @Autowired
    private ContactService contactService;
    
    @Autowired
    private OnlineStatusService onlineStatusService;
    
    /**
     * 通知所有在线好友：某用户上线了
     */
    public void notifyFriendsUserOnline(String userId, String username) {
        log.info("🔔 [OnlineStatus] 通知好友：用户 {} 已上线", username);
        
        // 1. 查询该用户的所有好友
        List<String> friendIds = contactService.getFriendIds(userId);
        log.info("🔔 [OnlineStatus] 用户 {} 有 {} 个好友", username, friendIds.size());
        
        // 2. 构建通知消息
        OnlineStatusNotification notification = OnlineStatusNotification.builder()
            .userId(userId)
            .username(username)
            .isOnline(true)
            .timestamp(System.currentTimeMillis())
            .build();
        
        // 3. 向所有在线好友推送通知
        int onlineCount = 0;
        for (String friendId : friendIds) {
            // 检查好友是否在线
            if (onlineStatusService.isUserOnline(friendId)) {
                messagingTemplate.convertAndSendToUser(
                    friendId,
                    "/queue/online-status",
                    notification
                );
                onlineCount++;
            }
        }
        
        log.info("✅ [OnlineStatus] 已通知 {} 个在线好友", onlineCount);
    }
    
    /**
     * 通知所有在线好友：某用户离线了
     */
    public void notifyFriendsUserOffline(String userId, String username) {
        log.info("🔔 [OnlineStatus] 通知好友：用户 {} 已离线", username);
        
        // 1. 查询该用户的所有好友
        List<String> friendIds = contactService.getFriendIds(userId);
        log.info("🔔 [OnlineStatus] 用户 {} 有 {} 个好友", username, friendIds.size());
        
        // 2. 构建通知消息
        OnlineStatusNotification notification = OnlineStatusNotification.builder()
            .userId(userId)
            .username(username)
            .isOnline(false)
            .timestamp(System.currentTimeMillis())
            .build();
        
        // 3. 向所有在线好友推送通知
        int onlineCount = 0;
        for (String friendId : friendIds) {
            // 检查好友是否在线
            if (onlineStatusService.isUserOnline(friendId)) {
                messagingTemplate.convertAndSendToUser(
                    friendId,
                    "/queue/online-status",
                    notification
                );
                onlineCount++;
            }
        }
        
        log.info("✅ [OnlineStatus] 已通知 {} 个在线好友", onlineCount);
    }
}
```

```java
@Service
public class ContactService {
    
    /**
     * 获取用户的所有好友ID列表
     */
    public List<String> getFriendIds(String userId) {
        // 查询该用户的所有好友关系（状态为 ACCEPTED）
        List<ContactRelation> relations = contactRelationRepository
            .findByUserIdAndStatus(userId, ContactStatus.ACCEPTED);
        
        // 提取好友ID
        return relations.stream()
            .map(ContactRelation::getToUserId)
            .collect(Collectors.toList());
    }
}
```

### 5. 在 WebSocket 事件监听器中调用通知

```java
@Component
@Slf4j
public class WebSocketEventListener {
    
    @Autowired
    private OnlineStatusNotificationService onlineStatusNotificationService;
    
    @EventListener
    public void handleWebSocketConnectListener(SessionConnectedEvent event) {
        StompHeaderAccessor headerAccessor = StompHeaderAccessor.wrap(event.getMessage());
        String userId = getUserIdFromSession(headerAccessor);
        String username = getUsernameFromSession(headerAccessor);
        
        if (userId != null) {
            log.info("🟢 [WebSocket] 用户 {} 已连接", username);
            
            // 标记用户在线
            onlineStatusService.markUserOnline(userId);
            
            // 🔔 通知所有在线好友：该用户已上线
            onlineStatusNotificationService.notifyFriendsUserOnline(userId, username);
        }
    }
    
    @EventListener
    public void handleWebSocketDisconnectListener(SessionDisconnectEvent event) {
        StompHeaderAccessor headerAccessor = StompHeaderAccessor.wrap(event.getMessage());
        String userId = getUserIdFromSession(headerAccessor);
        String username = getUsernameFromSession(headerAccessor);
        
        if (userId != null) {
            log.info("🔴 [WebSocket] 用户 {} 已断开", username);
            
            // 标记用户离线
            onlineStatusService.markUserOffline(userId);
            
            // 🔔 通知所有在线好友：该用户已离线
            onlineStatusNotificationService.notifyFriendsUserOffline(userId, username);
        }
    }
}
```

## 📡 WebSocket 消息格式

### 订阅地址

```

```

### 消息格式

```json
{
  "userId": "4fda11c3d3ff5cee46e7380b7b193e56",
  "username": "lyh2",
  "isOnline": true,
  "timestamp": 1709558400000
}
```

### 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|

## 🔄 完整流程

### 用户登录流程

```
1. 用户 A 登录
   ↓
2. AuthService.login()
   ↓
3. 标记用户 A 在线
   Redis: SET online:user:{userA} {timestamp}
   ↓
4. 查询用户 A 的所有好友
   假设有好友 B、C、D
   ↓
5. 检查好友是否在线
   B: 在线 ✅
   C: 离线 ❌
   D: 在线 ✅
   ↓
6. 向在线好友推送通知
   → 推送给 B: {"userId":"A","isOnline":true}
   → 推送给 D: {"userId":"A","isOnline":true}
   ↓
7. 好友 B、D 的前端收到通知
   ↓
8. 自动更新好友列表
   B 的好友列表：A 显示在线 🟢
   D 的好友列表：A 显示在线 🟢
```

### 用户退出流程

```
1. 用户 A 退出登录
   ↓
2. AuthService.logout()
   ↓
3. 标记用户 A 离线
   Redis: DEL online:user:{userA}
   ↓
4. 查询用户 A 的所有好友
   假设有好友 B、C、D
   ↓
5. 检查好友是否在线
   B: 在线 ✅
   C: 离线 ❌
   D: 在线 ✅
   ↓
6. 向在线好友推送通知
   → 推送给 B: {"userId":"A","isOnline":false}
   → 推送给 D: {"userId":"A","isOnline":false}
   ↓
7. 好友 B、D 的前端收到通知
   ↓
8. 自动更新好友列表
   B 的好友列表：A 显示离线 ⚫
   D 的好友列表：A 显示离线 ⚫
```

## 📊 性能考虑

### 优化点

1. **批量查询好友**
   - 一次查询获取所有好友ID
   - 避免 N+1 查询问题

2. **只推送给在线好友**
   - 推送前检查好友是否在线
   - 避免无效推送

3. **异步推送**
   - 推送操作不阻塞主流程
   - 使用 `@Async` 注解（可选）

4. **日志控制**
   - 生产环境可以降低日志级别
   - 避免大量日志影响性能

### 预期性能

- 单次推送延迟：< 100ms
- 支持并发推送：1000+ 用户
- Redis 查询：< 10ms
- WebSocket 推送：< 50ms

## ⚠️ 注意事项

### 1. 好友关系查询

确保 `ContactService.getFriendIds()` 只返回状态为 `ACCEPTED` 的好友：

```java
List<ContactRelation> relations = contactRelationRepository
    .findByUserIdAndStatus(userId, ContactStatus.ACCEPTED);
```

### 2. 双向好友关系

如果好友关系是双向的（A 添加 B，B 也添加 A），需要查询两个方向：

```java
public List<String> getFriendIds(String userId) {
    // 查询我添加的好友
    List<String> friends1 = contactRelationRepository
        .findByUserIdAndStatus(userId, ContactStatus.ACCEPTED)
        .stream()
        .map(ContactRelation::getToUserId)
        .collect(Collectors.toList());
    
    // 查询添加我的好友
    List<String> friends2 = contactRelationRepository
        .findByToUserIdAndStatus(userId, ContactStatus.ACCEPTED)
        .stream()
        .map(ContactRelation::getUserId)
        .collect(Collectors.toList());
    
    // 合并去重
    Set<String> allFriends = new HashSet<>();
    allFriends.addAll(friends1);
    allFriends.addAll(friends2);
    
    return new ArrayList<>(allFriends);
}
```

### 3. 异常处理

推送失败不应该影响主流程：

```java
try {
    onlineStatusNotificationService.notifyFriendsUserOnline(userId, username);
} catch (Exception e) {
    log.error("❌ 推送在线状态通知失败", e);
    // 不抛出异常，不影响登录流程
}
```

### 4. 用户名获取

确保在 WebSocket 事件监听器中能获取到用户名：

