# 后端需求：群名称修改接口

## 需求概述
前端已实现群名称修改功能，需要后端提供相应的 API 接口支持。所有群成员都可以修改群名称。

## API 接口规范

### 1. 更新群名称

**接口地址**
```
PUT /api/session/{sessionId}/name
```

**请求方式**
```
PUT
```

**路径参数**
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| sessionId | String | 是 | 群聊会话 ID |

**请求头**
```
Authorization: Bearer {token}
Content-Type: application/json
```

**请求体**
```json
{
  "name": "新的群名称"
}
```

**请求参数说明**
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| name | String | 是 | 新的群名称，长度限制 1-50 字符 |

**成功响应**
```json
{
  "sessionId": "session_123456",
  "name": "新的群名称",
  "updatedBy": "user_789",
  "updatedAt": "2024-03-04T10:30:00Z"
}
```

**响应参数说明**
| 参数名 | 类型 | 说明 |
|--------|------|------|
| sessionId | String | 会话 ID |
| name | String | 更新后的群名称 |
| updatedBy | String | 修改者的用户 ID |
| updatedAt | String | 更新时间（ISO 8601 格式） |

**错误响应**

1. 会话不存在
```json
{
  "code": 404,
  "message": "会话不存在"
}
```

2. 非群聊会话
```json
{
  "code": 400,
  "message": "该会话不是群聊"
}
```

3. 用户不是群成员
```json
{
  "code": 403,
  "message": "您不是该群的成员"
}
```

4. 群名称为空或过长
```json
{
  "code": 400,
  "message": "群名称长度必须在 1-50 字符之间"
}
```

5. 未授权
```json
{
  "code": 401,
  "message": "未授权，请先登录"
}
```

## 业务逻辑要求

### 权限验证
- 验证用户是否已登录（通过 JWT token）
- 验证会话是否存在
- 验证会话类型是否为群聊（GROUP）
- 验证用户是否为该群成员
- **注意：所有群成员都可以修改群名称，不限于群主**

### 数据验证
- 群名称不能为空
- 群名称长度：1-50 字符
- 去除首尾空格后再验证

### 数据更新
1. 更新 `session` 表中的群名称字段
2. 记录修改者和修改时间
3. 返回更新后的信息

### WebSocket 通知（可选但推荐）
更新成功后，通过 WebSocket 向该群的所有在线成员推送群名称变更通知：

```json
{
  "type": "GROUP_NAME_UPDATED",
  "data": {
    "sessionId": "session_123456",
    "name": "新的群名称",
    "updatedBy": "user_789",
    "updatedByName": "张三",
    "updatedAt": "2024-03-04T10:30:00Z"
  }
}
```

这样其他成员可以实时看到群名称的变化，无需刷新。

## 数据库设计建议

如果 `session` 表还没有相关字段，建议添加：

```sql
ALTER TABLE session ADD COLUMN updated_by VARCHAR(50);
ALTER TABLE session ADD COLUMN updated_at TIMESTAMP;
```

或者在现有的 `updated_at` 字段基础上，添加 `updated_by` 字段记录最后修改者。

## 前端调用示例

前端已实现的调用代码：

```typescript
// src/renderer/src/shared/api/session.ts
updateGroupName(sessionId: string, name: string) {
  return request.put<any, { sessionId: string; name: string }>(
    API_ENDPOINTS.SESSION.UPDATE_NAME(sessionId),
    { name }
  )
}

// src/renderer/src/pages/main/MainPage.vue
const handleUpdateGroupName = async (newName: string): Promise<void> => {
  if (!activeSession.value?.id) return

  try {
    console.log('🔨 更新群名称:', newName)
    await sessionApi.updateGroupName(activeSession.value.id, newName)
    
    // 更新本地显示
    currentGroupName.value = newName
    
    // 刷新会话列表
    await sessionStore.loadSessions()
    
    showAlert('群名称已更新', 'success')
  } catch (error) {
    console.error('❌ 更新群名称失败:', error)
    showAlert('更新失败: ' + (error instanceof Error ? error.message : String(error)), 'error')
  }
}
```

## 测试用例

### 正常流程
1. 群成员点击群名称进入编辑模式
2. 输入新的群名称
3. 按 Enter 或失焦保存
4. 后端验证并更新
5. 返回成功响应
6. 前端显示成功提示并刷新会话列表

### 异常流程
1. 输入空白群名称 → 返回 400 错误
2. 输入超长群名称（>50字符） → 返回 400 错误
3. 非群成员尝试修改 → 返回 403 错误
4. 未登录用户尝试修改 → 返回 401 错误
5. 对私聊会话尝试修改 → 返回 400 错误

## 注意事项

1. **权限控制**：所有群成员都可以修改，不需要是群主
2. **并发控制**：如果多个成员同时修改，以最后一次请求为准
3. **历史记录**：建议保留群名称修改历史（可选）
4. **通知机制**：建议通过 WebSocket 实时通知其他成员
5. **字符编码**：支持中文、英文、emoji 等 UTF-8 字符
6. **XSS 防护**：后端需要对群名称进行 HTML 转义，防止 XSS 攻击
