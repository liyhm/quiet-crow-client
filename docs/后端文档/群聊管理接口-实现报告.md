public class InviteMembersResponse {

### 2. Service 层实现

#### 邀请成员方法

**方法签名**:
```java
@Transactional
public InviteMembersResponse inviteMembers(String sessionId, String currentUserId, List<String> memberIds)
```

**实现逻辑**:
1. ✅ 验证当前用户是否是群成员（所有成员都可以邀请）
2. ✅ 验证会话是否是群聊
3. ✅ 查询当前群成员列表
4. ✅ 批量邀请成员
   - 检查用户是否存在
   - 检查用户是否已在群内
   - 创建 SessionUser 记录
5. ✅ 返回邀请结果（成功数量 + 失败列表）

**特性**:
- 使用事务保证原子性
- 自动跳过已在群内的用户
- 自动跳过不存在的用户
- 返回详细的失败原因
- **所有群成员都可以邀请新成员（不限于群主）**

---

#### 解散群聊方法

**方法签名**:
```java
@Transactional
public DissolveGroupResponse dissolveGroup(String sessionId, String currentUserId)
```

**实现逻辑**:
1. ✅ 验证当前用户是否是群主
2. ✅ 验证会话是否是群聊
3. ✅ 查询所有群成员
4. ✅ 软删除所有成员关系
   - 设置 `deleted_time`
   - 设置 `delete_type = "DISSOLVED"`
   - 设置 `is_visible = false`
5. ✅ 返回解散结果

**特性**:
- 使用软删除，保留历史记录
- 所有成员同时被移除
- 会话从所有人的列表中消失
- 记录解散时间

### 3. Controller 层实现

#### 邀请成员

**接口**: `POST /api/session/{sessionId}/invite`

**权限控制**:
- ✅ 需要 JWT Token 认证
- ✅ **所有群成员都可以操作（不限于群主）**

**错误处理**:
- ✅ 400: 用户列表不能为空
- ✅ 400: 该会话不是群聊
- ✅ 403: 只有群成员可以邀请成员
- ✅ 404: 会话不存在

---

#### 解散群聊接口

**接口**: `DELETE /api/session/{sessionId}/dissolve`

**权限控制**:
- ✅ 需要 JWT Token 认证
- ✅ 只有群主可以操作

**错误处理**:
- ✅ 400: 该会话不是群聊
- ✅ 403: 只有群主可以解散群聊
- ✅ 404: 会话不存在

## 🎯 功能特性

### 邀请成员
1. **批量邀请**: 支持一次邀请多个用户
2. **智能去重**: 自动跳过已在群内的用户
3. **用户验证**: 自动验证用户是否存在
4. **详细反馈**: 返回成功数量和失败原因
5. **事务保证**: 使用事务保证数据一致性
6. **权限开放**: 所有群成员都可以邀请新成员

### 解散群聊
1. **软删除**: 保留历史记录，不物理删除
2. **全员移除**: 所有成员同时被移除
3. **会话隐藏**: 会话从所有人的列表中消失
4. **时间记录**: 记录解散的具体时间
5. **类型标记**: 标记删除类型为 "DISSOLVED"

## 💻 前端调用示例

### JavaScript/TypeScript
```typescript
// 邀请成员
const inviteMembers = async (sessionId: string, memberIds: string[]) => {
  try {
    const response = await sessionApi.inviteMembers(sessionId, memberIds);
    console.log(`成功邀请 ${response.invitedCount} 人`);
    
    if (response.failedUsers.length > 0) {
      console.log('邀请失败的用户:');
      response.failedUsers.forEach(user => {
        console.log(`- ${user.userId}: ${user.reason}`);
      });
    }
    
    return response;
  } catch (error) {
    console.error('邀请失败:', error);
    throw error;
  }
};

// 解散群聊
const dissolveGroup = async (sessionId: string) => {
  if (!confirm('确定要解散这个群聊吗？此操作不可恢复！')) {
    return;
  }
  
  try {
    const response = await sessionApi.dissolveGroup(sessionId);
    console.log(`群聊已于 ${response.dissolvedAt} 解散`);
    
    // 刷新会话列表
    await loadSessions();
    
    return response;
  } catch (error) {
    console.error('解散失败:', error);
    throw error;
  }
};
```

### Vue 3
```vue
<script setup>
import { ref } from 'vue';
import { sessionApi } from '@/api/session';

const selectedMembers = ref([]);
const sessionId = 'group-123';

// 邀请成员（所有群成员都可以调用）
const handleInvite = async () => {
  const response = await sessionApi.inviteMembers(sessionId, selectedMembers.value);
  alert(`成功邀请 ${response.invitedCount} 人`);
  
  if (response.failedUsers.length > 0) {
    const failed = response.failedUsers.map(u => `${u.userId}: ${u.reason}`).join('\n');
    alert(`以下用户邀请失败:\n${failed}`);
  }
};

// 解散群聊（仅群主可以调用）
const handleDissolve = async () => {
  if (!confirm('确定要解散群聊吗？')) return;
  
  await sessionApi.dissolveGroup(sessionId);
  alert('群聊已解散');
};
</script>
```

## 🔄 业务流程

### 邀请成员流程
```
1. 任意群成员点击"邀请成员"
   ↓
2. 选择要邀请的用户
   ↓
3. 调用 POST /api/session/{sessionId}/invite
   ↓
4. 后端验证是否是群成员
   ↓
5. 批量创建 SessionUser 记录
   ↓
6. 返回邀请结果
   ↓
7. 前端显示成功/失败信息
   ↓
8. 刷新群成员列表
```

### 解散群聊流程
```
1. 群主点击"解散群聊"
   ↓
2. 前端二次确认
   ↓
3. 调用 DELETE /api/session/{sessionId}/dissolve
   ↓
4. 后端验证是否是群主
   ↓
5. 软删除所有成员关系
   ↓
6. 返回解散时间
   ↓
7. 前端显示成功信息
   ↓
8. 从会话列表移除该群聊
```
